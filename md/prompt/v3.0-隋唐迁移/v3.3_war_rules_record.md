# v3.3 军队、兵种、粮道、围城和战术规则实现记录

完成日期：2026-07-05

性质：规则与显示最小迁移。本文记录本轮真实改动；不表示胜利规则、完整内政外交、发布级 UI 或真实 LLM 已完成。

## 1. 目标和非目标

目标：

- 将 `ComponentType` 扩展为能直接表达隋唐兵种：步卒、骑军、弓弩、攻城器械、亲军、水师、乡兵。
- 保留 `tank` / `motorizedInfantry` / `artillery` 等 legacy rawValue，避免破坏阿登数据和旧测试语义。
- 让 `suitang_unit_templates.json` 使用新兵种 rawValue，不再用 motorized / artillery / tank 作为隋唐模板桥。
- 建立最小围城判断：城池/关隘守军在无粮道且敌军邻接时视为断粮被围。
- 将战术、军队、补给、经济资源和主要检查器显示迁移到隋唐口径。

非目标：

- 不大规模重命名 `Division`、`ProductionKind`、`EconomyResources` 等源码兼容类型。
- 不新增复杂士气、疲劳、水战、归附、天命或完整围城状态机。
- 不让围城、补员或生产绕过 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- 不修改测试文件，不运行本机 Xcode build / XCTest / 模拟器。

## 2. 规则变更

兵种：

- `ComponentType` 新增 `cavalry`、`archer`、`siegeEngine`、``guard``、`naval`、`militia`。
- 新增 `displayName` / `displayCode` / `isMobileShock` / `isRangedSupport` / `isSiegeCapable`。
- `Division` 新增 `hasCavalryShock`、`isMobileForce`、`hasRangedSupport`、`hasSiegeCapability`、`primaryComponentType`、`unitKindDisplayName`。
- AI 机动单位和远程支援判断改读新语义，但 directive 和底层命令仍走原管线。

战斗：

- 骑军/甲骑在平原获得机动冲击加成，在复杂地形受限制。
- 攻城器械攻击城池/关隘获得小幅攻城加成。
- 被围城池/关隘守军在防御计算中受到额外下调；既有低补给/包围对 `effectiveStats` 的影响仍保留。

粮道与围城：

- `SupplyRules.isBesieged` 从 hex 和单位状态派生，不新增独立权威状态。
- 判断条件：无己方粮道、所在 hex 为城池/关隘或 region 代表城邑、相邻有外交敌对单位。
- `SupplyRules.supplyState` 会将被围守军归入 `encircled`，并在状态变化时写围城日志。
- `EconomyRules.applyAutomaticReinforcement` 明确排除被围守军，安全后方补员仍从粮道和敌邻接派生。

经济与生产：

- `EconomyResources.displaySummary` 将字段显示为丁口、军械、粮草。
- `ProductionKind.displayName` 映射为募兵步军、骑军行营、轻骑游军、弓弩器械营、粮草转运。
- 新生产单位使用隋唐兵种 component，但 `ProductionKind` rawValue 保留旧 schema。

## 3. 数据变更

更新 `WWIIHexV0/Data/suitang_unit_templates.json`：

- `suitang_infantry_host`：步卒 + 弓弩 + 少量骑军。
- `suitang_cavalry_column`：骑军为主。
- `suitang_archer_camp`：弓弩为主。
- `suitang_siege_train`：攻城器械为主。
- `suitang_garrison`：亲军 / 步卒 / 弓弩。
- `suitang_frontier_raiders`：骑军 / 乡兵 / 弓弩。

`DataLoader.fallbackComponents(for:)` 同步识别上述 `suitang_*` 模板，避免 template 文件缺失时退回纯步兵。

## 4. 显示变更

已迁移主要可见路径：

- HUD：回合、势力、阶段、胜负、丁口、军械、粮草、征发。
- EconomyPanel：府库、征发队列和资源耗费。
- UnitInspector / UnitTooltip / UnitNode：军队、兵种、粮道、退守、状态、军牌短码。
- RegionInspector：州郡、城邑、关隘、粮仓、军械作坊、产出。
- AgentPanel：方面指令使用 `TacticName.displayName`，不再直接显示 tactic rawValue。
- GeneralCommandPanel：总管军令、固守、进军、所属军队等中文显示。
- `GamePhase.displayName` 和 `BaseTerrain.displayName` 改为通用中文。

## 6. 风险和后续

风险：

- 本轮只做单文件语法 parse，未做 Xcode 编译，SwiftUI/SpriteKit 符号可用性仍需云端或授权重测确认。
- 围城是派生式最小规则，没有独立 siege progress、士气、破城或投降状态。
- `VictoryRules` 仍是阿登语义，`wude_618` 的胜负目标仍需后续版本消费。
- `AgentConfiguration.guderian`、旧 sample state、旧测试和旧阿登数据仍保留二战兼容语义。

- v3.6 再做发布级 UI/美术收口，不在规则迁移轮强扩视觉资产。

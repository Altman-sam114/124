# v3.7-preflight.50 复核面板与记录摘要可见文案收口记录

完成日期：2026-07-06

## 目标

继续推进完整 v3.7 发布候选前置收口，处理战局复核面板、MapEditor 资源面板、战报面板、朝堂面板和外交面板中仍可能进入玩家视野的发布工程口径、技术兜底词和历史记录摘要直出问题。

本轮使用两个并发子 Agent 做只读扫描：

- 子 Agent Noether：扫描 MapEditor、战局复核和发布相关入口，定位“已接入 / 结构化记录 / 只读记录 / 审计记录 / 真实模型接入 / 导出到内存 / 资源目录”等可见工程口径。
- 子 Agent Kepler：扫描战报、朝堂和外交面板，定位“诊断 / 保底 / Hex 变体”和历史摘要直出路径。

## 改动范围

- `ReleaseChecklistView` 的当前可用、后续功能和说明文案改为玩家视角，减少“已接入、结构化、只读、审计、真实模型、观察口径”等发布工程词。
- `MapEditorView` 的“导出战局数据到内存”改为“生成战局数据预览”，“资源目录已定位”改为“默认资源已连接”，“胜负点编号”改为“胜负点代号”。
- `MapEditorGameResourceBridge` 缺资源错误不再暴露具体 JSON 文件名，改为“缺少默认战局资源”。
- `AppContainer` 的 TurnManager provider 展示名从“本地模拟朝堂”改为“朝堂系统”。
- `EventLogView` 与 `AgentPanelView` 的展示净化表把 `fallback` 显示为“备用处置”、`diagnostic` 显示为“军情说明”，补齐 `Hex` / `Hexes` / `HexTile` / `hexToTheater` 等变体，并把 raw JSON 相关展示改为“军情记录”。
- `AgentPanelView` 中缺 directive type 的方面军令摘要从“诊断”改为“军情说明”。
- `DiplomacyPanelView` 对外交事件、归附交接、善后压力、善后处置、君主理由和使者摘要统一套用可见文本净化，避免旧存档或历史记录中的工程词直出。

## 明确非目标

- 不改 Swift 类型名、内部函数名、record id、JSON key、schema、rawValue、资源文件名或历史兼容 id。
- 不改 MapEditor 导出结构、加载顺序、胜负条件、地图数据、单位数据、规则数值或命令执行。
- 不改 AI 决策策略、`Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 管线。
- 不跑 Xcode build、XCTest、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full、性能测试或 app 启动。

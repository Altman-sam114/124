# v3.1 多势力、外交关系和通用回合阶段实现记录

完成日期：2026-07-04

性质：v3.1 最小兼容迁移。本文记录本轮真实代码改动；默认剧本仍是阿登，主 UI 尚未完成隋唐发布级迁移。

## 1. 目标和非目标

目标：

- 在不删除旧 `germany/allies` 路径的前提下，为隋唐多势力建立可 Codable 的 `Faction` 入口。
- 新增通用 `GamePhase.playerCommand` / `GamePhase.aiCommand`，保留 `germanAI` / `alliedPlayer` legacy 阶段。
- 将核心敌对判断从二元 `.opponent` 迁移到 `DiplomacyState` 关系接口。
- 让 AI fallback、Ruler / Marshal 配置、MapEditor 势力显示、SpriteKit 势力颜色能覆盖新增势力。
- 保持玩家和 AI 行动仍经 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。

非目标：

- 不切换默认剧本到 `wude_618`。
- 不修改阿登 JSON、隋唐 JSON 或 Xcode project。
- 不重写全部补给、战斗、经济、胜利条件。
- 不删除 legacy Agent D、`runGermanAITurn` 兼容入口或旧 `Faction.opponent` 属性。
- 不运行本机 Xcode build / XCTest / 模拟器。

## 2. 核心改动

### 2.1 Faction 多势力兼容

文件：`WWIIHexV0/Core/Faction.swift`

新增势力：

- `tang`
- `luoyangSui`
- `wagang`
- `xia`
- `qinXue`
- `liuWuzhou`
- `tujue`

新增辅助：

- `legacyCombatants`
- `suitangTurnOrder`
- `usesDefaultHumanControl`

保留：

- `germany`
- `allies`
- `opponent` legacy convenience

说明：`.opponent` 不再被生产主路径直接调用；仅保留兼容旧代码和旧测试。

### 2.2 GamePhase 通用阶段

文件：`WWIIHexV0/Core/GamePhase.swift`

新增：

- `playerCommand`
- `aiCommand`
- `allowsPlayerInput`
- `allowsAIExecution`

保留 legacy：

- `germanAI`
- `alliedPlayer`

`CommandValidator.phaseAllowsCommands` 对 legacy 阶段仍保持旧行为；对 `playerCommand` / `aiCommand` 允许当前 active faction 的命令进入规则系统。

### 2.3 DiplomacyState 关系接口

文件：`WWIIHexV0/Core/DiplomacyState.swift`

新增关系状态：

- `truce`
- `vassal`
- `submitted`

新增接口：

- `relationStatus(between:and:)`
- `isHostile(_:_:)`
- `canAttack(_:_:)`
- `isFriendly(_:_:)`

`initial(for:)` 现在可为旧二战势力和隋唐势力生成初始 country / bloc。旧 JSON 没有 factions 时 fallback 到 `Faction.legacyCombatants`，避免无意创建所有隋唐势力。

### 2.4 规则层解除 `.opponent` 主路径依赖

受影响文件：

- `WWIIHexV0/Rules/CommandValidator.swift`
- `WWIIHexV0/Rules/MovementRules.swift`
- `WWIIHexV0/Rules/SupplyRules.swift`
- `WWIIHexV0/Rules/RegionSupplyRules.swift`
- `WWIIHexV0/Rules/RegionCombatRules.swift`
- `WWIIHexV0/Rules/FrontLineManager.swift`
- `WWIIHexV0/Rules/WarDeploymentManager.swift`
- `WWIIHexV0/Commands/WarCommandExecutor.swift`
- `WWIIHexV0/Agents/AgentContexts.swift`
- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`

结果：

- 攻击校验使用 `DiplomacyState.canAttack`。
- ZOC、补给通行、安全撤退、region 压力、旧 Agent D 摘要、元帅目标损失统计使用 `DiplomacyState.isHostile`。
- `WarCommandExecutor` 的敌控 hex 排序使用 hostile controller 判断。
- `FrontLineManager` 的 `factionB` 从真实 segment 敌方控制方推导，不再二元翻转。

轻量扫描结果：

```sh
rg -n "Faction\\.opponent|\\.opponent" WWIIHexV0/Core WWIIHexV0/Rules WWIIHexV0/Commands WWIIHexV0/Agents WWIIHexV0/Turn
```

无命中。

### 2.5 回合推进兼容

文件：`WWIIHexV0/Rules/CommandExecutor.swift`

行为：

- legacy `.germanAI`：仍推进到 `.allies / .alliedPlayer`。
- legacy `.alliedPlayer`：仍推进到 `.germany / .germanAI`，并增加 turn。
- generic `.playerCommand` / `.aiCommand`：按 `Faction.suitangTurnOrder`、legacy combatants 和当前状态实际势力集合推导下一 active faction。
- generic phase 下，`tang` 和 `allies` 默认进入 `playerCommand`，其他势力默认进入 `aiCommand`。

说明：玩家可选择势力尚未数据化；后续 v3.5 需要把 player faction 选择写入状态或场景配置。

## 5. 风险和后续

风险：

- 本轮只做轻量 parse，未做 Xcode build；跨 target membership、scheme、资源加载和真实运行时行为未验证。
- `GamePhase.playerCommand/aiCommand` 已存在，但 player faction 选择尚未写入 `GameState`，当前默认把 `tang/allies` 视作人类控制入口。
- `VictoryRules` 仍是阿登专用，隋唐胜负条件需 v3.2/v3.3 另行实现。
- `Faction.allCases` 现在包含隋唐势力，MapEditor 导出 factions 会包含所有 case；v3.2 需要让场景数据明确声明启用势力。
- 测试文件仍大量使用 `germany/allies` 和二战语义，本轮未修改测试。

下一步建议：

- v3.2 新增 `wude_618` 剧本 JSON、region JSON、unit templates、general data 和 power profiles。
- `DataLoader` 增加默认剧本切换策略，但在真实数据齐备前不要把 README/AGENTS 改成完成态。
- MapEditor 需要在 v3.2 明确“启用势力集合”和隋唐地形/粮仓/关隘字段。

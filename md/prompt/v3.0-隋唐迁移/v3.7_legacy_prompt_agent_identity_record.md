# v3.7-preflight.65 legacy prompt 决策者身份净化

完成日期：2026-07-06

## 目标

在 legacy prompt 直通文本净化之后，继续处理旧 `AgentPromptBuilder.systemPrompt` 中 `context.agentId` 和 `context.personality` 作为自由文本原样进入 LocalLLM prompt 的问题，避免 raw agent id、英文 personality 或工程词在系统提示层诱导模型回写到中文摘要字段。

## 实现范围

- `AgentPromptBuilder.systemPrompt` 中“决策者”改为展示名：
  - `guderian` 显示为“古德里安”。
  - 其他纯内部 id 形式的 agent id 显示为“本地军议决策者”。
  - 非内部 id 文本原样保留。
- `AgentPromptBuilder.systemPrompt` 中“性格”改走 `sanitizePromptText(_:)`，复用 `.64` 的 prompt-local 自由文本净化。
- `GameAgent.sample` 的默认 personality prompt 从英文改为中文“遵守角色职责，保持建议结构清晰。”，traits 从 `disciplined` 改为“守纪”。
- JSON schema 中的 `"agentId": "\(context.agentId)"` 保持不变，继续满足 `LocalLLMDecisionProvider` 与 `AgentDecisionParser` 的 expected agent id 合同。

## 边界

- 不改 `AgentContext`、`AgentDecisionEnvelope`、`AgentDecisionParser`、`LocalLLMDecisionProvider` 或 `AgentCommandMapper`。
- 不改 JSON key、schema version、agent id rawValue、division id、region id、target id 或命令 type rawValue。
- 不改默认战争 AI 主路径；旧 `AgentPromptBuilder` 仍是 legacy LocalLLM / Agent D 回归参考路径。
- 不改 AI 决策策略、命令结构、规则执行、存档字段或运行时状态。

## 遗留风险

- JSON schema 仍必须暴露 raw `agentId`，否则 parser 的 agent mismatch 校验会失效；本轮只净化自由文本身份描述。
- `sanitizePromptText(_:)` 是 `AgentPromptBuilder` 局部 helper，尚未抽成共享 formatter；这是刻意控制范围，避免牵动 UI 展示层。
- 本轮没有启用真实 LocalLLM 或 legacy Agent D 管线验证输出质量，只做 prompt 文本入口收口。

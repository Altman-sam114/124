# v3.7-preflight.16 归附实体交接命令

完成日期：2026-07-05

## 目标

在 v3.7-preflight.15 已能盘点归附目标残余实体的基础上，补上最小可执行交接命令：归附接收方可在自己行动阶段，通过 `Command -> CommandValidator -> RuleEngine -> CommandExecutor` 接管已归附目标的未毁灭军队和可通行受控 hex。

## 已完成

- `Command` 新增 `resolveSubmissionHandoff(submitted:recipient:)`，作为归附实体交接的唯一执行入口。
- `CommandValidator` 新增交接校验：必须在允许命令的阶段执行，`recipient` 必须是当前行动势力，双方必须有 country profile，且 `DiplomacyState` 证明 `submitted` 当前归附于 `recipient`。
- `DiplomacyState` 的归附判断改为按每个 target 的最新外交事件判断；只有完全没有外交事件记录的旧存档才 fallback 到 `.submitted` 关系状态。
- `CommandExecutor` 执行交接时，将归附目标的未毁灭军队改属接收方并标记本回合已行动，将归附目标受控的可通行 hex 改属接收方，然后刷新 region / theater / front / deploy 派生层。
- `AppContainer` 新增玩家侧交接能力判断和提交方法；`DiplomacyPanelView` 在“归附实体”区提供“接管”按钮。
- `WarCommandExecutor` 的辅助 switch 识别该命令为无 acting division 的系统命令。
- `ReleaseChecklistView`、README、flow、flowchart、plan、update_log 同步到 v3.7-preflight.16。

## 边界

- 本轮只处理最小实体交接：未毁灭军队和可通行受控 hex。
- 不移除 country profile，不删除外交关系，不清理旧事件记录，不处理贡赋、俘虏、忠诚、叛乱、善后安置或多轮谈判。
- 不直接修改 `regionToTheater`；动态 theater/front/deploy 仍由同步器和 bootstrapper 从 hex / division 状态刷新。
- UI 只提交 `Command.resolveSubmissionHandoff`，不直接改 `GameState`。

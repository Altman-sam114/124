# v3.2 隋唐地图、剧本数据和默认入口实现记录

完成日期：2026-07-04

性质：默认数据与加载入口迁移。本文记录本轮真实改动；不表示兵种、围城、胜利规则和发布级 UI 已完成。

## 1. 目标和非目标

目标：

- 新增首张隋末唐初默认剧本 `wude_618_guanzhong_luoyang`。
- 新增隋唐 region、unit template、general registry、power profile 和 terrain rule 数据。
- 让主游戏默认新局优先加载 `wude_618`，失败时保留阿登 fallback。
- 保持 MapEditor 导出的 scenario/region 临时目录仍能只靠旧 fallback template 加载。
- 把新 JSON 加入 iOS / macOS 主游戏 resource phase，避免非 DEBUG 只回退阿登。

非目标：

- 不改 MapEditor 默认桥接文件名；编辑器默认仍指向阿登资源。
- 不引入真正的 `neutral` faction；当前 schema 下所有 region 必须有可表示的 controller。
- 不实现完整古代兵种 enum、围城、天命、民心或外交事件。
- 不修改测试文件，不运行本机 Xcode build / XCTest / 模拟器。

## 2. 数据交付

新增数据：

- `WWIIHexV0/Data/wude_618_scenario.json`
- `WWIIHexV0/Data/wude_618_regions.json`
- `WWIIHexV0/Data/suitang_unit_templates.json`
- `WWIIHexV0/Data/suitang_generals.json`
- `WWIIHexV0/Data/suitang_power_profiles.json`
- `WWIIHexV0/Data/suitang_terrain_rules.json`

剧本摘要：

- scenario id：`wude_618_guanzhong_luoyang`
- displayName：`武德元年：关中河洛争衡`
- 地图：10x9，共 90 hex。
- 州郡：36 region，覆盖关中、河东、河洛、河北南缘、淮北边缘、陇右和北线压力区。
- 势力：唐、洛阳隋、瓦岗、夏、薛秦、刘武周、东突厥。
- 初始单位：19 支军队。
- 关键节点：长安、晋阳、洛阳、虎牢、洛口仓、洺州、潼关、黎阳仓。

当前兼容策略：

- 粮仓 / 关隘用现有 `isSupplySource`、`supplyValue`、`city`、`fortress` 表达。
- 隋唐 unit template 仍映射到 legacy `ComponentType`：骑军映射 `motorizedInfantry`，器械映射 `artillery`，步卒/守军映射 `infantry`。
- 南缘地方区暂归瓦岗，避免 `RegionDataSet.toRegions()` 对 null owner/controller fallback 到 `.allies`。

## 3. 加载入口

`DataLoader.loadInitialGameState()` 现在优先尝试：

```text
scenarioName: wude_618_scenario
regionName: wude_618_regions
unitTemplatesName: suitang_unit_templates
generalRegistryName: suitang_generals
```

若任一关键加载失败，则 fallback 到 legacy 阿登路径：

```text
scenarioName: ardennes_v0_scenario
regionName: ardennes_v02_regions
```

`loadGameState(...)` 新增可选资源名参数，但 unit templates / generals 缺失时仍 fallback 到旧的 component / empty registry 行为，保证 MapEditor 临时导出目录不必附带全套资源。

`AppContainer.bootstrap()` 现在根据 `gameState.scenarioId` 选择 general registry；`wude_618` 使用 `suitang_generals`。默认玩家势力跟随可人控 active faction，`wude_618` 默认为唐。

## 6. 风险和后续

风险：

- 本轮未运行 Xcode build，resource membership 仅通过 pbxproj 结构检查，未做真实 bundle 加载验证。
- 旧测试仍大量假设默认启动为阿登；v3.2 默认入口切换后测试口径需要后续迁移。
- `VictoryRules` 仍是阿登语义，`wude_618` 的胜负条件只在 JSON 中记录，运行时未完整消费。
- `ComponentType` 仍是二战兼容枚举，隋唐兵种只是模板显示与权重映射。
- `RegionNode` 仍不能表达无 controller 的战略 region，真正地方中立需要后续 schema / Faction 策略。

下一步建议：

- v3.3 迁移兵种、粮道、围城和战术显示。
- v3.3 或 v3.5 增加隋唐胜利规则消费路径。
- v3.2 后续子轮可再迁移 MapEditor 默认桥到 `wude_618`，但需要同步更新旧测试语义。

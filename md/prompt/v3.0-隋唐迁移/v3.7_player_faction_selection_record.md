# v3.7-preflight.40 本局执掌势力选择记录

完成日期：2026-07-05

## 目标

补齐总提示词中“玩家可选择至少一个势力，首发优先保证唐可玩”的最小产品闭环：新局默认唐可玩，玩家可在基础设置中切换本局执掌势力，并随 `GameState` 本地存档保存。该切片只处理本局玩家势力判定，不做完整多势力平衡、不新增剧本选择页、不重排当前回合顺序。

## 实现摘要

- `GameState.playerFaction` 成为本局执掌势力字段，并进入 Codable；旧存档缺字段或字段 rawValue 异常时按 scenario 和 active faction 兼容默认。
- `DataLoader` 从 scenario `playerFaction` 初始化本局执掌势力，`wude_618` 冷启动默认唐。
- `CommandExecutor` 的通用回合推进根据 `state.playerFaction` 选择 `.playerCommand` 或 `.aiCommand`。
- `AppContainer` 从 `gameState.playerFaction` 派生玩家势力；切换时清空选择、保存本局，并按当前 active faction 调整 phase。
- `GameSettingsView` 增加“执掌势力” picker；`HUDView` 和 `ReleaseChecklistView` 展示执掌势力。

## 边界

- 玩家行动仍必须满足 active faction 等于 `playerFaction` 且 phase 允许玩家输入。
- 切换执掌势力不会把当前回合跳到该势力；若当前行动势力不是玩家势力，会让自动回合继续推进，设置页切换可能因此保存并触发自动回合。
- 新增字段是存档 schema 扩展，旧存档会兼容默认，继续存档会把执掌势力规范到当前局势可选范围；旧版本 app 读取新存档未验证。
- 不绕过 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。

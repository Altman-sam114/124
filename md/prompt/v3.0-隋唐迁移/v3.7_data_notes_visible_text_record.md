# v3.7-preflight.42 默认数据说明中文化记录

完成日期：2026-07-06

## 目标

在 MapEditor 可见文案收口之后，继续处理默认隋唐数据、数据加载入口和补给规则战报中仍可能进入战报、导出 JSON 或人工验收视野的英文工程说明，减少 `Sui-Tang scenario bridge`、`MapEditor-compatible JSON`、`legacy combat bridge`、`strength`、`retreated from` 等残留口径。

## 实现摘要

- `DataLoader` 的 MapEditor 兼容数据加载初始战报改为中文：“已载入地图编辑器战局数据：...”，不再显示 `Loaded ... from MapEditor-compatible JSON`。
- `MapEditorExportMetadata.suitangDraft.dataNotes` 改为中文说明，保留“地块是战术权威、州郡和方面是战略聚合/初始归属”的架构边界。
- `MapEditorExportMetadata.legacyArdennes.dataNotes` 改为中文旧战局说明，避免 legacy 导出 metadata 继续写入英文说明。
- MapEditor 从补给地块派生 `keyLocations` 时，默认地点名从 `Supply q,r` 改为 `粮仓 q,r`。
- `wude_618_scenario.json` 的 `dataNotes` 改为中文，说明当前隋唐数据已从 v3.2 后持续补上胜负、存档、外交、州郡经营、归附和展示层收口。
- `SupplyRules` 写入战报的补员、撤退、撤退失败、围困损耗和退却整顿事件改为中文。
- 保留 JSON key、rawValue、scenario id、objective id、unit id、胜负条件、地图数据和加载顺序不变。

## 边界

- 本轮只改数据说明文案、导出说明、初始化战报和补给/撤退相关事件文案。
- 不改 `ScenarioDefinition` schema，不改 `DataLoader` 加载顺序，不改 `GameState` 字段。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`，不改变胜负、补给、移动、战斗、外交或归附规则。
- legacy 阿登资源和旧数据校验路径继续保留作兼容与回归参考。

# v3.7-preflight.69 UI/战报/外交记录净化 helper 对齐

完成日期：2026-07-06

## 目标

在模拟元帅输出纯 JSON 收口之后，继续处理 AI 面板、战报面板和外交面板中的记录 / 诊断展示净化。目标是让三处 helper 都先清理 raw id，再做工程词和模型词替换，避免 `agent_*`、`war_directive_*`、`player_directive_*` 等审计 id 被普通词替换提前破坏。

## 实现范围

- `EventLogView.displayDiagnosticText` 改为先执行 `sanitizeRawIdentifiers(in:)`，再做展示词表替换。
- `DiplomacyPanelView.displayRecordText` 改为先执行 `sanitizeRawIdentifiers(in:)`，再做展示词表替换。
- `AgentPanelView.displayDiagnosticText` 保持先清 raw id 的顺序，并补齐缺失词表。
- 三处 helper 补齐 `rawJson`、`raw JSON`、`Provider`、`Schema`、`AI`、`LLM`、`OpenAI`、`GPT`、`Claude`、`Gemini`、`record`、`Legacy Pipeline` 和裸 `pipeline` 等展示替换。
- 三处 helper 统一 `RuleEngine`、`local-model`、`Model`、`model` 的展示口径。
- `DiplomacyPanelView.sanitizeRawIdentifiers` 补齐常见军令、地块、军队和命令 raw id 前缀。

## 边界

- 不改 UI 布局、面板结构、排序、视觉样式或交互。
- 不抽新的公共 sanitizer API，避免扩大改动范围和跨文件耦合。
- 不改 `AgentDecisionRecord`、`WarDirectiveRecord`、`DiplomacyEventRecord`、`CourtDecisionRecord` 或善后记录 schema。
- 不改日志来源、record id、存档字段、AI 决策、命令结构、规则执行或运行时状态。
- 不把真实 `rawTheaterJSON`、record id 或 schema 字段改名；本轮只处理玩家可见展示层净化。

## 并发扫描结论

- `AgentPanelView` 原本已经采用先 raw id sanitizer、后语义替换的较安全顺序。
- `EventLogView` 与 `DiplomacyPanelView` 原本先替换 `agent` / `directive` 等词，再清理 raw id，存在把审计 id 提前破坏的风险。
- 三处词表对 `AI`、`LLM`、模型品牌词、`rawJson`、`raw JSON`、`Provider`、`Schema`、`record`、`Legacy Pipeline` 和裸 `pipeline` 覆盖不一致。
- 最小化方案是只改三个私有 helper，不改 UI 布局，不抽公共 API。

## 遗留风险

- 本轮未运行 app、模拟器或 UI 点击烟测，无法确认所有真实记录路径的最终视觉效果。
- 本轮只处理 AI 面板、战报面板和外交面板三处 helper，不代表全项目玩家可见文本已经穷尽。
- 词表替换仍是展示层兜底；长期若净化逻辑继续扩张，可以考虑在更完整设计后抽共享 helper。

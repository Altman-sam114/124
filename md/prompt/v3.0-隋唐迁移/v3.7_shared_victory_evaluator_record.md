# v3.7-preflight.90 共享隋唐胜负 evaluator 收口记录

完成日期：2026-07-07

## 目标

在 `RegionVictoryRules` 隋唐胜负摘要对齐之后，继续消除主胜负规则和 region 分析摘要之间重复维护 `wude_618` 胜负判断的风险。本轮抽出共享只读 evaluator，让 `VictoryRules` 和 `RegionVictoryRules` 复用同一份隋唐胜负评估逻辑。

## 改动范围

- `WWIIHexV0/Core/VictoryState.swift`
  - 新增 `VictoryAssessment`，表达只读胜负评估结果。
  - `VictoryState.apply(_:)` 负责把评估结果写回主胜负状态。
- `WWIIHexV0/Rules/VictoryRules.swift`
  - 新增规则层 `Wude618VictoryEvaluator`，集中读取 `obj_luoyang`、`obj_luokou`、`obj_tongguan`、`obj_changan`，并统一处理终局长安结算。
  - `wude_618` 分支改为调用 `Wude618VictoryEvaluator.assess(in:)`，再写回 `GameState.victoryState`。
  - legacy fallback 胜负逻辑保持原路径。
- `WWIIHexV0/Rules/RegionVictoryRules.swift`
  - `RegionVictoryAssessment` 改为 `VictoryAssessment` 的 typealias。
  - `wude_618` 分支直接复用 `Wude618VictoryEvaluator.assess(in:)`。
  - legacy fallback 摘要保持独立兼容路径。

## 明确不改

- 不新增文件，不改 `WWIIHexV0.xcodeproj/project.pbxproj`。
- 不把 `VictoryConditionDefinition` 扩为通用解释器。
- 不改 objective id、地图数据、胜负阈值、回合推进、命令管线、AI 决策或存档格式。
- 不移除 legacy fallback 胜负规则、legacy `VictoryReason` case 或 `VictoryState` 兼容字段。

# v3.7-preflight.12 AI 使者主动外交记录

完成日期：2026-07-05

## 1. 目标

在 v3.7-preflight.9 已有 `Command.updateDiplomacy`、v3.7-preflight.11 已有 AI 太守主动经营的基础上，补上 AI/观战自动回合的最小使者外交路径：

- AI 使者能在保守条件下生成停战或归附关系命令。
- 外交命令必须继续走 `Command -> CommandValidator -> RuleEngine -> CommandExecutor`。
- 不直接改 `GameState`、hex controller、region controller、军队位置、动态方面、前线或行军防区。

## 2. 实现范围

### 2.1 AI 回合接入点

`TurnManager.executeDirectiveEnvelope` 在完成以下事项后执行使者外交：

1. `WarCommandExecutor` 执行战争 directive。
2. 写入 `RulerDecisionRecord` / `CourtDecisionRecord`。
3. 尝试 AI 太守州郡经营。

随后调用 `executeDiplomatCommands(for:in:courtRecord:)`，并在 `.endTurn` 前完成最多一条 `Command.updateDiplomacy`。

### 2.2 外交选择

候选目标来自当前地图、军队和 `DiplomacyState.countries` 中出现的 faction，并必须满足：

- 不是当前行动势力。
- 与当前行动势力处于 hostile / atWar。
- 目标 faction 在 `DiplomacyState` 中有 country profile。

保守执行条件：

- 敌方已经没有存活军队，且没有可通行受控州郡时，生成 `submitted`。
- 己方 primary country `warSupport <= 45`，或己方存活军队不超过 1，或己方可通行受控州郡不超过 1 且同时面对多个敌对势力时，向压力最高的敌对势力生成 `truce`。

压力评分按目标可通行受控州郡、存活军队和 primary warSupport 粗略计算，只用于选择停战目标。

### 2.3 结果记录

AI 使者命令结果复用 `CommandResultSummary.systemCommand(...)`，进入 `AgentDecisionRecord.commandResults`。命令被拒绝时，`TurnManager` 会把拒绝原因追加到 diagnostics。

## 3. 边界

- 不新增完整外交谈判状态机。
- 不改变 `Command.updateDiplomacy` 的校验和执行语义。
- 不直接转移 hex、region、unit、theater、front 或 deploy。
- 不处理借兵、同盟、贡赋、长期信任、称臣方向性或多轮谈判。
- `submitted` 只更新 `DiplomacyState` 关系，不代表地图/军队自动归属交接。

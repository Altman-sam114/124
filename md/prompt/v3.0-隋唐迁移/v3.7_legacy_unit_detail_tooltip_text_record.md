## 实现摘要

- `UnitInspectorView` 单位标题改走 `displayDivisionName(_:)`。
- `UnitTooltipView` 可见单位标题和 `accessibilityLabel` 同步使用展示净化后的单位名，避免辅助功能播报继续直出 legacy 名称。
- 两个文件各自保留局部 helper，展示前清理 `division_*` / `unit_*` raw id，并把“德军 / 盟军 / 装甲 / 摩托化 / 炮兵 / 步兵 / 师”等旧 fallback 单位词转成发布前置口径。
- 兵种行仍使用 `ComponentType.displayName` / `unitKindDisplayName`，保留现有兵种显示权威。

## 并发扫描结论

- 单位面板子 Agent 确认 `UnitInspectorView` 和 `UnitTooltipView` 原先直接显示 `division.name`，tooltip 的 `accessibilityLabel` 也拼接 `division.name`；当前默认隋唐数据安全，但 legacy fallback 仍可能带入“德军装甲第1师 / 盟军步兵第1师”等旧题材名。
- 文档子 Agent 建议本轮使用 `v3.7-preflight.72`，记录文件名为 `v3.7_legacy_unit_detail_tooltip_text_record.md`，并同步 `README.md`、`update_log.md`、`md/flow/*` 与 `md/plan/plan.md`。

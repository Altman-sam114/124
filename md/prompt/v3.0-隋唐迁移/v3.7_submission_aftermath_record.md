# v3.7-preflight.19 归附善后压力记录

## 目标

在 v3.7-preflight.16-.18 已有归附实体交接命令、交接审计记录和 AI 侧交接的基础上，补上最小“善后压力”只读记录。交接成功后，系统根据转移军队、受控 hex 和影响州郡数量生成 `SubmissionAftermathRecord`，用于外交面板、日志和后续验收判断是否需要安民、整军或道路粮仓治理。

## 范围

- `DiplomacyState` 新增 `SubmissionAftermathRiskLevel`、`SubmissionAftermathRecord` 和 `submissionAftermathRecords`。
- 旧存档缺少 `submissionAftermathRecords` 时默认空数组，最多保留最近 80 条。
- `CommandExecutor.executeSubmissionHandoff` 在交接成功后追加善后压力记录，并写入一条 `.diplomacy` 日志，`relatedRecordId` 指向该记录。
- `DiplomacyPanelView` 新增“善后”区，展示最近一次善后压力、摘要、州郡数、回合和边界说明。
- `ReleaseChecklistView` 版本推进为 `天命开唐 Agent · v3.7-preflight.19`，把善后压力只读记录列为已接入。

## 非目标

- 不实现忠诚、叛乱、贡赋、俘虏、安置、封赏或长期善后系统。
- 不改变 `HexTile.controller`、`Division.coord`、`hexToTheater`、`hexToFrontZone` 的权威边界。
- 不让 UI 或记录层直接修改 `GameState`。
- 不新增玩家或 AI 的治理自动执行逻辑；后续安民、整军或粮仓治理仍必须单独经 `Command` / `RuleEngine`。

## 数据流

```text
Command.resolveSubmissionHandoff
  -> CommandValidator 校验归附关系和残余实体
  -> CommandExecutor 转移未毁灭军队和可通行受控 hex
  -> StrategicStateSynchronizer / StrategicStateBootstrapper 刷新派生层
  -> appendSubmissionHandoffRecord
  -> appendSubmissionAftermathRecord
  -> EventLog(.diplomacy, relatedRecordId: aftermath.id)
  -> DiplomacyPanelView 只读展示
```

## 风险计算

`SubmissionAftermathRiskLevel` 是保守的只读提示：

- 转移军队越多，压力越高。
- 转移可通行受控 hex 越多，压力越高。
- 影响州郡越多，压力越高。

风险等级只影响摘要和 UI 展示，不触发资源扣减、叛乱、忠诚变化、额外事件或归属转移。

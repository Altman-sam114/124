# v3.7-preflight.57 legacy LLM prompt 语言收口记录

完成日期：2026-07-06

## 目标

继续推进隋末唐初迁移发布候选前置收口，处理旧 LocalLLM / legacy Agent 管线中仍会诱导英文 `intent`、`reason`、`summary` 回流到军情面板的 prompt 文案，以及数据加载失败时的古德里安 fallback 英文展示名。

本轮只改 prompt 自然语言、示例值和 fallback 展示文案，不改 JSON schema、command type、agent id、rawValue、解析合同、AI 决策结构、规则或命令管线。

## 改动范围

- `WWIIHexV0/Agents/AgentPromptBuilder.swift`
  - system prompt 改为中文，并明确 `intent`、`reason`、`stance` 摘要必须中文。
  - user prompt 的任务说明、可用命令、战场摘要、补给、近期战报和玩家意图说明中文化。
  - 目标、己方军队、敌军和可见州郡摘要使用中文字段标签。
  - JSON 示例中的 `intent` / `reason` / `stance` 示例值改为中文。
- `WWIIHexV0/Agents/AgentConfiguration.swift`
  - `guderianFallback` 的展示名改为“古德里安”。
  - fallback personality prompt 改为中文。

## 边界

- 保留 JSON key、`responseFormat`、`schemaVersion`、`type` 的 `move|attack|hold|resupply` 合同值。
- 保留 `traits: ["breakthrough"]` 等内部 raw 值。
- 不改 `LocalLLMDecisionProvider`、解析器、命令映射、RuleEngine 或任何运行时规则。
- 不运行 Xcode build、XCTest、模拟器或 app。

按 `md/test/test.md`，本轮不跑 Xcode build/test、XCTest、模拟器、UI test、性能测试或全量构建。

## 遗留风险

- 未真实启用 LocalLLM 管线验证模型输出是否完全中文。
- `AgentPromptBuilder` 仍保留 command type 和 region id 等结构化合同字段，后续展示层仍需继续净化英文工程词。
- DiplomacyPanel、AgentPanel 和 EventLog 的诊断 sanitizer 仍有进一步合并与兜底空间。

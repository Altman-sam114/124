# v3.7-preflight.20 AI 善后太守优先治理

## 目标

在 v3.7-preflight.19 已有 `SubmissionAftermathRecord` 善后压力只读记录的基础上，让 AI 太守在后续 AI 回合优先治理高/需安抚善后压力州郡。该增量把“善后压力提示”接入既有 AI 州郡经营选择，不新增命令类型，也不绕过 `RuleEngine`。

## 范围

- `TurnManager.governorCommand` 读取最新 `SubmissionAftermathRecord`。
- 仅当当前 AI 势力是记录中的 `recipient`，且 `riskLevel` 不是 `.low` 时，善后州郡才进入优先候选。
- 善后州郡必须仍由当前势力控制、可通行，并且存在当前势力实际受控 hex；最终执行仍交给 `CommandValidator` 复核。
- 善后州郡排在普通 `CourtDecisionRecord` 太守关注点之前。
- 善后州郡优先尝试 `RegionGovernancePolicy.pacifyPopulation`；若安民不可用或府库不足，则回到既有屯田/修道/安民择优。
- AI 太守仍每个 AI 回合最多提交一条 `Command.governRegion`。
- `ReleaseChecklistView` 版本推进为 `天命开唐 Agent · v3.7-preflight.20`。

## 非目标

- 不新增忠诚、叛乱、贡赋、俘虏、安置、封赏或长期善后状态机。
- 不在归附交接同一时刻额外追加第二条治理命令。
- 不批量处理历史善后记录；当前只读取最新记录。
- 不直接修改 `RegionNode`、`HexTile.controller`、`Division.coord`、`hexToTheater`、`hexToFrontZone` 或府库。
- 不让记录层或 UI 直接修改 `GameState`。

## 数据流

```text
SubmissionAftermathRecord(latest)
  -> TurnManager.aftermathGovernorRegionIds
  -> governorCommand focusedRegionIds
  -> governorPolicy(prefersPacification: true)
  -> Command.governRegion
  -> CommandValidator
  -> RuleEngine / CommandExecutor
  -> RegionNode + EconomyState 府库
  -> AgentDecisionRecord.commandResults
```

## 边界说明

该版本只让 AI 更早选择善后州郡，并优先尝试安民。实际能否执行仍取决于当前行动阶段、州郡控制权、实际受控 hex、经营上限和府库资源。命令失败时只进入 AI diagnostics，不补发第二条治理命令。

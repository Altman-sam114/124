# v3.7-preflight.88 legacy objective lookup 字面量收口记录

完成日期：2026-07-07

## 目标

在静态 `GameState` / `MapState` fallback 可见文本收口之后，继续处理规则和旧 MockAI 回归路径中残留的旧战役 objective 名称字面量。本轮只收口源码中的兼容查找文本和局部命名，不改变 objective id、胜负阈值、AI 行动策略、命令管线、规则执行或存档字段。

## 改动范围

- `WWIIHexV0/Rules/VictoryRules.swift`
  - `updateArdennesVictoryState` 改为 `updateLegacyFallbackVictoryState`。
  - legacy fallback 胜负查找改为 objective id 优先，并只 fallback 到中性 `旧战局要地甲 / 旧战局要地乙` 展示名。
  - 局部变量从旧地名语义改为 `primaryObjectiveController`、`secondaryObjectiveController`、`legacyMobileForces`。
- `WWIIHexV0/Rules/RegionVictoryRules.swift`
  - region 胜负评估同样按 legacy objective id 优先，再按中性旧战局城邑名 fallback。
- `WWIIHexV0/Agents/MockAIClient.swift`
  - 旧 MockAI 主目标查找按 legacy objective id 优先，再按中性旧战局目标名 fallback。

## 明确不改

- 不改 `bastogne`、`st_vith` objective id。
- 不改 `.germany` / `.allies` enum rawValue、`VictoryReason` case、`VictoryState` 存档字段名。
- 不改 legacy 胜负阈值、AI 排序、移动/攻击/整补策略。
- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` 管线。
- 不改 JSON schema、加载顺序、地图坐标或存档格式。

## 轻量检查

- `swiftc -parse WWIIHexV0/Rules/VictoryRules.swift`
- `swiftc -parse WWIIHexV0/Rules/RegionVictoryRules.swift`
- `swiftc -parse WWIIHexV0/Agents/MockAIClient.swift`
- 聚焦扫描三处源码中的旧战役可见地名字面量。
- 本轮改动文件尾随空白、冲突标记和 `git diff --check`。

## 遗留风险

- 本轮未启动游戏、MapEditor 或模拟器，未验证旧 fallback 胜负和 MockAI 路径的运行时表现。
- `VictoryState` 中仍保留 `germanBastogneHeldSinceTurn` 等兼容字段名和 `VictoryReason` legacy case；这些属于存档 / enum 兼容合同，不是本轮可见文本。

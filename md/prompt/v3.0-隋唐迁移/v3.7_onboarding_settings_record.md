# v3.7-preflight.3 开局引导、基础设置和发布前检查面板记录

完成日期：2026-07-05

## 1. 本轮定位

本轮继续补完整 v3.7 发布候选前置缺口，但仍不是完整发布候选。

目标是把总提示词中“新手引导、设置、版本说明和发布前检查清单”的最小 UI 入口接入主游戏。该入口只读取现有状态，或切换已有的地图图层/观战状态，不新增规则、不直接修改 `GameState` 战术权威，也不替代后续授权运行时重测。

## 2. 实现摘要

### 2.1 HUD 筹备菜单

`HUDView` 新增可选发布候选前置入口：

- 开局引导。
- 基础设置。
- 发布前检查。

`RootGameView` 通过 sheet 展示对应面板。菜单触发只改变 SwiftUI 本地展示状态；不会直接执行命令或修改地图、单位、战区、前线、部署。

### 2.2 开局引导

新增 `FirstTurnGuideView`：

- 显示当前剧本、回合和行动势力。
- 给出第一回合顺序：看大势、选军队、查州郡、交给总管、结束回合。
- 明确所有行动仍经军令校验和规则引擎。
- 提醒完整发布前仍需授权运行时烟测、观察者多回合和 Xcode 构建验证。

### 2.3 基础设置

新增 `GameSettingsView`：

- 可切换观战模式。
- 可选择地图图层。
- 显示本地存档是否可继续。
- 明确当前存档范围仅为战局状态，地图图层和观战模式不写入存档。

该设置页复用 `RootGameView -> AppContainer.setMapDisplayLayer` 和 `AppContainer.setObserverModeEnabled`，不引入新持久化 schema。

### 2.4 发布前检查和版本说明

新增 `ReleaseChecklistView`：

- 显示工作产品名 `天命开唐 Agent` 和 `v3.7-preflight.3` 迁移阶段。
- 展示当前剧本、当前势力、本地存档和发布状态。
- 列出已接入事项：默认隋唐剧本、新局/继续/重置和自动存档、核心胜负条件、玩家/外交/战报/朝堂中文信息闭环。
- v3.7-preflight.3 原始清单列出待确认事项：Xcode build、iOS/macOS 启动、UI 点击烟测、观察者多回合、外交归附、州郡经营命令、正式地图资产、渡口/港口图标、AI 计划箭头。本清单已在 v3.7-preflight.6 更新为最小地图显示已接入、运行时截图仍待确认。
- 明确未授权前不跑本机重测试，也不声称已可发布。

## 4. 架构边界

- UI 面板不直接修改 `HexTile.controller`、`Division.coord`、`regionToTheater`、`hexToTheater` 或 `hexToFrontZone`。
- 开局引导和发布前检查只读 `GameState`。
- 基础设置只调用既有地图图层和观战 setter；不写入本地存档。
- 不新增 `Command`、`ZoneDirective`、validator、executor 或规则分支。
- 不引入外部素材、第三方依赖、真实 LLM、网络调用或 API key。
- 不更新 `AGENTS.md` 的项目身份；当前仍保留真实工程历史口径。

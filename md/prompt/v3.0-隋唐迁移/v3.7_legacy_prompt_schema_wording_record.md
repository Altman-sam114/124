# v3.7-preflight.67 legacy prompt 工程说明词收口

完成日期：2026-07-06

## 目标

在 legacy MockAI stance 文案收口之后，继续处理旧 `AgentPromptBuilder` 中面向 LocalLLM 的 prompt 说明词。目标是把 `hex`、`schema`、`Markdown` 和 `JSON schema` 等工程化说明收口为中文结构化军令口径，减少 legacy Agent D / LocalLLM 回归路径把工程词写入 `intent`、`reason`、`stance` 的诱导。

## 实现范围

- `AgentPromptBuilder.systemPrompt` 中“回合制 hex 策略战局”改为“回合制六角格策略战局”。
- `AgentPromptBuilder.systemPrompt` 中“符合 schema 的有效 JSON / Markdown”说明改为“符合下方结构化格式的 JSON 内容 / 排版标记”。
- `AgentPromptBuilder.systemPrompt` 中“JSON key 或英文 type 值”改为“字段名或命令类型原值”。
- `AgentPromptBuilder.userPrompt` 中编号用途改为“结构化军令字段取值”。
- `AgentPromptBuilder.userPrompt` 中 `JSON schema:` 标题改为“结构化输出格式：”。

## 边界

- 不改 `LLMRequest.responseFormat = "json_object"`。
- 不改 JSON key、schema version、agent id、turn、orders、division id、region id、target id 或 `null` 语义。
- 不改 `AgentOrderType` 的 `move`、`attack`、`hold`、`resupply` rawValue。
- 不改 `AgentDecisionEnvelope`、`AgentOrder`、`AgentDecisionParser`、`LocalLLMDecisionProvider` 或 `AgentCommandMapper`。
- 不改默认战争 AI 主路径；旧 `AgentPromptBuilder` 仍是 legacy LocalLLM / Agent D 回归参考路径。
- 不改 AI 决策策略、命令结构、规则执行、存档字段或运行时状态。

## 并发扫描结论

- prompt 自然语言里的 `hex`、`schema`、`Markdown`、`JSON schema:` 可安全中文化。
- `json_object`、JSON 字段名、`schemaVersion`、`agentId`、`turn`、`orders`、`divisionId`、`targetDivisionId`、`toRegionId` 和 `type` rawValue 必须保留。
- legacy v1 的 `to.q` / `to.r` 坐标 key 仍是解析合同，不能在 schema 兼容路径中文化。

## 轻量检查

- `swiftc -parse WWIIHexV0/Agents/AgentPromptBuilder.swift`：通过，无输出。
- 定向残留扫描：`hex 策略`、`JSON schema`、`schema 的有效 JSON`、`Markdown` 在 `AgentPromptBuilder.swift` 无命中。
- 本轮改动文件尾随空白扫描：通过，无命中。
- 本轮改动文件行首冲突标记扫描：通过，无命中。
- `git diff --check`：通过，无输出。

## 遗留风险

- prompt 示例仍必须保留 JSON 结构、字段名、内部编号和命令类型原值，否则 legacy parser / mapper 合同会失效。
- 本轮只处理 `AgentPromptBuilder` 的 prompt 可见工程说明词；源码注释、变量名和内部 API 名仍保留英文工程语义。
- 本轮没有启用真实 LocalLLM 或 legacy Agent D 管线验证输出质量，只做 prompt 文本入口收口。

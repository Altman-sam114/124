# v3.7-preflight.14 归附空势力轮转收口

完成日期：2026-07-05

## 目标

补上归附关系后的最小生命周期收口。若某个势力已经作为 `submitted` 目标记录，并且当前已经没有存活军队、没有任何可通行受控 hex，则该势力不再进入通用隋唐回合轮转。

## 范围

- `CommandExecutor.turnOrder(in:)` 汇总 diplomacy countries、军队和 hex controller 后，过滤已归附且无实体存在的势力。
- 新增保守 helper 判定：
  - 优先读取 `DiplomacyEventRecord.target == faction && status == .submitted`。
  - 旧存档缺事件记录时，fallback 到关系状态中的 `.submitted`。
  - 实体存在定义为至少一支未毁灭军队，或至少一个可通行受控 hex。
- `ReleaseChecklistView`、README、flow、flowchart、plan、update_log 同步到 v3.7-preflight.14。

## 非目标

- 不转移 hex controller。
- 不转移 region、军队、动态方面、前线或部署归属。
- 不跳过仍有存活军队或受控可通行 hex 的归附势力。
- 不实现完整归附后续事件、借兵、贡赋或天命/民心连锁。
- 不新增本机 Xcode / XCTest / 模拟器验证。

## 架构边界

本轮只影响回合顺序：

```text
Command.endTurn
  -> CommandExecutor.executeEndTurn
  -> advanceActiveFaction
  -> turnOrder
  -> filter submitted target with no runtime presence
```

该过滤不执行地图/军队交接，也不改变外交关系本身。归附后如果目标仍有军队或受控可通行 hex，仍会继续进入轮转，等待后续专门的交接规则。

## 验收标准

- 已归附且无存活军队、无可通行受控 hex 的势力不会出现在通用隋唐 turn order 中。
- 仍有存活军队或受控可通行 hex 的势力不会被跳过。
- legacy `germanAI/alliedPlayer` 分支保持原逻辑。
- 文档一致说明：本轮只做轮转收口，不做地图/军队归属交接。

## 遗留风险

- 尚未做真实 AI/观战多回合运行验证。
- 旧存档若只有 `.submitted` 关系、没有事件记录，fallback 无法区分 issuer/target，只能保守视为相关 faction 已处于归附关系；但仍要求无实体存在才会跳过。

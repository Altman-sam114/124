# v3.7-preflight.103 渡口港口粮道补给减免记录

完成日期：2026-07-07

## 目标

从总提示词 §0.2 “水战、渡河、港口补给与粮道扩展规则”中切出低风险子问题：让既有渡口、港口、海港地点对补给路径产生实际效果。

## 实现

- `SupplyRules.supplyPathCost(from:to:for:in:)` 仍按 hex 图搜索粮道。
- 当粮道跨越 `riverEdges` 时，原规则会增加渡河成本。
- 新增局部 helper：
  - `riverSupplyCrossingCost(from:to:in:)`
  - `hasWaterTransit(at:in:)`
- 若跨河两端任一 hex 存在 `MapFeatureKind.ferry` / `port` / `harbor`，该段补给渡河额外成本为 0。
- 其他跨河路径仍保留原本额外成本。

## 非目标

- 不改移动范围和移动成本。
- 不改战斗隔河防御加成。
- 不把港口/海港新增为补给源。
- 不新增水师、水战、渡河命令或运输命令。
- 不改 JSON schema、MapEditor 文档结构或存档字段。

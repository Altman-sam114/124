# v3.7-preflight.115 水路补给 anchor 粮道显示对齐记录

切片 ID：v3.7-preflight.115

来源：总提示词 §0.2 “粮草与粮道权重”与人工最新方向：古代作战必须重视粮草补给、粮仓、渡口和行军粮道。本轮选择低风险显示切片，让已有规则层补给 anchor 在地图粮道虚线中可见。

当前基线：

- v3.7-preflight.106 已让己控渡口、港口、海港作为 `SupplyRules` 战术补给 anchor。
- `SupplyRules.hasSupplyLine` 已使用永久补给源 + 己控水路通行点。
- `BoardScene.nearestSuppliedSource` 仍只从 `state.map.supplySources(for:)` 选择粮道虚线端点，导致己控渡口/港口/海港只参与规则，不参与地图粮道显示。

## 目标

- 让地图粮道虚线端点复用规则层有效补给 anchor。
- 己控、可通行的渡口、港口和海港能成为玩家可见粮道提示落点。
- 避免在 `BoardScene` 复制一份水路 anchor 判断，降低显示层和规则层口径分叉风险。

## 非目标

- 不改变补给判定、补给成本、补给最大距离、移动、战斗、水战、部署或 JSON schema。
- 不把粮道虚线改成真实逐格路径；当前仍是单位到最近可达 anchor 的直线提示。
- 不新增水路控制模型、运输/登船或 AI 粮道目标权重。

## 实现摘要

- `SupplyRules.effectiveSupplyAnchors(for:in:)` 从私有 helper 调整为内部规则 API，返回稳定去重后的永久补给源和己控水路通行点坐标。
- `BoardScene.nearestSupplyAnchor` 复用 `SupplyRules.effectiveSupplyAnchors(for:in:)` 选择最近可达 anchor。
- `BoardScene` 的 `SupplyRouteKey` 从 `sourceHex` 改为 `anchorHex`，避免语义误导。
- `RuleEngineCoreTests.testControlledWaterTransitCanActAsSupplyAnchor` 增加断言：己控渡口出现在有效 anchor 集合，敌控后不再出现。
- 发布检查说明补充“粮道虚线会提示永久补给源和己控渡口、港口、海港补给落点”。

## 并发子 Agent 发现

- 粮道显示调查确认 `BoardScene` 当前候选只用 `map.supplySources(for:)`，而 `SupplyRules` 的真实补给 anchor 已包含己控 `ferry` / `port` / `harbor`。
- 英雄人物调查建议下一条低风险切片可做“名将军令署名”：复用 `PlayerPlannedOperation.createdByGeneralId` 和 `WarDirectiveRecord.commanderAgentId`，在军令/战报摘要显示李世民等名将姓名，不改规则。
- AI 粮草权重仍需独立切片，避免本轮把地图显示、AI 选择和战报人物同时混改。

## 风险与后续

- 粮道虚线仍是直线提示，不代表真实逐格路径；后续若要显示完整粮道路径，需要独立设计 path rendering。
- 地图显示未做本机运行时截图复核，层级、遮挡和水路点重叠仍需后续验证。
- 后续优先切片：名将军令署名、AI 粮草/粮道目标权重、正式地图资产与运行时截图复核。

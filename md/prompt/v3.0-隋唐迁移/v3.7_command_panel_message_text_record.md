# v3.7 CommandPanel 命令消息展示净化记录

日期：2026-07-06

## 目标

继续推进 `codex-v3.0-隋末唐初aiagent历史策略迁移总提示词.md` 的 v3.7 发布候选前置收口，补齐 `CommandPanelView` 对 `lastCommandMessage` 的最后展示兜底，避免玩家可见命令消息直出 raw id、工程词、模型词或旧题材词。

## 范围

- 修改 `WWIIHexV0/UI/CommandPanelView.swift`。
- 新增命令面板本地展示 helper：`displayCommandMessage(_:)` 和 `sanitizeCommandRawIdentifiers(in:)`。
- 更新 `README.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 与 `md/plan/plan.md`。

## 实现摘要

- `CommandPanelView` 显示 `lastCommandMessage` 前先调用 `displayCommandMessage(_:)`。
- `sanitizeCommandRawIdentifiers(in:)` 清理 `war_directive_*`、`player_directive_*`、`player_operation_*`、`submission_handoff_*`、`submission_aftermath_*`、`diplomacy_event_*`、`region_*`、`theater_*`、`front_zone_*`、`obj_*`、`hex_*`、`division_*`、`unit_*` 和 `command_*` 等常见内部 id。
- 展示词表补齐 `rawJSON`、`JSON`、`schema`、`RuleEngine`、`ZoneDirective`、`WarDeploymentState`、`FrontZone`、`Division`、`Command`、`Provider`、`local-model`、`OpenAI`、`GPT`、`Claude`、`Gemini`、`LLM`、`MockAI`、`AI`、`pipeline`、`fallback` 和 `hex` 等玩家可见兜底。

## 并发扫描结论

- CommandPanel 子 Agent 确认 `CommandPanelView` 过去原样显示 `lastCommandMessage`，是玩家可见出口；普通命令反馈多为稳定中文，但总管军令诊断和后续上游改动仍可能带入 raw id 或工程词。
- 文档子 Agent 建议本轮使用 `v3.7-preflight.71`，记录文件名为 `v3.7_command_panel_message_text_record.md`，并同步 `README.md`、`update_log.md`、`md/flow/*` 与 `md/plan/plan.md`。
- UI 抽样子 Agent 额外发现 `UnitInspectorView`、`UnitTooltipView`、`RegionInspectorView`、`GeneralProfileView` 与 `MapDisplayAdapter` 仍适合作为下一轮 legacy 名称直出复扫候选；本轮不混入这些文件。

## 边界

- 不改 `AppContainer.lastCommandMessage` 的存储语义。
- 不改交互日志、存档字段、`Command` case、`CommandResult` schema、`CommandValidationError` rawValue、校验逻辑、`RuleEngine` 执行语义、AI 决策、命令结构、hex / region / theater / front / deploy 权威或运行时状态。
- 不抽共享 sanitizer，避免在脏工作区中扩大公共 API 和文件冲突面。

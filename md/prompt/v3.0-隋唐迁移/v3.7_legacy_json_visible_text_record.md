# v3.7-preflight.45 Legacy JSON 可见文本收口记录

完成日期：2026-07-06

## 目标

继续推进 v3.7 发布候选前置文案收口，处理 legacy fallback JSON 中仍可能经人工验收、MapEditor 兼容路径、旧剧本加载或数据查看进入视野的英文展示字段。

本轮只清理低风险展示字段，不迁移 legacy schema，不替换 id，不改 rawValue，不改变默认 `wude_618` 隋唐入口，也不改变任何规则、AI 或命令管线。

## 改动范围

- `WWIIHexV0/Data/ardennes_v0_scenario.json`
  - `displayName` 从 `MapEditor Scenario` 改为中文旧剧本名称。
  - `dataNotes` 从英文导出说明改为中文说明。
  - `keyLocations[].name` 的 `City q,r` / `Supply q,r` 改为 `城邑 q,r` / `补给点 q,r`。
  - `map.tiles[].cityName` 的 `City q,r` 改为 `城邑 q,r`。
- `WWIIHexV0/Data/ardennes_v02_regions.json`
  - `displayName` 改为中文旧剧本州郡名称。
  - `regions[].city.name` 的 `City q,r` 改为 `城邑 q,r`。
- `WWIIHexV0/Data/unit_templates.json`
  - 仅把 legacy 单位模板 `displayName` 改为中文：装甲师、摩托化师、步兵师、炮兵师、反装甲师、守备师。

## 保留边界

- 保留所有 JSON key、schemaVersion、scenario id、region id、theater id、unit id、template id、objective id。
- 保留 legacy `germany` / `allies`、`alliedPlayer`、`tank`、`motorizedInfantry` 等 rawValue 和兼容枚举。
- 保留 `unit_templates.json` 的 `components[].type` 和权重，不把旧模板改造成隋唐兵种模板。
- 暂不处理 `generals.json`、`general_agents.json` 和 `suitang_generals.json` 的姓名、rank、biography 或 `personalityPrompt`，这些需要独立语义审计。
- 不修改 `DataLoader` 加载顺序、MapEditor 导出合同、`Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 管线或任何规则数值。

## 轻量检查要求

- `jq empty WWIIHexV0/Data/ardennes_v0_scenario.json WWIIHexV0/Data/ardennes_v02_regions.json WWIIHexV0/Data/unit_templates.json`
- `git diff --check`
- 本轮改动文件尾随空白扫描。
- 本轮改动文件冲突标记扫描。
- 定向残留扫描：`MapEditor Scenario`、`Generated by MapEditor`、`Region neighbors`、`City q,r`、`Supply q,r`、旧单位模板英文显示名。

## 未执行

按 `md/test/test.md` 当前规则，本轮不跑 Xcode build、XCTest、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full、性能测试或 app 启动。运行时加载和 UI 展示仍需后续授权验证或云端结果包复核。

## 遗留风险

- legacy faction rawValue、旧剧本 id、region id、unit id、component rawValue 仍会在内部 JSON 和调试审计中保留。
- `generals.json` 与 `general_agents.json` 仍含英文姓名、军衔、履历或内部 prompt，后续应单独处理。
- 本轮只做 JSON 解析级检查，未确认旧 fallback 剧本在真实 UI / MapEditor 中的展示效果。

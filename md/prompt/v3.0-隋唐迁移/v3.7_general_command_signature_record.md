# v3.7-preflight.116 名将军令署名显示记录

完成日期：2026-07-07

## 切片定位

来源：人工最新方向要求突出“天下局势”沉浸感和隋唐英雄人物存在感；并发子 Agent 定位显示，`PlayerPlannedOperation.createdByGeneralId` 与 `GeneralRegistry.general(id:)` 已能支撑预备军令署名，不需要新增存档字段或命令 schema。

目标：让玩家在军令面板和提交反馈中看见“某将军令”，把既有总管/将领分配转化为更明确的历史人物存在感。

非目标：

- 不改 `PlayerPlannedOperation`、`WarDirectiveRecord`、`Command` 或 `ZoneDirective` schema。
- 不改 AI 决策、命令校验、规则执行、战斗、补给、部署或 JSON 数据。
- 不实现将领技能消费、忠诚影响、统率数值加成、战报全链路署名或真实头像资产。

## 实现

- `GeneralCommandPanelView` 标题从“总管军令”改为“名将军令”。
- `GeneralCommandPanelView` 接收 `GeneralRegistry`，在预备军令摘要中用 `createdByGeneralId` 解析将领名。
- 可解析将领时显示“某将军令：进军/固守：目标”；旧记录或缺失将领时保守显示“总管军令”。
- `RootGameView` 把 `container.generalRegistry` 传入两个 `GeneralCommandPanelView` 调用点。
- `AppContainer` 玩家军令提交反馈使用同一 registry 口径显示“某将军令已提交”；缺少将领时保留“总管军令已提交”。

轻量检查按 `md/test/test.md` 执行；本机不跑 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。云端重验证由 push `origin/main` 后的 GitHub Actions 结果包承担。

遗留风险：

- 运行时 UI 布局未在本机模拟器或截图中复核。
- 战报列表、军情面板和 AI 决策 rationale 的全链路署名仍待后续切片。
- 名将统率、武勇、谋略、忠诚和技能仍未实际影响规则。

# v3.7-preflight.26 AI 太守跳过诊断

## 目标

在 v3.7-preflight.11/.20/.23/.24 已让 AI 太守主动经营、优先善后州郡并在全部处置后退出善后优先队列之后，补上无经营命令时的确定性诊断：当朝堂存在太守步骤，或仍有未完成的高/需安抚善后上下文，但 `TurnManager.governorCommand` 未生成 `Command.governRegion` 时，把原因写入 `AgentDecisionRecord.errors`，避免 AI 决策面板只看到静默空结果。

## 范围

- `TurnManager.executeGovernorCommands` 在 `governorCommand` 返回 nil 时调用 `governorSkipDiagnostics`。
- 新增 `actionableAftermathRecord`，复用最新善后记录筛选逻辑：接收方匹配、风险不是低、且善后处置尚未完成。
- `aftermathGovernorRegionIds` 改用 `actionableAftermathRecord`，保持善后优先候选和跳过诊断口径一致。
- `governorSkipDiagnostics` 只在存在朝堂太守步骤或可行动善后记录时输出原因。
- 发布前检查面板和文档同步到 `v3.7-preflight.26`。

## 非目标

- 不新增命令类型，不增加 AI 回合行动次数。
- 不改变 `Command.governRegion` 的校验、消耗、执行效果或优先级评分。
- 不让 AI 太守绕过 `CommandValidator`、`RuleEngine` 或 `CommandExecutor`。
- 不清零、降级、删除或解决 `SubmissionAftermathRecord`。
- 不新增忠诚、叛乱、贡赋、俘虏、安置、治安长期状态机或额外资源变化。
- 不批量处理历史善后压力；当前仍只读取最新一条可行动善后记录。

## 数据流

```text
CourtDecisionRecord governor step
  或 latest actionable SubmissionAftermathRecord
  -> TurnManager.governorCommand(...)
  -> 未生成 Command.governRegion
  -> governorSkipDiagnostics(...)
  -> AgentDecisionRecord.errors
  -> AgentPanelView 错误/诊断区展示
```

诊断原因包括：

- 没有当前势力控制、可通行且包含实际受控 hex 的州郡。
- 没有适用或府库可负担的经营政策。
- 最新善后记录仍有多少受影响州郡未处置。
- 兜底说明没有候选通过评分和政策检查。

## 架构边界

- Hex 仍是战术权威；本轮不修改 `HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- Region 仍是战略聚合层；本轮只读取州郡是否可由太守考虑，不改变州郡状态。
- 善后完成状态仍是审计和 UI 口径，不代表善后压力在规则层被清除。
- AI 太守仍最多生成一条既有 `Command.governRegion`；生成失败时只写诊断，不补发其他命令。
- 所有实际治理效果仍必须经 `CommandValidator -> RuleEngine -> CommandExecutor`。

## 风险与后续

- 没有真实 AI/观战多回合验证时，只能确认源码路径和轻量语法，不能确认运行中一定触发每一种诊断。
- 诊断只说明为什么本回合没有生成太守经营命令，不证明太守候选评分或府库资源状态完全正确。
- 后续若实现忠诚、叛乱、贡赋、俘虏或安置，必须新增清晰的命令、validator 和 executor，不能让跳过诊断承担规则效果。

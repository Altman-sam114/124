# v3.7-preflight.105 MapEditor 河边绘制擦除入口记录

完成日期：2026-07-07

## 目标

在 v3.7-preflight.104 已保证 `riverEdges` 数据往返保真的基础上，为 MapEditor 增加最小河边编辑入口，让编辑器可以绘制和擦除当前 hex 的河边数据。

## 实现

- `MapEditorHexTool` 增加 `.riverEdge`。
- `MapEditorViewModel` 新增：
  - `beginPaintingRiverEdges()`
  - `beginErasingRiverEdges()`
  - `applyRiverEdgeAction(at:direction:)`
- `MapEditorCanvasScene` 在河边工具下，把点击点映射到当前 hex 最近的六边形边，并调用 ViewModel 修改该方向。
- `MapEditorCanvasScene` 为 `hex.riverEdges` 绘制蓝色边线，作为即时视觉反馈。
- `MapEditorView` 地块模式新增“绘制河边 / 擦除河边”按钮，信息面板显示选中地块河边方向。
- `MapEditorOutputTests` 增加 ViewModel 河边添加/擦除语义断言。

## 非目标

- 不自动写入相邻 hex 的反向河边。
- 不改默认 `wude_618_scenario.json`。
- 不改 JSON schema。
- 不改运行时移动、战斗、补给、水战或河流判定规则。
- 不把河边编辑扩展到 region 边或战略层。

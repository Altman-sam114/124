# v3.7-preflight.66 legacy MockAI stance 文案收口

完成日期：2026-07-06

## 目标

在 legacy prompt 决策者身份净化之后，继续处理 legacy `MockAIClient` 生成的 `AgentOrder.stance` 中仍保留 `recover`、`fireSupport`、`breakthrough`、`roadAdvance`、`frontRecovery` 等英文自由文本的问题，避免旧 Agent D / MockAI 回归路径把英文姿态写入 raw JSON、AI 面板或战报摘要。

## 实现范围

- `MockAIClient` 中所有直接赋值的 `AgentOrder.stance` 改为中文短语：
  - `recover` -> “整补恢复”
  - `fireSupport` -> “火力支援”
  - `breakthrough` -> “突破”
  - `roadAdvance` / `advance` -> “沿路推进” / “稳步推进”
  - `hold` -> “固守”
  - `frontRecovery` -> “前线整补”
  - `closePocket` / `frontAttack` -> “收紧包围” / “前线进攻”
  - `containPocket` / `holdFront` -> “围堵敌军” / “固守前线”
  - `depthReinforce` / `depthReserve` -> “纵深驰援” / “纵深待命”
  - `garrison` -> “驻防”
  - `frontUnassigned` / `operationalReserve` -> “前线待命” / “战役预备”

## 边界

- 不改 `AgentOrderType` 的 `move`、`attack`、`hold`、`resupply` rawValue。
- 不改 `divisionId`、`targetDivisionId`、`toRegionId`、`regionId`、objective id 或任何 parser / mapper 合同。
- 不改 MockAI 决策策略、排序、目标选择、行军目的地、规则执行或运行时状态。
- 不改默认战争 AI 主路径；`MockAIClient` 仍是 legacy / fallback / 回归参考路径。

## 遗留风险

- `AgentOrder.type`、内部 id 和 objective id 仍保持英文 / rawValue，这是解析和规则管线合同，不能在本片中文化。
- `MockAIClient` 内部排序、变量名和 legacy 注释仍有英文开发语义；本轮只处理可能进入模型输出 / raw JSON / UI 摘要的 `stance` 自由文本。
- 本轮没有启用 legacy MockAI 回合运行或 UI 截图验证，只做静态语法和定向文本检查。

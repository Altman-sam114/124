# v3.7-preflight.7 存档错误反馈记录

完成日期：2026-07-05

## 1. 本轮定位

本轮继续补 v3.7 发布候选前置的“错误恢复”缺口，目标是让本地存档读取、保存和删除结果进入玩家可见 UI。

本轮不改变存档 JSON schema，不新增多槽位、导入导出、确认弹窗、自动重试或迁移器，也不改变 `GameState`、命令、战区、部署、前线或规则系统。

## 2. 实现摘要

### 2.1 存档状态模型

`GameSaveStore.swift` 新增：

```text
GameSaveStatus
  -> severity: success / notice / failure
  -> title
  -> detail
  -> needsAttention
```

该类型只用于运行时 UI 反馈，不编码进本地存档。

### 2.2 AppContainer 生命周期反馈

`AppContainer` 新增：

```text
@Published private(set) var saveStatus: GameSaveStatus?
```

更新点：

- 启动读取本地存档成功：显示“已载入本地存档”。
- 启动读取失败并 fallback 默认剧本：显示“读取存档失败 / 已开启默认剧本”。
- 继续时没有存档：显示“没有可继续的存档”。
- 继续读取失败：显示读取失败原因。
- 新局、玩家命令、玩家总管军令、AI 回合后的自动保存成功：显示“本地存档已更新”。
- 自动保存失败：显示“本地存档失败”和系统错误原因。
- 重置删除失败：显示“删除存档失败”，并说明旧存档仍在本地。

### 2.3 UI 显示

新增 `SaveStatusBanner`：

- HUD 只在 `saveStatus.needsAttention` 时显示，避免正常自动保存常驻占屏。
- `GameSettingsView` 显示当前存档反馈。
- `ReleaseChecklistView` 显示当前存档反馈，并把版本标记更新为 `天命开唐 Agent · v3.7-preflight.7`。

## 3. 架构边界

本轮数据流是：

```text
GameSaveStore load / save / delete
  -> AppContainer.saveStatus
  -> HUD / GameSettingsView / ReleaseChecklistView
```

它不写回：

- `GameState` schema
- `Command` / `ZoneDirective`
- `WarCommandExecutor`
- `RuleEngine`
- `HexTile.controller`
- `Division.coord`
- `hexToTheater`
- `hexToFrontZone`

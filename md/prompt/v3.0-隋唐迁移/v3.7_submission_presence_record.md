# v3.7-preflight.15 归附实体盘点与误判收口

完成日期：2026-07-05

## 目标

在 v3.7-preflight.13/.14 已有归附事件记录和空势力退出轮转的基础上，补上归附目标的运行时实体盘点，让玩家和后续验收能看到归附势力是否仍因存活军队或受控可通行 hex 留在通用隋唐回合轮转中。

## 已完成

- `DiplomacyState` 新增 `isSubmittedTarget(_:)` 和 `submittedTargetFactions()`，统一归附目标判定。
- 新归附事件记录存在时，只以 `DiplomacyEventRecord.target` 作为归附目标；旧存档缺事件记录时，才 fallback 到对称的 `.submitted` 关系状态。
- `CommandExecutor.turnOrder(in:)` 复用 `DiplomacyState.isSubmittedTarget(_:)`，避免规则层复制归附判定。
- `AppContainer.submissionPresenceSummaries` 只读统计归附目标的存活军队和受控可通行 hex。
- `DiplomacyPanelView` 新增“归附实体”区，显示归附目标是否仍有实体存在，以及对应的回合轮转影响。
- `ReleaseChecklistView`、README、flow、flowchart、plan、update_log 同步到 v3.7-preflight.15。

## 边界

- 本轮只做实体盘点、展示和归附目标误判收口。
- 不直接转移 hex controller、region controller、军队 faction、动态方面、前线或部署归属。
- 不新增交接按钮，不绕过 `Command` / `RuleEngine`。
- 有实体存在时的实际地图/军队归属交接执行规则仍待后续版本设计和实现。

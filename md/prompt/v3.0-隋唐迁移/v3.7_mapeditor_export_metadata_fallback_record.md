# v3.7-preflight.60 MapEditor 导出元数据 fallback 收口

完成日期：2026-07-06

## 目标

在战报意图屏蔽与中文分类收口之后，继续处理 MapEditor 普通导出路径中未显式传入 `MapEditorExportMetadata` 时，未知文档 id 可能默认落到 legacy 阿登元数据的问题。该版本只收紧 metadata 推断口径，不改变 JSON schema、MapEditor 文档字段、keyLocations 合并规则、主游戏加载规则或运行时玩法。

## 实现范围

- `MapEditorExportMetadata.inferred(for:)` 保留 `wude_618` 文档使用 `.wude618Default`。
- 文档 id 或展示名明确包含隋唐口径时使用 `.suitangDraft`。
- 只有文档 id 或展示名明确包含 `legacy`、`ardennes`、`wwii`、`阿登` 或 `旧战局` 时才使用 `.legacyArdennes`。
- 其他未知自定义文档默认使用 `.suitangDraft`，避免普通新地图误导出 legacy faction / phase / dataNotes。

## 边界

- 不改 `MapEditorExportMetadata` Codable / 数据结构。
- 不改 `MapEditorGameResourceBridge.defaultExportMetadata()`，默认覆盖保存仍优先读取当前 `wude_618_scenario` 作为 metadata 权威。
- 不改 `keyLocationsAreAuthoritative` 语义；当前它只控制 metadata keyLocations fallback，不禁用 hex 派生地点。
- 不改 `generatedKeyLocations`、objective 合并、region 导出、DataLoader、规则或地图运行时。

## 遗留风险

- 静态 `.wude618Default` 仍只是降级 metadata，不完整携带当前默认 JSON 的渡口、港口、海港和 objective 点数；默认资源桥覆盖保存仍应优先使用当前 JSON 读取出的 metadata。
- `keyLocationsAreAuthoritative` 与 hex 派生地点的长期语义仍可后续单独整理，但不应混入本小片。

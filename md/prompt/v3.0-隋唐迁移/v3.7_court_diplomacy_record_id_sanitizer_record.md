# v3.7-preflight.81 朝堂外交记录 id 展示净化收口

完成日期：2026-07-06

## 目标

在 legacy fallback 行军总管配置收口之后，继续按并发子 Agent 扫描结论补齐朝堂、君主和外交记录实际 id 的展示净化缺口，避免 `ruler_*_turn_*`、`court_*_turn_*`、`court_<turn>_*` 和 `diplomacy_<turn>_*` 在战报、朝堂面板、外交面板、命令面板或 LocalLLM prompt 中以 raw id 形式出现。

## 范围

- 只改玩家可见文本、源头战报 message 和 prompt 输入净化。
- 只修正 `DiplomacyState.summary(for:)` 的展示文案语义，把“敌对关系 N 条”改为“敌对国家 N 个”。
- 不改 `RulerDecisionRecord`、`CourtDecisionRecord`、`CourtAgentStepRecord`、`DiplomacyEventRecord` 或 `AgentDecisionRecord` 的 Codable 字段。
- 不改已持久化 id 生成格式、`relatedRecordId`、raw JSON 审计合同、外交关系判定、命令管线、规则或存档 schema。

## 实现

- `TurnManager`：
  - `courtRecord.summary` 写入 `parsedIntent` 前先走 `userFacingDiagnostic`。
  - “朝堂决策完成”事件写入前先净化 summary。
  - `canonicalCourtDirectiveJSON` 的朝堂步骤摘要使用展示净化文本，保留 directive JSON 结构化审计合同。
  - `sanitizeRawIdentifiers` 补齐实际 `diplomacy_<turn>_*`、`court_<faction>_turn_<turn>`、`court_<turn>_*` 和 `ruler_*_turn_*` 记录 id。
- `DiplomacyState`：
  - `summary(for:)` 改为“敌对国家 N 个”，匹配 `hostileCountryIds(to:)` 的实际去重国家计数。
  - summary 中 legacy `.germany` / `.allies` 和常见旧国家名使用中性展示口径。
- `AgentPanelView`、`DiplomacyPanelView`、`EventLogView`、`CommandPanelView`、`GameLogEntry`、`AgentPromptBuilder`：
  - 同步补齐实际朝堂 / 君主 / 外交记录 id 的展示净化 regex。
  - 使用较精确的实际记录形态，避免宽泛匹配未来业务词。

## 边界

- 不重命名任何 record id。
- 不把展示净化文本反写进外交关系、朝堂记录或规则输入。
- 不把 UI 摘要文本作为规则权威。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> CommandValidator -> RuleEngine` 统一入口。
- 不运行本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。

## 轻量检查

本记录要求本轮至少执行：

- `swiftc -parse WWIIHexV0/Core/DiplomacyState.swift`
- `swiftc -parse WWIIHexV0/Turn/TurnManager.swift`
- 针对改动 Swift 文件的 `swiftc -parse` 能通过的入口。
- 聚焦扫描确认实际 id regex 已同步到相关展示净化 helper。
- 尾随空白、冲突标记和 `git diff --check`。

## 遗留风险

- 未跑运行时 AI 回合，无法证明实际朝堂记录在多回合中所有 UI 路径都被覆盖。
- 多处展示净化 helper 仍有重复实现，未来若继续扩大记录类型，建议抽共享展示 sanitizer，但本轮为降低冲突面没有新增公共 API。
- `rawJSON` 仍保留结构化审计原文，不承诺完全去工程字段。

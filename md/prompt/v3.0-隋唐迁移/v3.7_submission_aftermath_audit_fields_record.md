# v3.7-preflight.111 归附善后四类审计字段记录

切片 ID：v3.7-preflight.111

来源：`codex-v3.0-隋末唐初aiagent历史策略迁移总提示词.md` §0.2 当前队列中的“完整忠诚、叛乱、俘虏、安置等归附善后实际规则”。本轮先做低风险只读审计字段，不进入真正规则效果。

当前基线：

- `origin/main` 已包含 v3.7-preflight.110，commit `a802ba7`。
- v3.7-preflight.101 已让善后压力写入州郡治安/顺从。
- v3.7-preflight.102 已让治安/顺从折算贡赋效率。
- 当前仍没有完整忠诚状态机、叛乱生成、俘虏处置或安置命令。

## 目标

- 扩展 `SubmissionAftermathRecord`，记录归附交接后的四类后续复核信息：
  - 忠诚压力。
  - 叛乱风险。
  - 俘虏整编数量。
  - 安置州郡候选。
- 在外交面板“善后”区展示这些只读字段。
- 保持旧存档兼容：旧 `SubmissionAftermathRecord` 缺新字段时可按既有风险和数量派生默认值。

## 非目标

- 不新增叛乱触发、叛军单位、俘虏处置命令或安置命令。
- 不扣资源，不改变军队/hex/region/theater/front/deploy 权威。
- 不新增 JSON 数据 schema，不改变 `Command` case，不增加 AI 行动次数。

## 实现摘要

- `SubmissionAftermathRecord` 新增 `loyaltyPressure`、`rebellionRisk`、`captiveReviewDivisionCount`、`settlementRegionIds`。
- 为 `SubmissionAftermathRecord` 增加显式 init / decode，旧记录缺字段时：
  - `loyaltyPressure` 默认等于既有 `riskLevel`。
  - `rebellionRisk` 在无受影响州郡时为低，否则默认等于既有 `riskLevel`。
  - `captiveReviewDivisionCount` 默认等于交接军队数量。
  - `settlementRegionIds` 默认等于受影响州郡并稳定排序。
- `DiplomacyPanelView` 展示四类审计字段，并把边界提示更新为“当前只影响治理优先级”。
- `RuleEngineCoreTests` 增加语义断言，保护四类字段派生结果。

## 并发子 Agent 发现

- 善后调查子 Agent 确认当前最接近承载点是 `SubmissionAftermathRecord` + `OccupationState` + `Command.governRegion`。
- 代码中已有将领/Agent 层 `loyalty` 字段，但没有归附势力/州郡忠诚模型。
- 当前没有 rebellion/captive/settlement 模型或命令；直接实现完整规则会触碰更大设计面。
- 因此本轮采用只读审计模型，避免提前引入未设计完成的规则效果。

## 风险与后续

- 本轮没有启动 App，因此外交面板布局仍需后续运行时截图复核。
- 忠诚、叛乱、俘虏和安置目前只是复核字段；真正规则应后续单独切片，并继续走 `Command` / validator / executor 或明确的新规则入口。
- 水战与更细水域控制仍在队列中；本轮未混入隔河攻击、水师部署或水域控制校验。

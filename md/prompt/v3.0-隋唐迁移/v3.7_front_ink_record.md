# v3.7-preflight.6 前线墨线记录

完成日期：2026-07-05

## 1. 本轮定位

本轮继续补 v3.7 发布候选前置的地图可读性缺口，目标是在普通地图层直接显示动态前线。

本轮只读 `FrontLineState`，不新增前线规则、不改变动态战区、不改变部署层、不修改 `GameState` 战术权威，也不改变玩家和 AI 仍需走 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 的执行边界。

## 2. 实现摘要

### 2.1 普通地图层前线墨线

`BoardScene.redraw()` 在 roads、rivers 之后、粮道/围城/计划线之前调用：

```text
drawFrontInkOverlays(renderState:layout:)
```

该层通过：

```text
MapLayerOverlayCalculator(state: gameState).frontLineChains()
```

读取现有前线链，并在非 `.frontLine` 图层绘制最小墨线。

### 2.2 视觉规则

- 普通、稳定前线绘制为墨色接触线。
- `encirclement` 或 `collapsing` 前线加粗，并追加朱色警示虚线。
- `breakthrough` 前线使用偏朱褐色墨线。
- `pressure` 会轻微增加线宽和透明度。
- 单点接触绘制为墨色接触标记，避免短前线不可见。

`.frontLine` 专用深色图层仍使用原 `MapLayerOverlayNode`，本轮不替换它。

## 3. 架构边界

本轮数据流是：

```text
FrontLineState
  -> MapLayerOverlayCalculator.frontLineChains()
  -> BoardScene 墨线 / 朱色警示虚线
```

它只做地图显示，不写回：

- `HexTile.controller`
- `Division.coord`
- `TheaterState.hexToTheater`
- `WarDeploymentState.hexToFrontZone`
- `FrontLineState`
- `GameState`

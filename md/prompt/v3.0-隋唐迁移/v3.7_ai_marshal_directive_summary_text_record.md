# v3.7-preflight.43 AI 元帅/方面军令摘要中文化记录

完成日期：2026-07-06

## 目标

在默认数据说明和补给战报中文化之后，继续处理 `MarshalAgent -> TheaterDirective -> ZoneDirective` 链路中仍可能进入 AI 面板、战报摘要、审计记录或未来 raw JSON 展示的英文工程口径，减少 `zone directive(s)`、`theater directive(s)`、`Simulated marshal JSON`、`Compiled ... zone directive(s)`、`front status` 和元帅 fallback 诊断等残留。

## 实现摘要

- `TheaterCommanderPool` 生成的 `theaterContext` 改为中文“生成 N 条方面军令”。
- 模拟元帅写入 `TheaterDirective.rationale` 的攻守理由改为中文军议说明，并用 `TacticName.displayName` 展示战术名。
- 模拟元帅 `TheaterDirectiveEnvelope.summary` 改为中文前线汇总摘要。
- `strategicIntent` 三种战略偏向的字符串改为中文，保留 JSON 字段名和解析合同。
- `TheaterDirectiveCompiler` 编译后的 `theaterContext` 改为中文“已编成 N 条方面军令”。
- 元帅势力不匹配、元帅军令解析或编译失败的 fallback 诊断改为中文。
- `ZoneCommanderAgent` 自动 fallback 名称改为“势力名总管”，避免新增可见 `Commander` 口径。

## 边界

- 本轮只改 AI 元帅/方面军令摘要、rationale、context 和诊断字符串。
- 不改 `TheaterDirectiveEnvelope` / `TheaterDirective` JSON schema，不改 `strategicIntent`、`rationale`、`rawTheaterJSON` 等字段名。
- 不改 `TacticName.rawValue`、front status 判定值、directive id、zone id、region id 或任何 rawValue。
- 不改 AI 决策分支、战术选择、priority、intensity、reserveBias、maxCommittedUnits 或编译规则。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`，不改变移动、攻击、补给、外交、州郡经营、胜负或回合推进规则。

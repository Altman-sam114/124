# v3.7-preflight.2 新局、继续、重置与本地自动存档记录

完成日期：2026-07-05

## 1. 性质

本轮是完整 v3.7 发布候选前置补洞，不是完整 v3.7 发布候选。

目标是补齐总提示词中“新局 / 继续 / 重置”的最小运行时入口，并为默认隋唐剧本提供本地自动存档。新手引导、设置页、版本说明、发布候选检查清单和运行时授权重测仍不在本轮范围内。

## 2. 实现范围

### 2.1 GameSaveStore

新增 `WWIIHexV0/App/GameSaveStore.swift`：

```text
GameSaveStore
  -> saveURL: Documents/WWIIHexV0-current-game.json
  -> hasSavedGame
  -> load() throws -> GameState
  -> save(_ state: GameState) throws
  -> deleteSave() throws
```

存档内容只包含 `GameState`，依赖现有 Codable 模型，不引入新 schema 或外部依赖。

### 2.2 AppContainer 生命周期

`AppContainer.bootstrap()` 启动时：

```text
尝试读取 GameSaveStore
  -> 成功：以保存的 GameState 初始化容器
  -> 失败：回到 DataLoader.loadInitialGameState()
```

自动保存触发点：

- 玩家底层 `Command` 执行后。
- 玩家总管 `ZoneDirective` 执行后。
- AI 回合结算后。

新增/调整入口：

- `startNewGame()`：加载默认剧本、清空临时选择、写入新局日志并立即保存。
- `continueSavedGame()`：读取本地存档、刷新运行时派生状态、写入继续日志。
- `resetGame()`：删除存档、恢复默认剧本、清空临时选择并写入重置日志。

### 2.3 UI 入口

HUD：

- 原单一新局按钮扩展为“局势”菜单。
- 菜单项：新局、继续、重置。
- 继续在 `hasSavedGame == false` 时禁用。

macOS 菜单：

- 顶部菜单从 `Game` 改为“局势”。
- 菜单项：结束回合、新局、继续、重置。

## 4. 架构边界

- 本轮不改 `Command`、`ZoneDirective`、`WarCommandExecutor`、`RuleEngine`。
- 保存/加载只作用于 `GameState`，不直接修改 `HexTile.controller`、`Division.coord`、`regionToTheater`、`hexToTheater` 或 `hexToFrontZone`。
- UI 只调用 `AppContainer` 生命周期方法，不直接读写文件。
- 存档不覆盖 UI 临时态：选择单位、选择 hex、面板展开、地图图层、观察模式和高亮都仍由当前容器会话管理。
- 不引入多槽位、云存档、导入导出、版本迁移器或确认弹窗。

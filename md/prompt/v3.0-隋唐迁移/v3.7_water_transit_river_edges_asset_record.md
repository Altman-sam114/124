# v3.7-preflight.107 默认水路地点河边资产补录记录

完成日期：2026-07-07

## 目标

在 v3.7-preflight.103-.106 已接入渡口/港口跨河补给、MapEditor 河边保真/编辑入口和受控水路补给投送点后，补齐默认 `wude_618` 场景中水路地点自身 hex 的最小河边资产，让后续规则可以直接消费真实 `riverEdges` 输入。

## 实现

- 蒲津渡 `(4,3)` 补录 `riverEdges: ["northEast"]`。
- 孟津渡 `(6,4)` 保留既有 `east`，补录 `northEast`。
- 黎阳津 `(7,4)` 保留既有 `east`，不重复写入对侧河边。
- 洛口津 `(6,6)` 补录 `riverEdges: ["west"]`。
- 四个默认 `ferry` / `port` 地点自身 hex 均有非空 `riverEdges`。

## 非目标

- 不改变 JSON schema、地图尺寸、地形、地点、单位或势力。
- 不新增移动、战斗、水战、港口部署点或港口补给源规则。
- 不自动写入邻接 hex 的反向河边，避免 SpriteKit 单边/双边混合渲染时重复描边。
- 不声称默认地图真实河网已经完整补齐。

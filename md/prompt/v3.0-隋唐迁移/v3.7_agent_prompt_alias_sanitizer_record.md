# v3.7 AgentPromptBuilder prompt 别名与展示净化记录

日期：2026-07-06

## 目标

继续 v3.7 发布候选前置补洞，按并发子 Agent 扫描结论收口旧 `AgentPromptBuilder` 面向 LocalLLM 的 prompt 输入风险，避免目标、军队、州郡、势力和内部 id 在 prompt 自由文本中继续暴露 legacy 二战口径或 raw id。

## 改动

- `AgentPromptBuilder` 的系统 prompt 势力名改走 `displayFactionName(_:)`，legacy `.germany` / `.allies` 显示为“旧剧本势力”。
- 战场摘要中的目标、己方军队、敌军、可见州郡和编号旁说明名改走 `displayMapName` / `displayDivisionName` / `displayFactionName`。
- `sanitizePromptText(_:)` 补齐 legacy 势力名、旧地名、旧题材单位词、模型/工程词、`objective_*` raw id 的净化。
- 新增 `AgentPromptAliasBook`，把 prompt 内的 `divisionId` / `targetDivisionId` / `toRegionId` / `agentId` 可选取值显示为“军队一 / 敌军一 / 州郡一 / 本地决策者”等临时编号。
- `LocalLLMDecisionProvider` 在解析 LLM JSON 后用同一本别名表回填真实 id，再交给既有 mapper。

## 边界

- 不改 `AgentDecisionEnvelope` / `AgentOrder` schema。
- 不改 JSON 字段名、`AgentOrderType` rawValue、parser 支持版本、命令结构、`AgentCommandMapper`、规则系统或 AI 默认主链路。
- 不改 `Division.name`、`RegionNode.name`、objective id、Faction rawValue 或存档格式。
- 旧 LocalLLM 仍是 deprecated / 非默认路径；本轮只降低该路径被启用时的 prompt 泄漏风险。

## 轻量检查

- `swiftc -parse WWIIHexV0/Agents/AgentPromptBuilder.swift`：通过，无输出。
- `swiftc -parse WWIIHexV0/Agents/LocalLLMDecisionProvider.swift`：通过，无输出。

## 遗留风险

- 本轮未实际调用 LocalLLM，未验证模型是否稳定返回临时编号。
- JSON 字段名和命令 type 原值仍保留为解析合同，若未来要求 LLM 完全不可见工程字段，需要引入更大的工具协议 / adapter 层。
- `ZoneCommanderAgent` fallback config、`AgentConfiguration` legacy fallback 指挥官和 `RulerAgent` 朝堂 step 记录仍需下一轮继续收口。

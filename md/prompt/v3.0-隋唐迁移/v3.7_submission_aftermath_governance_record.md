# v3.7-preflight.21 善后处置审计记录

## 目标

在 v3.7-preflight.19 归附善后压力记录和 v3.7-preflight.20 AI 善后太守优先治理之后，补上最小处置审计闭环：当既有 `Command.governRegion` 成功治理最新善后压力记录涉及的州郡时，生成结构化 `SubmissionAftermathGovernanceRecord`，并在外交面板展示处置摘要。

## 范围

- 在 `DiplomacyState` 中新增善后处置记录类型、数组、最近记录 helper、追加方法和 Codable 兼容默认值。
- 在 `CommandExecutor.executeRegionGovernance` 成功执行既有州郡经营后判断是否命中最新善后压力记录。
- 若当前行动势力是善后接收方，且治理州郡属于最新善后压力影响州郡，则追加处置记录并写入一条外交战报。
- 在 `DiplomacyPanelView` 的“善后”区展示当前最新善后记录对应的最近处置记录；若无处置，显示空态。
- 更新发布前检查面板、README、核心流程文档、流程图、计划文档和更新日志。

## 非目标

- 不新增新的命令类型，不新增 AI 回合额外行动次数。
- 不绕过 `CommandValidator -> RuleEngine -> CommandExecutor`，不直接修改 `GameState`。
- 不清零、删除、降级或解决 `SubmissionAftermathRecord`。
- 不新增忠诚、叛乱、贡赋、俘虏、安置、治安长期状态机或额外资源变化。
- 不批量处理历史善后压力记录；当前只关联最新一条善后记录。

## 数据流

```text
Command.governRegion
  -> CommandValidator 校验当前行动势力、州郡控制、府库和经营上限
  -> RuleEngine
  -> CommandExecutor.executeRegionGovernance
  -> 扣除府库并更新 RegionNode 战略字段
  -> 如果命中最新 SubmissionAftermathRecord
  -> DiplomacyState.appendSubmissionAftermathGovernanceRecord
  -> EventLog(.diplomacy, relatedRecordId)
  -> DiplomacyPanelView 善后区展示处置摘要
```

## 架构边界

- Hex 仍是战术权威；本轮不修改 `HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- Region 仍是战略聚合层；处置记录只说明既有州郡经营已经发生。
- 归附善后压力仍是审计与提示层，不是忠诚/叛乱/安置系统。
- AI 善后治理仍沿用每回合最多一条 `Command.governRegion` 的既有节奏。
- 记录进入 `DiplomacyState`，用于存档、战报关联和外交面板复盘。

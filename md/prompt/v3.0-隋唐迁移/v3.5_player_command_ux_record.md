# v3.5 玩家军令、州郡、外交和战报体验记录

完成日期：2026-07-05

性质：玩家信息闭环与 UI 可读性最小迁移。仍不是完整州郡经营、外交执行器或发布级 UI。

## 目标

- 让玩家在现有地图和面板中更快理解当前可下达的军令、州郡价值、外交态势和本回合战报。
- 不新增绕过规则系统的 UI 操作；玩家操作仍提交 `Command` 或 `ZoneDirective`。
- 用现有 `EventLog`、`WarDirectiveRecord`、`AgentDecisionRecord`、`CourtDecisionRecord` 聚合战报摘要。

## 已完成

- `EventLogView` 从普通事件列表升级为“战报”面板：
  - 显示朝堂姿态、AI 意图、方面军令数量、执行成功数、拒绝数。
  - 从最近日志中聚合战斗、撤退、围困、粮道、前线、方面、州郡、外交等重点事件。
  - 日志分类标签改为中文：战斗、撤退、补员、围困、粮道、前线、方面、州郡、外交、事件。
- `RootGameView` 的 Log tab 传入 `lastAgentDecisionRecord`、`lastWarDirectiveRecords` 和 `latestCourtRecord`，并将信息 tabs 改为中文：军队、州郡、总管、战报、粮饷、外交、AI。
- `DiplomacyPanelView` 改为隋唐中文口径：
  - 面板标题、势力、盟从、关系、君主、朝堂、谋主、军令、使者等显示中文化。
  - 显示最新朝堂记录中的谋主、军令数和使者摘要。
- `DiplomaticStatus.displayName` 改为中文：盟友、协同讨伐、中立、停战、称臣、归附、敌对、交战。
- `RegionInspectorView` 增加州郡重要性摘要：
  - 粮仓要地、可供征发、前线承压、敌军可见、胜负要点。
  - 后方无紧急军情时明确显示“后方州郡，当前无紧急军情。”
- `CommandPanelView` 改为中文军令入口：军令、固守、准退、整军、结束回合，并给出中文可用/拒绝状态。
- `AppContainer` 的主要玩家交互日志改为中文，覆盖军令执行/拒绝、AI 结算、选择地块/州郡/军队、总管军令拒绝与提交、生产拒绝等玩家可见信息。

## 关键边界

- 本轮不新增内政、外交、归附、停战、借兵或征发执行器。
- UI 只读取现有状态并提交既有 `Command` / `ZoneDirective`。
- 战报摘要来自现有日志和记录，不伪造未发生的行动结果。
- 没有修改 `RuleEngine`、`WarCommandExecutor`、hex 占领、动态方面或部署权威。

## 遗留事项

- 玩家仍不能执行真正的州郡经营命令，征发、屯田、修道、治安和归附安抚需要后续 `Command` / validator / executor。
- 外交面板仍是只读态势展示，停战、同盟、称臣、借兵、招降还没有执行链路。
- 战报面板未做筛选、按回合折叠或更完整的战役复盘。
- 发布级 UI、美术、地图图标、粮道线、围城圈和计划箭头仍需 v3.6 继续收口。

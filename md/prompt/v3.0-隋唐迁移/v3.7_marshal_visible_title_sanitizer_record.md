# v3.7-preflight.82 行军总管可见称谓净化收口

完成日期：2026-07-06

## 目标

在朝堂/外交实际记录 id 展示净化之后，继续收口 UI 和角色展示层残留的“元帅 / Field Marshal / Guderian”旧口径，使玩家可见文本统一使用“行军总管 / 军议 / 历史总管”。

## 范围

- 只改展示层文案、展示净化词表和角色 `displayName`。
- 保留 `AgentRole.fieldMarshal` rawValue、`MarshalDirective` 内部 provider suffix、record id、Codable schema 和既有存档兼容。
- 不改 AI 决策、命令结构、规则、JSON 数据或存档格式。

## 实现

- `AgentPanelView`：
  - `MarshalDirective` 来源显示从“元帅军议”改为“军议”。
  - `Field Marshal` 展示净化从“元帅”改为“行军总管”。
- `EventLogView`、`DiplomacyPanelView`：
  - `Field Marshal` 展示净化从“元帅”改为“行军总管”。
  - `DiplomacyPanelView` 的裸 `Guderian` 展示从“古德里安”改为“历史总管”。
- `CommandPanelView`、`EventLogView`：
  - 补齐 `directive_*command_*`、`order_*` 和朝堂 agent 前缀记录的展示净化。
- `GameAgent`：
  - `AgentRole.fieldMarshal.displayName` 改为“行军总管”，不改 rawValue。

## 边界

- 不重命名 `fieldMarshal`、`MarshalDirective` 或任何历史 id。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> CommandValidator -> RuleEngine` 统一入口。
- 不把展示文本反向写入规则状态。
- 不运行本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。

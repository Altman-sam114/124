# v3.7-preflight.93 自动总管默认指挥风格收口记录

完成日期：2026-07-06

## 目标

在阶段与 legacy 总管展示口径收口之后，继续处理一个低风险 AI 配置残留：`ZoneCommanderAgent.defaultConfig(for:)` 仍用 `zone.faction == .germany` 判断自动方面总管的默认指挥风格。这会让隋唐主路径中唐、瓦岗、薛秦、刘武周等本应更主动的势力全部落到 `.balanced`，与 `AppContainer` 手动生成总管配置时的多势力映射不一致。

本轮只统一默认配置口径，不改变 directive schema、命令管线、规则执行、AI 阈值、战术选择函数、势力 rawValue 或存档格式。

## 改动范围

- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
  - `defaultConfig(for:)` 不再用 `.germany` 二元判断决定 `commandStyle`。
  - 新增私有 `defaultCommandStyle(for:)`，与 `AppContainer` 默认配置口径对齐：旧东路势力、唐、瓦岗、薛秦、刘武周默认 `.aggressive`；旧西路势力、洛阳隋、夏、东突厥默认 `.balanced`。
  - 保留 `ZoneCommanderAgentConfig.CommandStyle` rawValue、Codable、`.cautious` case 和既有战术阈值逻辑。
- `md/prompt/v3.0-隋唐迁移/codex-v3.0-隋末唐初aiagent历史策略迁移总提示词.md`
  - 将当前交接状态推进到 v3.7-preflight.93。
  - 明确 v3.7-preflight.93 之后后续 Agent 应继续从 v3.7+ 风险和 v3.8+ 队列切片，而不是重做历史路线。

## 明确不改

- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` 管线。
- 不改 `MarshalAgentConfig`、`DirectiveEnvelope`、`TheaterDirectiveEnvelope`、`WarDirectiveRecord` 或 `AgentDecisionRecord` schema。
- 不改 `Faction` case、rawValue、JSON 数据、DataLoader、胜负规则、补给规则或存档格式。
- 不把 `.cautious` 分配给任何默认势力；该 case 继续保留给数据配置或后续策略扩展。
- 不跑本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。

## 并发子 Agent 审计

本轮主 Agent 启动了两个只读子 Agent：

- UI/可见文本残留扫描：聚焦玩家可见旧题材词、工程词和 UI 文案风险。
- 规则/AI 合同扫描：聚焦展示文本驱动逻辑和 legacy 语义依赖。

主 Agent 在子 Agent 返回前先完成低冲突源码补丁与文档同步；本轮没有让子 Agent 写文件，也没有多写者同时修改同一 public API 或同一文件。

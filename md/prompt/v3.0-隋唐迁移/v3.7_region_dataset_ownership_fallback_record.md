# v3.7-preflight.98 RegionDataSet owner/controller 兜底收口

完成日期：2026-07-06

## 切片 ID

v3.7-preflight.98-region-dataset-ownership-fallback

## 来源

总提示词 §0.2 / §0.4 的 P0 数据 fallback：`RegionDataSet.toRegions()` 中 owner/controller 缺省时会静默落到旧 `.allies`，与注释中的中立语义和隋唐多势力迁移边界冲突。

## 目标

只处理 `RegionDataSet.toRegions()` 的 null / 缺省 owner-controller fallback。常规隋唐或自定义数据缺少 owner 时必须显式报错；旧战局 fallback 数据可通过明确 legacy 判定保留 `.allies` 兼容，不再让任意坏数据静默回到旧西路势力。

## 非目标

- 不新增 `Faction.neutral`。
- 不把 `RegionNode.owner/controller` 改成 optional。
- 不改 `RegionDataSet` JSON schema。
- 不处理 P2 场景语义 helper。
- 不处理 P3 MapEditor 非法 unit faction fallback。
- 不改 MapEditor unit 导入、胜负规则、命令管线、动态 theater/front/deploy 或 AI 决策。

## 核心更新

- `RegionNodeDefinition` 注释改为：`controller` 缺省回退 `owner`；`owner` 不得省略，除非明确旧战局数据走 legacy 兼容兜底。
- `RegionDataSet.toRegions()` 改为 `throws`，缺 owner 的非 legacy 数据会抛出 `DataLoaderError.validationFailed`，错误消息包含具体州郡 id。
- 旧战局数据通过 `scenarioId` / `displayName` 的明确 legacy 判定保留 `.allies` 兼容 fallback，用于不破坏现有 `ardennes_v02_regions.json`。
- `DataLoader.validate(_ regionData:)` 与 `DataLoader.apply(_:to:)` 改为传播 `toRegions()` 的数据错误。
- `DataLoader.loadInitialGameState()` 的最末 legacy 叠加 fallback 只有在旧 region 数据也能成功转换时才挂载战略层。

## 验收标准

- `RegionDataSet.toRegions()` 不再包含 `?? .allies` 的通用 fallback。
- 当前默认 `wude_618_regions.json` 因每个 region 已显式 owner/controller 而不需要 fallback。
- 旧 `ardennes_v02_regions.json` 缺 owner/controller 的兼容只通过明确 legacy 判定发生。
- 非 legacy / 自定义坏数据缺 owner 时会得到数据校验错误，而不是生成旧西路势力 region。
- 文档同步更新总提示词 §0.2 / §0.4、`update_log.md`、`md/flow/*`、`README.md` 和 `md/plan/plan.md`。

# v3.7-preflight.53 总管与将领档案防区展示名 raw id 收口记录

完成日期：2026-07-06

## 目标

继续推进隋末唐初迁移发布候选前置收口，处理自动总管配置和将领档案中仍可能把 `FrontZoneId.rawValue` 或 raw 防区名展示给玩家的问题。

本轮只改展示名 fallback，不改内部 id、`assignedZoneId`、front zone schema、AI 决策、命令结构、规则数值或执行语义。

## 改动范围

- `WWIIHexV0/Agents/MockAICommander.swift`
  - `defaultConfig(for:)` 的本地模拟总管 `name` 不再拼接 `zone.id.rawValue`。
  - 新增局部 `displayZoneName(_:)`，当防区名为空、含下划线、以 `theater/front` 开头或等于 raw id 时，回落为“某势力方面”。
- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
  - `TheaterCommanderPool.defaultConfig(for:)` 的自动总管 `name` 不再拼接 `zone.id.rawValue`。
  - 新增同口径 `displayZoneName(_:)`，只用于展示名。
- `WWIIHexV0/UI/GeneralProfileView.swift`
  - 将领档案“所属防区”不再直接 `Text(zone.name)`，改用展示 fallback。

## 边界

- 不改 `ZoneCommanderAgentConfig.id`。
- 不改 `assignedZoneId`。
- 不改 `FrontZone.name`、`FrontZoneId`、`TheaterId` 或任何 JSON / Codable schema。
- 不改 `ZoneDirective`、`WarCommandExecutor`、`RuleEngine` 或 AI 战术分类。
- 不处理 legacy JSON 中的坐标式地点名和旧履历文本；这些作为后续数据文案收口风险保留。

## 并发复查来源

本轮使用并发只读子 agent 辅助定位：

- Huygens 建议把自动总管展示名中的 raw 防区 id 作为首选小切片。
- Cicero 补充发现 `GeneralProfileView` 直接展示 `zone.name`，同属防区展示名 raw id 风险。

```sh
git diff --check
```

按 `md/test/test.md`，本轮不跑 Xcode build/test、XCTest、模拟器、UI test、性能测试或全量构建。

## 遗留风险

- 未启动 app 打开将领档案或 AI 面板做视觉确认。
- 同势力多个 fallback 防区在展示名上可能同名；内部 id 和 assigned zone 未变，调度不受影响。
- legacy fallback JSON 中仍有坐标式城邑/补给点名、草稿州郡名和旧剧本履历，需后续单独做数据文案收口。

# v3.7-preflight.28 MapEditor 默认隋唐资源桥记录

完成日期：2026-07-05

## 目标

把 MapEditor 的默认游戏资源桥从阿登 legacy JSON 对齐到当前主游戏默认 `wude_618` 隋唐剧本，避免编辑器“读取默认资源 / 覆盖保存”仍默认操作阿登文件。

## 实现范围

- `MapEditorGameResourceBridge` 默认读取和覆盖：
  - `WWIIHexV0/Data/wude_618_scenario.json`
  - `WWIIHexV0/Data/wude_618_regions.json`
- `MapEditorExporter` 新增 `MapEditorExportMetadata`，让默认覆盖保存保留现有场景元数据。
- MapEditor 默认新文档、模式标题、面板文案、单位模板、城池/关隘自动命名改为隋唐口径。
- 发布检查面板和文档状态更新到 `v3.7-preflight.28`。

## 关键边界

- 不改 `ScenarioDefinition` / `RegionDataSet` JSON schema。
- 不新增水战、渡河、港口补给或水路地点编辑规则。
- 不改变主游戏 `DataLoader` 加载顺序。
- 不改变 hex、region、theater、front、deploy 权威边界。
- 不绕过 `Command` / `ZoneDirective -> RuleEngine`。
- 不执行 Xcode build、XCTest、模拟器、app 启动或 MapEditor UI 烟测。

## 元数据保护

`MapEditorGameResourceBridge.overwriteDefaultGameResources(document:)` 会先读取现有 `wude_618_scenario.json`，再把以下字段传给 `MapEditorExporter`：

- `factions`
- `maxTurns`
- `initialTurn`
- `initialPhase`
- `playerFaction`
- `aiFaction`
- `keyLocations`
- `objectives`
- `victoryConditions`
- `dataNotes`

导出器会把编辑器文档生成的 objective / keyLocation 与既有元数据合并，避免覆盖保存时清掉：

- 唐、洛阳隋、瓦岗、夏、薛秦、刘武周、东突厥势力列表。
- `playerCommand` / `tang` / `luoyangSui` 起始回合配置。
- 长安、洛阳、洛口仓、潼关胜负条件。
- objective 原有点数。
- 蒲津渡、孟津渡、黎阳津、洛口津等水路地点记录。

## 后续风险

- 未授权运行 macOS MapEditor，未验证真实读取、覆盖保存、重新加载和 UI 点击流程。
- 水路地点仍只是从既有 `keyLocations` 元数据保留；编辑器还没有 ferry / port / harbor 的独立新增、移动、删除 UI。

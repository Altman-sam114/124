# v3.7-preflight.117 AI 粮草目标权重记录

完成日期：2026-07-11

## 切片定位

来源：人工最新方向要求古代战争突出粮草、粮道和粮仓节点；v3.7-preflight.115 已让粮道显示对齐规则 anchor，但 AI 目标选择仍未充分体现粮草价值。

目标：让本地 AI / 模拟元帅在选择进攻目标和生成理由时更重视高粮草、粮仓州郡。

非目标：

- 不改 `Command`、`ZoneDirective`、`TheaterDirective` 字段合同或执行语义。
- 不改补给规则、移动规则、战斗规则、水战、部署、JSON schema 或地图数据。
- 不实现真实截粮命令、多回合粮道封锁、粮道逐格路径、LLM 接入或运行时截图验收。

## 实现

- `ZoneCommanderAgent` 的攻击候选排序使用粮草权重，`supplyValue` 从一倍价值提高为三倍价值。
- 突破 / 骑袭 focus 的 `breakthroughRegionSortKey` 同步使用三倍粮草权重，避免焦点目标绕开粮仓。
- `MarshalBattlefieldSummary` schemaVersion 升到 6，`MarshalFrontSummary` 增加高粮草 / 粮仓目标摘要。
- `SimulatedMarshalLLMClient` 生成攻势 directive 时优先把粮草目标放入 `weightedRegions` 和 `focusRegionId`，并提高包含粮草目标的攻势优先级。
- 模拟元帅 rationale 和 strategicIntent 会说明“某地牵动粮仓粮道”。
- `AgentPromptBuilder` 的可见州郡 prompt 加入粮草值，高粮草或名称含仓的州郡标注为“粮仓要地”。
- `MockAIClient` 的攻击评分、部署攻击排序和行军目的地排序纳入粮草值；高粮草目标的 reason 会说明夺粮草、压迫补给或沿粮道逼近粮仓。

轻量检查按 `md/test/test.md` 执行；本机不跑 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。云端重验证由 push `origin/main` 后的 GitHub Actions 结果包承担。

遗留风险：

- 未在运行时观察 AI 是否实际选择洛口仓、黎阳仓等粮仓目标。
- 粮道仍不是逐格路径目标，也没有真实截粮 / 封锁命令。
- 高粮草权重数值仍需后续 playtest 和云端/人工运行时验证。

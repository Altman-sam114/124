# v3.7-preflight.37 App/UI 边界文案收口记录

完成日期：2026-07-05

## 1. 本轮目标

在 v3.7-preflight.36 收口规则、军令、战略同步和经济源头事件之后，继续处理 App 与 UI 边界仍可能露出的系统错误、provider、scenario raw id、发布检查工程词和详情面板 raw id。

## 2. 范围

- `AppContainer`：存档读取/删除/写入失败、自动回合完成、玩家命令结果、新局事件、no-op 自动回合记录。
- `ReleaseChecklistView`：发布检查面板的工程词、旧版本标题、剧本 ID、hex、Command / RuleEngine、Xcode / UI / build 等玩家可见文案。
- `MapDisplayAdapter`、`RegionInspectorView`、`UnitInspectorView`：州郡/军队详情面板的州郡、动态方面和行军防区名称映射。
- 文档：`README.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/plan/plan.md`。

## 3. 关键改动

- 存档读取、继续、删除和自动保存失败不再把 `localizedDescription` 写入 HUD、设置、发布检查或交互日志，改用中文泛化失败说明。
- 自动回合完成日志不再显示 `AI \(provider)`、`System` 或 `MarshalDirective` 等来源字符串，改为“自动回合完成，结算 N 条命令结果”。
- 命令面板的 `lastCommandMessage` 不再直接显示 `CommandResult.message`，成功/失败改为稳定中文摘要；交互日志保留中文命令摘要和中文校验原因。
- 新局交互日志使用剧本中文名，不再直接显示 `scenarioId`。
- 发布检查面板改为玩家语义：统一规则系统、自动回合、地块、当前局势快照、运行复核等；未知剧本 fallback 不再直出 raw id。
- 州郡和军队详情面板通过 `MapDisplayAdapter` 传入州郡名、方面名和防区名，不再显示相关 raw id；地图颜色派生保持 tile/hex 优先，再 fallback 到 region 聚合层。
- 子 agent 复核后继续把旧剧本 fallback 从“调试剧本”改为“旧版剧本”，发布检查未知剧本 fallback 与 App 入口统一为“当前剧本”。

## 4. 并发扫描

本轮使用两个只读子 agent：

- App/存档扫描定位 `AppContainer` 中 7 组直接玩家可见风险和 2 处内部 raw id 生成点；本轮处理直接可见风险，内部 id 保持不展示。
- 发布检查扫描定位 `ReleaseChecklistView` 中 38 个玩家可见候选，均适合低风险替换；本轮按建议收口。
- 末轮 App/UI 复核定位旧剧本 fallback 与地图显示优先级风险；本轮已修正。`AttackParameters.targetTheaterId` 仍按旧字段名兼容 `FrontZoneId` 执行桥，不在本轮改公共 directive schema。

第三个详情面板扫描因并发线程上限未启动，主线程直接阅读 `RegionInspectorView`、`UnitInspectorView` 和 `MapDisplayAdapter` 完成。

## 5. 架构边界

- 不改变 `GameSaveStore` 文件路径、存档 JSON schema 或错误类型。
- 不改变 `GameState`、`GameLogEntry`、`WarDirectiveRecord`、`PlayerPlannedOperation`、内部 production / directive id。
- 不改变命令校验、规则执行、AI 决策、自动回合步数、地图数据或发布验证结论。
- `MapDisplayAdapter` 只增加展示名字段，仍只读 `GameState` 派生 UI state。

## 6. 轻量检查

本轮轻量检查以最终交付为准。计划执行：

- 针对 `localizedDescription`、provider 原文、scenario raw id、详情面板 rawValue 和发布检查工程词的 `rg` 扫描。
- touched-file 尾随空白和冲突标记扫描。

## 7. 遗留风险

- 本轮未做真实运行 UI 验证，发布检查面板、设置存档反馈、命令面板和详情面板的布局、滚动、动态字体和 VoiceOver 仍未确认。
- 仍可能有其他未扫描面板或未来新增 message 把 raw id / 工程词带回玩家路径。
- 完整发布候选仍需要授权构建、启动、界面点击烟测、观察者多回合和云端结果包验收。

# v3.7-preflight.91 指令结果语义化固守判定收口记录

完成日期：2026-07-07

## 目标

在命令、战报和 AI 诊断多轮中文化之后，消除一处逻辑继续依赖旧英文展示文本的风险：`ZoneCommanderAgent.hasRecentStaticDefense` 原先用 `commandDisplayName?.hasPrefix("Hold")` 判断上一轮是否全员固守，但 `Command.displayName` 已中文化为“坚守”。本轮改为让命令结果携带独立语义字段，展示文本只负责展示。

## 改动范围

- `WWIIHexV0/Agents/AgentDecisionRecord.swift`
  - 新增 `CommandSummaryKind`，覆盖现有 `Command` case 的轻量语义分类。
  - `CommandResultSummary` 新增可选 `commandKind`。
  - AI order、directive command、system command 和 end turn 的结果构造都会写入 `commandKind`。
  - 为 `CommandResultSummary` 显式实现 Codable，旧记录缺少 `commandKind` 时按 nil 解码。
- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
  - `hasRecentStaticDefense` 改为判断 `summary.commandKind == .hold`。
  - 不再依赖旧英文 `Hold` 或当前中文“坚守”展示文本。

## 明确不改

- 不改 `Command` case、`Command.displayName`、`AgentOrderType`、`ZoneDirective` schema、`WarDirectiveRecord` schema、AI tactic 分类阈值或规则执行。
- 不新增 Swift 文件，不改 `WWIIHexV0.xcodeproj/project.pbxproj`。
- 不跑本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。

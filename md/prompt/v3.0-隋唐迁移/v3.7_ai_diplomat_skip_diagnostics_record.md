# v3.7-preflight.27 AI 使者/交接跳过诊断

## 目标

在 v3.7-preflight.12 已让 AI 使者主动外交、v3.7-preflight.18 已让 AI 接收方自动交接归附实体、v3.7-preflight.26 已补 AI 太守跳过诊断之后，继续补齐 AI 回合不静默失败的审计链路：当朝堂存在使者步骤但未生成外交命令，或存在已归附目标上下文但未生成交接命令时，把确定性原因写入 `AgentDecisionRecord.errors`。

## 范围

- `TurnManager.executeDiplomatCommands` 在 `diplomatCommand` 返回 nil 时调用 `diplomatSkipDiagnostics`。
- 新增 `hasDiplomatStep`，复用朝堂使者步骤判断，避免多处重复写 `courtRecord?.steps.contains`。
- `diplomatSkipDiagnostics` 只在朝堂存在使者步骤时输出原因。
- `TurnManager.executeSubmissionHandoffCommands` 在 `submissionHandoffCommand` 返回 nil 时调用 `submissionHandoffSkipDiagnostics`。
- `submissionHandoffSkipDiagnostics` 只在朝堂存在使者步骤且存在已归附目标上下文时输出原因，避免普通无归附目标回合产生额外交接噪音。
- 发布前检查面板和文档同步到 `v3.7-preflight.27`。

## 非目标

- 不新增外交状态，不新增命令类型，不增加 AI 回合行动次数。
- 不改变 `Command.updateDiplomacy` 或 `Command.resolveSubmissionHandoff` 的校验、消耗、执行效果或优先级评分。
- 不让 AI 使者或交接流程绕过 `CommandValidator`、`RuleEngine` 或 `CommandExecutor`。
- 不直接转移 hex、region、unit、theater、front 或 deploy。
- 不实现借兵、同盟、贡赋、忠诚、叛乱、俘虏、安置或长期外交谈判状态机。

## 数据流

```text
CourtDecisionRecord diplomat step
  -> TurnManager.diplomatCommand(...)
  -> 未生成 Command.updateDiplomacy
  -> diplomatSkipDiagnostics(...)
  -> AgentDecisionRecord.errors
  -> AgentPanelView 错误/诊断区展示
```

```text
CourtDecisionRecord diplomat step
  + DiplomacyState submittedTargetFactions()
  -> TurnManager.submissionHandoffCommand(...)
  -> 未生成 Command.resolveSubmissionHandoff
  -> submissionHandoffSkipDiagnostics(...)
  -> AgentDecisionRecord.errors
  -> AgentPanelView 错误/诊断区展示
```

诊断原因包括：

- 使者势力缺少国家档案。
- 当前没有带国家档案的敌对势力。
- 没有敌对势力达到归附条件。
- 未达到停战阈值，并记录战意、存活军队、受控可通行州郡和敌对势力数量。
- 已归附目标不属于当前接收方。
- 已归附目标没有残余军队或受控可通行 hex 可交接。

## 架构边界

- Hex 仍是战术权威；本轮不修改 `HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- 外交和归附交接仍只通过既有 `Command.updateDiplomacy` / `Command.resolveSubmissionHandoff` 进入 `RuleEngine`。
- 跳过诊断只说明为什么本回合没有生成 AI 使者或交接命令，不代表外交关系、残余实体或善后压力被规则层改变。
- 归附交接诊断只在已有归附目标上下文时输出，避免把正常无归附目标回合误写成异常。

## 风险与后续

- 没有真实 AI/观战多回合验证时，只能确认源码路径和轻量语法，不能确认运行中一定触发每一种诊断。
- 诊断只说明为什么本回合没有生成外交或归附交接命令，不证明外交启发式、归附关系或残余实体盘点完全正确。
- 后续若实现借兵、忠诚、叛乱、贡赋、俘虏或安置，必须新增清晰的命令、validator 和 executor，不能让跳过诊断承担规则效果。

# v3.7-preflight.36 战报源头事件文案收口记录

完成日期：2026-07-05

## 1. 本轮目标

在 v3.7-preflight.35 已收口战报展示层 formatter 之后，继续处理 `CommandExecutor`、`WarCommandExecutor`、`StrategicStateSynchronizer` 和 `EconomyRules` 写入 `GameState.eventLog` 的上游源头 message，减少玩家可见战报在进入 UI 前携带英文工程词、raw region/theater id、validation rawValue 或调试编号的概率。

## 2. 范围

- `CommandExecutor`：移动、战斗、反击、固守、准退、归附交接、回合推进、动态方面推进和 combatLog 文案。
- `WarCommandExecutor`：方面军令拒绝、州郡控制权变化、动态方面推进和前线变化文案。
- `StrategicStateSynchronizer`：州郡控制权同步事件文案。
- `EconomyRules`：府库初始化、生产部署事件和新征发军队展示名。
- 文档：`README.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/plan/plan.md`。

## 3. 关键改动

- 战斗日志从 `attacked / counterattacked / strength / automatic retreat / was destroyed` 改为进攻、反击、兵力、自动撤退和击溃等中文战报语义。
- 行军、固守、准退、归附交接和回合推进事件改为中文，不再显示 `HOLD`、`RETREATABLE`、`Turn advanced` 或英文 `hex`。
- 方面军令拒绝原因改用 `CommandValidationError.displayName`，不再把 validation `rawValue` 写入前线事件。
- 州郡控制权变化使用 `region.name`，动态方面推进使用方面名和州郡名，不再把 `regionId.rawValue` 或 `advancingTheaterId.rawValue` 写进玩家战报。
- 府库 bootstrap 事件中文化；生产部署事件使用州郡名；新生产军队展示名不再带 `createdTurn-index` 调试编号。内部 production id 仍保持原格式，避免影响存档兼容。

## 4. 架构边界

- 不改变 `Command`、`ZoneDirective`、`WarCommandExecutor`、`RuleEngine` 的执行链路。
- 不改变 combat、movement、supply、economy、diplomacy、theater、front 或 deployment 规则。
- 不改变 `GameState`、`EventLogEntry`、存档 JSON schema、生产订单 id 或地图数据。
- 不处理 UI 运行时布局、VoiceOver、滚动显示和真实多回合行为验证。

## 5. 并发扫描与遗留线索

本轮使用两个只读子 agent 并发扫描：

- Rules/Commands 扫描确认本轮四个源头文件存在英文战报、raw region/theater id、validation rawValue、经济 bootstrap 和生产部署调试文案风险，已纳入本轮处理。
- App/UI 扫描另发现存档错误 `localizedDescription`、AI provider、scenario id、发布检查面板、州郡/军队详情 raw id 等 26 个玩家可见风险；这些不属于本轮源头规则事件范围，建议作为下一刀处理。

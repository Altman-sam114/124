# v3.7-preflight.95 DataLoader 场景阶段兜底收口记录

完成日期：2026-07-06

## 目标

继续处理并发只读扫描指出的低风险迁移残留：`DataLoader.loadGameState` 在 `scenario.initialPhase` 无效时仍 fallback 到 `.germanAI`，而 `initialActiveFaction(for:)` 又用 `.alliedPlayer` 作为独立 fallback。这会让无效或自定义隋唐场景在加载时回到 legacy 阶段口径，且 `GameState.phase`、初始战报 phase 与 active faction 可能各自走不同 fallback。

本轮只收口 DataLoader 加载兜底，不改变合法 JSON 数据、`GamePhase` case/rawValue、旧阿登显式阶段、命令管线、规则执行、AI 决策或存档 schema。

## 改动范围

- `WWIIHexV0/Data/DataLoader.swift`
  - 新增 `initialPhase(for:)`，先读取合法 `scenario.initialPhase`；无效时 legacy 阿登场景 fallback `.alliedPlayer`，隋唐或未知自定义场景 fallback `.playerCommand`。
  - `loadGameState(scenario:regionData:)` 只解析一次 `initialPhase` 和 `initialActiveFaction`，并同时用于 `GameState` 与初始 `GameLogEntry`。
  - `initialActiveFaction(for:phase:)` 改为接收已解析 phase，不再内部另用 `.alliedPlayer` 作为 fallback。
  - 新增 `initialPlayerFaction(for:)`，无效 player faction 时 legacy 阿登 fallback `.allies`，隋唐或未知自定义场景 fallback `.tang`。

## 明确不改

- 不改 `GamePhase` enum case、rawValue、Codable 或显示名。
- 不改 `ScenarioDefinition` schema、JSON key、默认 `wude_618_scenario.json` 或 legacy `ardennes_v0_scenario.json`。
- 不把合法 `.germanAI` / `.alliedPlayer` 值强行规范为通用阶段；本轮只处理无效字段 fallback。
- 不改 `CommandValidator`、`CommandExecutor`、`TurnManager`、`RuleEngine` 或回合推进规则。
- 不改 `Faction` case、rawValue、DataLoader 默认资源顺序、胜负规则、补给规则或存档格式。
- 不跑本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。

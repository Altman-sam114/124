# v3.7-preflight.33 玩家可见外交/朝堂文案收口记录

完成日期：2026-07-05

## 目标

在第一批玩家可见调试文案收口之后，继续压低外交面板、朝堂决策摘要、州郡/军队详情和基础可访问性入口中的 raw id、英文 fallback 与工程术语暴露风险。

## 实现范围

- `DiplomacyPanelView` 的势力、盟从和关系显示不再直出 `blocId.rawValue` 或 country raw id，改用国家/盟从中文显示名。
- `DiplomacyPanelView` 的君主/谋主显示不再直出 `Agent` 标签和 agent id，改为角色或人物名 fallback；重点防区不再显示 front zone raw id。
- 外交动作说明去掉 `Command` / `RuleEngine` 工程词，改为玩家语义的规则校验说明。
- `DiplomacyState` 的外交摘要、legacy 盟从/国家名称、归附交接和善后摘要中的英文/hex 术语中文化。
- `RulerAgent` 写入 `RulerDecisionRecord.rationale` 和 directive context 的君主决策摘要中文化，不再把 ruler id 或 raw front zone id 写进摘要。
- `MapDisplayAdapter` 的州郡要地状态从 `None` / `controlled` 改为中文。
- `RegionInspectorView` 与 `UnitInspectorView` 收口少量玩家可见工程文案：州郡经营说明、操控状态、前线 raw id 和“动态方面”标签。
- `MapDisplayLayer.dynamicTheater` 显示为“当前方面”，`RootGameView` 地图 accessibility 不再朗读 `scenarioId` raw 值。
- `GeneralCommandPanelView` / `GeneralProfileView` 的未知技能 fallback 改为“未知军略”，将领命令面板头像 accessibility 从“画像”改为“头像”。
- `AgentPanelView` 标题改为“朝堂决策”，legacy `AgentOrderType` fallback 改为中文，并隐藏映射失败的英文细节。

## 关键边界

- 不改外交关系、命令结构、AI 决策策略、规则执行、存档字段、JSON schema、地图数据或运行时权威层。
- 不改 `Command.displayName` 的全局语义；命令显示名仍需后续独立收口。
- 不在本轮解决所有 region/theater/front zone raw id；需要传入名称映射或 display label 的位置留给后续切片。
- 不处理 `TurnManager` 深层诊断的全部英文/工程术语；该文件散点多，后续应独立做 AI 诊断文案收口。
- 不执行 Xcode build、XCTest、模拟器、app 启动或 UI 烟测。

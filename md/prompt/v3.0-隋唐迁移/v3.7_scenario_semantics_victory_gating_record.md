# v3.7-preflight.99 场景语义与胜负 fallback 门禁收口记录

## 切片

- 切片 ID：P2-a 场景语义 helper + P2-b 胜负 fallback gating。
- 来源：`codex-v3.0-隋末唐初aiagent历史策略迁移总提示词.md` §0.2 / §0.4。
- 当前基线：`main` 工作区已有 v3.7-preflight.98 相关未提交改动；本轮不回滚既有改动，只在其上小步收口。

## 目标

- 抽出共享 `ScenarioSemantics`，集中判断明确旧战局、明确 `wude_618`、隋唐草稿和未知自定义场景。
- 让 DataLoader、存档默认玩家势力、自动总管默认势力和玩家可选势力不再各自散落 `wude_618` / `ardennes` 判断。
- 让 `VictoryRules` / `RegionVictoryRules` 只有在明确旧战局时才走旧 Bastogne / St Vith fallback；未知自定义场景保持未决，不静默套用旧胜负规则。

## 非目标

- 不改 JSON schema。
- 不删除 `Faction.germany/allies`、`GamePhase.germanAI/alliedPlayer` 或旧阿登兼容数据。
- 不改 objective id、胜负阈值、回合顺序、AI 决策或规则数值。
- 不处理 MapEditor 非法 unit faction fallback；该问题已由并发子 Agent 只读审计确认，留作 P3 后续切片。

## 核心实现

- `ScenarioSemantics` 放入 `WWIIHexV0/Core/Faction.swift`，提供：
  - `family`：`.legacy`、`.wude618`、`.suitangDraft`、`.custom`。
  - `defaultInitialPhase`、`defaultPlayerFaction`、`defaultAIFaction`。
  - `resolvedPlayerFaction`、`resolvedAIFaction`、`resolvedActiveFaction`。
  - `preferredCommandFaction`。
- `DataLoader` 复用 `ScenarioSemantics` 解析无效 initial phase、player faction、AI faction、active faction、隋唐将领资源选择和 region owner legacy fallback 判定。
- `GameState` 旧存档缺 `playerFaction` 时通过 `ScenarioSemantics.defaultPlayerFaction(activeFaction:)` 推断。
- `AgentConfiguration.defaultCommandFaction` 改为使用 `ScenarioSemantics.preferredCommandFaction`，未知自定义场景优先玩家势力、隋唐势力或现有可用势力，不再先尝试旧 `.germany`。
- `AppContainer.playableFactions` / `playableFaction` 复用场景语义，只有明确旧战局才把 legacy combatants 放进优先段。
- `VictoryRules` 与 `RegionVictoryRules` 复用 `ScenarioSemantics`：
  - `.wude618` 走 `Wude618VictoryEvaluator`。
  - `.legacy` 走旧胜负 fallback。
  - `.suitangDraft` / `.custom` 保持 `.ongoing`。

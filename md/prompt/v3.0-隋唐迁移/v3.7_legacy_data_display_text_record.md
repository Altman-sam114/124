# v3.7-preflight.54 legacy fallback 数据展示文案收口记录

完成日期：2026-07-06

## 目标

继续推进隋末唐初迁移发布候选前置收口，处理 legacy fallback 数据中仍可能进入玩家视野的草稿名、坐标式地点名和旧剧本文案。

本轮只改 JSON 展示字段，不改 id、rawValue、schema、坐标、faction、unit template、objective kind、加载顺序、规则或命令管线。

## 改动范围

- `WWIIHexV0/Data/ardennes_v0_scenario.json`
  - 场景展示名从“地图编辑器旧剧本”改为“阿登预检剧本”。
  - dataNotes 去掉“地图编辑器生成”和“州郡”工程/迁移口径，改为“阿登预检场景数据”和“区域邻接...”。
  - keyLocations 的“城邑 q,r / 补给点 q,r”改为巴斯托涅、圣维特、马尔梅迪、迪南、纳缪尔、侯法利兹、拉罗什、圣于贝尔、色当、马尔什、纽沙托和补给站名称。
  - map.tiles 的 `cityName` 与对应 keyLocations 保持同名。
- `WWIIHexV0/Data/ardennes_v02_regions.json`
  - 数据集展示名改为“阿登地区州郡”。
  - region name 从“新省份N”改为方位州郡名。
  - city name 与场景地点名保持一致。
- `WWIIHexV0/Data/generals.json`
  - legacy 将领履历去掉“旧剧本总管 / 旧剧本装甲总管 / 旧剧本集团军总管”。
  - `general_bock.localizedName` 从“博客”修正为“博克”。

## 并发复查来源

本轮使用两个并发只读子 agent 辅助定位：

- Godel 扫描 `ardennes_v02_regions.json`，列出草稿州郡名和坐标式城邑名。
- Feynman 扫描 `ardennes_v0_scenario.json` 与 `generals.json`，列出 keyLocations、tile cityName、dataNotes、displayName 和履历残留。

## 边界

- 不改 JSON key、id、rawValue、coord、faction、templateId、objective kind 或 schemaVersion。
- 不改 `DataLoader` 加载顺序或 fallback 逻辑。
- 不改 `RegionInspectorView`、地图显示、胜负规则、补给规则或命令管线。
- `initialUnits[].name` 仍是“师 N”泛化名，后续可单独做单位展示名收口。

## 遗留风险

- 未启动 app 或 MapEditor 验证 legacy fallback 场景在 UI 中的实际展示效果。
- 阿登 fallback 仍保留二战势力 raw id、unit template id 和内部文件名，这是 legacy 兼容层的一部分，不在本轮展示文案范围内。
- `initialUnits[].name` 仍是“师 N”，后续可按阵营和兵种做更完整的用户可见单位名收口。

# v3.7-preflight.32 玩家可见调试文案收口记录

完成日期：2026-07-05

## 目标

在玩家可见旧英文兜底收口之后，继续压低朝堂、战报、将领面板中第一批 raw/debug 文案暴露风险，让默认隋唐体验少看到 `MockAI`、rawValue、英文 bootstrap 日志、无数据 JSON 占位和技能 id。

## 实现范围

- `SubmissionPresenceSummary.presenceText` 把“受控可通行 hex”改为“受控可通行地块”。
- `AppContainer` 的本地模拟 AI 来源从 `MockAI` 改为“本地模拟朝堂”，AI 空转摘要、临时总管名称、总管军令提交日志和州郡选择日志改为中文玩家语义。
- `StrategicStateBootstrapper` 的外交、方面、前线和行军防区初始化战报中文化。
- `DirectiveType` 新增中文 `displayName`。
- `AgentPanelView` 的默认决策者、来源、命令 fallback、全局/诊断 fallback 和指令类型显示中文化；没有真实 `rawJSON` 时不再显示英文 placeholder JSON。
- `AgentDecisionRecord.mappingFailed` 的摘要改为“军令转换失败”。
- `GeneralCommandPanelView` 和 `GeneralProfileView` 将当前将领技能 id 映射为中文特性名，并去掉画像 accessibility 里的“占位”字样。

## 关键边界

- 不改命令结构、AI 决策、规则执行、存档字段、JSON schema、地图数据或运行时权威层。
- 不隐藏完整朝堂面板；只收口第一批最明确的玩家可见 debug 文案。
- 不处理外交面板、州郡/军队详情、预备军令目标和 TurnManager 深层诊断中的全部 raw id；这些留给后续独立收口。
- 不执行 Xcode build、XCTest、模拟器、app 启动或 UI 烟测。

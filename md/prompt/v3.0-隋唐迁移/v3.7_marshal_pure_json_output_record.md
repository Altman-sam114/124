# v3.7-preflight.68 模拟元帅输出纯 JSON 收口

完成日期：2026-07-06

## 目标

在 legacy prompt 工程说明词收口之后，继续处理默认战争 AI 上游的模拟元帅输出格式。目标是让 `SimulatedMarshalLLMClient` 不再主动把 `TheaterDirectiveEnvelope` 包成 Markdown 代码围栏，改为直接返回纯 JSON 字符串，减少内置模拟路径继续制造排版标记和工程格式的机会。

## 实现范围

- `SimulatedMarshalLLMClient.completeTheaterDirectiveJSON` 编码 `TheaterDirectiveEnvelope` 后直接返回 UTF-8 JSON 字符串。
- 保留 `JSONEncoder` 的 `.prettyPrinted` 和 `.sortedKeys` 输出，保持模拟输出稳定可读。
- 保留 `TheaterDirectiveDecoder.extractJSON` 对带 json 标记的代码围栏、普通代码围栏和纯 JSON 的兼容。
- 同步 `README.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/flow/03_ai_zone_directive_pipeline.mermaid` 和 `md/plan/plan.md` 的当前进度口径。

## 边界

- 不改 `TheaterDirectiveEnvelope`、`TheaterDirective`、`TheaterDirectiveDecoder` 或 `TheaterDirectiveCompiler` 的 schema 合同。
- 不改 `schemaVersion`、`issuerId`、`turn`、`faction`、`directives`、`zoneId`、`targetRegionId`、`tactic` 或 `rationale` 字段。
- 不改 `MarshalAgent` 的策略选择、fallback、诊断流程、编译流程或 `rawTheaterJSON` 存档字段。
- 不改 `ZoneDirective`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或运行时 `GameState`。
- 不收窄外部模型输出合同；真实模型或历史记录仍可输出 fenced JSON，由 decoder 兼容处理。

## 并发扫描结论

- `ZoneCommanderAgent.swift` 中最高置信的 Markdown 包装来源是 `SimulatedMarshalLLMClient.completeTheaterDirectiveJSON` 的返回值。
- decoder 已经具备 fenced JSON 与纯 JSON 双兼容，调整模拟客户端不会破坏历史输入和外部模型兼容。
- 本轮不适合顺手改 UI / EventLog / Diplomacy 的记录净化词表；该方向可作为下一切片单独处理，避免和元帅 JSON 合同收口混在一起。

# v3.7-preflight.89 RegionVictoryRules 隋唐胜负摘要对齐记录

完成日期：2026-07-07

## 目标

在 legacy objective lookup 字面量收口之后，继续处理 region 分析层仍只返回 legacy 胜负摘要的问题。`RegionRuleSystem.analyze(_:)` 会把 `RegionVictoryRules.assessVictory(in:)` 放入 `RegionRuleAnalysis.victoryAssessment`，因此该摘要也需要按 `scenarioId` 区分默认隋唐剧本和旧 fallback 剧本。

本轮只让 region 层胜负摘要与主 `VictoryRules` 的 `wude_618_guanzhong_luoyang` 目标口径对齐，不改变真正写入 `GameState.victoryState` 的主胜负执行路径。

## 改动范围

- `WWIIHexV0/Rules/RegionVictoryRules.swift`
  - `assessVictory(in:)` 新增 `scenarioId == "wude_618_guanzhong_luoyang"` 分支。
  - `assessWude618Victory(in:)` 按 `obj_luoyang`、`obj_luokou`、`obj_tongguan`、`obj_changan` objective id 评估唐、洛阳隋和终局长安胜负摘要。
  - legacy fallback 规则下沉到 `assessLegacyFallbackVictory(in:)`，继续保留旧 objective id 和中性目标名兼容。
  - 终局检查和势力顺序 helper 与主 `VictoryRules` 保持一致。

## 明确不改

- 不改 `VictoryRules.updateVictoryState(in:)` 主胜负执行路径。
- 不改 `GameState.victoryState`、`VictoryState` 字段、`VictoryReason` case 或 Codable schema。
- 不改 objective id、地图数据、胜负阈值、回合推进、命令管线或存档格式。
- 不把 `VictoryConditionDefinition` 扩成通用解释器。
- 不移除 legacy fallback 胜负摘要。

## 轻量检查

- `swiftc -parse WWIIHexV0/Rules/RegionVictoryRules.swift`
- 聚焦扫描 `RegionVictoryRules.swift` 中旧战役可见地名字面量。
- 本轮改动文件尾随空白、冲突标记和 `git diff --check`。

## 遗留风险

- 本轮未运行 region 分析调用链、旧 fallback 剧本或默认隋唐剧本运行时流程。
- `RegionVictoryRules` 与 `VictoryRules` 仍各自维护一份 `wude_618` 胜负摘要逻辑；后续若胜负规则扩展，应抽共享只读 evaluator，避免分叉。

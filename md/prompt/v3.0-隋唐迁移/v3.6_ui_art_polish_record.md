# v3.6 发布级 UI、美术和交互收口记录

完成日期：2026-07-05

## 1. 本轮目标

本轮进入 v3.6 的第一步，不做大规模 SpriteKit 重绘、不引入外部素材、不改变规则和命令管线。目标是先把主游戏第一屏、常用信息面板和地图关键节点从开发调试观感推进到统一的隋唐历史策略 UI 基底。

范围：

- 建立共享隋唐视觉 token，统一面板底色、边线、圆角、阴影和最小触控目标。
- 将 HUD、图层选择、军情面板、战报、外交、州郡、军令、军需、朝堂、将领档案等常用 UI 套用统一视觉基底。
- 清理玩家可见的显眼英文调试文案：`NEW GAME`、`Observer`、`[ INFO ]`、地图图层 `Hex / Province / Initial / Dynamic / Front / Deploy`、将领档案英文标题和字段。
- 在 SpriteKit hex 节点上绘制最小城池、关隘和粮仓图形标识，去掉旧 `FORT` 与 `SUP A/G` 调试标签。
- 保持 UI 只读状态或调用既有 `AppContainer` 回调，不把规则写入 View。

非目标：

- 不新增真实美术资源、头像、图标 asset catalog 或版权不明素材。
- 不做大规模 SpriteKit 地图地形重绘或计划箭头。
- 不实现存档、新手引导、胜利规则、外交归附、太守内政或真实 LLM。
- 不运行 Xcode build/test、模拟器、Smoke、Probe 或性能测试。

## 2. 关键实现

### 2.1 视觉 token

`WWIIHexV0/UI/PlatformStyles.swift` 新增：

- `SuitangDesignTokens`
- `SuitangPanelProminence`
- `View.suitangPanel(_:)`

当前 token 以绢帛底、墨色文字、朱印、铜色、青绿和水色作为 UI 基础。面板圆角固定 8，保留 `minimumTapTarget = 44` 作为触控约束。

### 2.2 主界面 chrome

`RootGameView`：

- 顶部 HUD 去掉外层调试材质包装，交给 `HUDView` 自己的隋唐面板样式。
- 地图图层 Picker 改为“地图图层”，并使用 `suitangPanel(.chrome)`。
- `Observer` 改为“观战”。
- `[ INFO ]` 改为“军情 / 收起军情”，按钮触控高度不低于 44。
- 信息面板外框改用隋唐 panel background / 朱印边线。
- 军情 tab 中 `AI` 改为“朝堂”。
- fallback 文案 `No general selected.` 改为“未选择将领。”。

`InfoPanelToggle` 同步把 `[ INFO ]` 改为“军情 / 收起军情”。

### 2.3 HUD 与图层显示

`HUDView`：

- 标题使用 `Label` 与旗帜图标。
- HUD 面板使用 `suitangPanel(.elevated)`。
- 指标标签改用青绿色，数值改用墨色。
- `Ardennes V0` fallback 标题改为“调试剧本”，避免玩家路径出现二战剧本标题。

`MapDisplayLayer.displayName`：

- `Hex` -> `地块`
- `Province` -> `州郡`
- `Initial` -> `初始方面`
- `Dynamic` -> `动态方面`
- `Front` -> `前线`
- `Deploy` -> `部署`

### 2.4 常用面板

以下面板改用 `suitangPanel()` 或 `suitangPanel(.inset)`：

- `EventLogView`
- `DiplomacyPanelView`
- `CommandPanelView`
- `RegionInspectorView`
- `EconomyPanelView`
- `AgentPanelView`
- `GeneralCommandPanelView`
- `UnitInspectorView`
- `UnitTooltipView`

`NewGameButton` 改为“新局”，并确保触控高度不低于 44。

### 2.5 将领档案

`GeneralProfileView`：

- `General Profile` -> `将领档案`
- `Close` -> `关闭`
- `Biography` -> `生平`
- `Command Style` -> `用兵`
- `Assigned Zone` -> `所属防区`
- `HQ region contested` -> `本营州郡受压`
- `Relationship` -> `亲附`
- `Loyalty` -> `忠诚`
- `Satisfaction` -> `满意`
- `Player Interventions` -> `干预`
- `Skills` -> `特性`
- `Assigned Units` -> `所属军队`
- `Aggressive / Balanced / Cautious` -> `锐进 / 持重 / 谨慎`

### 2.6 SpriteKit 地图关键节点

`HexNode`：

- 城池 hex 或带 cityName 的代表 hex 会绘制最小城池图形标识。
- 关隘 / fortress hex 会绘制门楼和垛口图形标识。
- 补给源会按当前归属势力色绘制粮仓 / 仓储堆叠标识。
- 旧调试标签 `FORT`、`SUP A`、`SUP G` 已移除，避免地图上出现二战/调试缩写。
- 标识仍位于 hex 节点内，受现有 visibility / fog 逻辑覆盖，不在未可见区域额外泄露信息。

`BoardScene`：

- 空态标题从 `Hex Campaign Board` 改为“战役地图”。
- 新增最小粮道虚线：对当前玩家可见的己方军队，或观战模式下的可见军队，从最近可达粮仓 / 补给源画到军队所在 hex。
- 新增最小围城圈：对 `SupplyRules.isBesieged` 判定为真的可见军队，在其所在 hex 绘制红色双层围城圈。
- 粮道和围城均只读 `GameState`、`MapState.supplySources`、`SupplyRules` 和可见单位 placement，不修改任何规则状态。

## 3. 架构边界

- 本轮没有改 `Command`、`ZoneDirective`、`WarCommandExecutor`、`RuleEngine`、`GameState` 或任何战术权威状态。
- UI 操作仍通过 `AppContainer` 的既有回调进入命令系统。
- `SuitangDesignTokens` 只服务显示层，不承载规则。
- 新增样式没有引入第三方框架或外部素材。

## 5. 遗留事项

- SpriteKit 仍只是代码绘制的最小标识，不是正式美术资产；渡口/港口图标已由 v3.7-preflight.4 补上最小显示，仍待运行时截图验证。
- 粮道虚线和围城圈仍是直线/圆环的最小可视化，不是按真实路径节点逐段绘制的正式美术。
- AI 计划箭头和前线墨线已由 v3.7-preflight.5 / v3.7-preflight.6 补上最小显示，仍需授权运行时验证其与玩家计划、战区前线和战役复盘的可读性。
- 将领头像仍是文字占位，不是正式风格化印章或头像。
- 胜利规则、外交归附、州郡经营命令、存档/新手引导和真实模型接入仍是 v3.6 后续或 v3.7/v3.8 工作。
- 未运行本机 Xcode / XCTest / 模拟器 / app 启动，真实布局、动态字体、SF Symbol 可用性和运行时 UI 仍需云端或人工授权重测确认。

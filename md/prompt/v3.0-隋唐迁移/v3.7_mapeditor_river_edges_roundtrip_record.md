# v3.7-preflight.104 MapEditor 河边数据往返保真记录

完成日期：2026-07-07

## 目标

修复 MapEditor 默认资源读写链路中 tile `riverEdges` 被清空的问题。该问题会让“读取默认资源 -> 覆盖保存为游戏资源”丢失河边数据，进而影响运行时河流移动、战斗和补给规则的输入。

## 实现

- `MapEditorHex` 新增 `riverEdges: Set<HexDirection>`。
- `MapEditorHex` 使用自定义 `Codable`，旧 `.mapeditor` 文档缺少 `riverEdges` 时按空集合读取。
- `MapEditorGameResourceBridge.makeDocument` 导入 scenario tile 时，把 `tile.riverEdges` 转为 `Set<HexDirection>` 写入编辑器文档。
- `MapEditorExporter.makeScenarioDefinition` 导出 tile 时，按 `HexDirection.ordered` 稳定输出 `riverEdges` rawValue。
- `MapEditorOutputTests` 的 probe 文档增加非空河边断言，保护导出和 DataLoader 读回路径。

## 非目标

- 不新增河流绘制 UI。
- 不新增河流编辑控件。
- 不自动镜像相邻 hex 的反向 river edge。
- 不修改 JSON schema。
- 不修改运行时移动、战斗、补给或河流判定规则。
- 不补录默认隋唐地图的真实河边资产。

# v3.7-preflight.29 MapEditor 地点字段化编辑记录

完成日期：2026-07-05

## 目标

把 scenario `keyLocations` 从 MapEditor 默认导出的隐藏 metadata 保护，推进为可读取、可编辑、可导出的编辑器文档字段，补上 v3.0 总提示词中城池、关隘、渡口、粮仓、港口字段的最小编辑闭环。

## 实现范围

- `MapEditorDocument` 新增 `MapEditorKeyLocationDraft`、`keyLocations`、`keyLocationsAreAuthoritative` 和坐标级地点抑制记录。
- 旧 MapEditor 文档缺地点字段时仍可解码，导出时继续使用现有 scenario metadata 兜底。
- `MapEditorGameResourceBridge.loadDefaultDocument()` 从 `wude_618_scenario.json` 还原既有城池、关隘、粮仓、渡口和港口地点。
- 右键信息面板可编辑选中 hex 的地点名称、类型、势力和 objectiveId，并可保存或删除独立地点记录。
- MapEditor 画布在地块模式显示最小地点徽标。
- `MapEditorExporter` 导出时以文档 `keyLocations` 优先，再合并旧文档 metadata 兜底和 hex 派生地点；导出会过滤当前文档不存在或已抑制的地点坐标。
- `MapEditorViewModel.export()` 对 `wude_618` 文档传入默认场景 metadata，让“导出 JSON 到内存”和覆盖默认资源保持同一场景元数据口径。
- 发布检查面板更新为 `天命开唐 Agent · v3.7-preflight.29`。

## 关键边界

- 不改 `ScenarioDefinition` / `RegionDataSet` JSON schema。
- 不新增水战、渡河、港口补给、水师规则、移动规则、战斗规则或胜负规则。
- 不改变主游戏 `DataLoader` 加载顺序。
- 不改变 hex、region、theater、front、deploy 权威边界。
- 不绕过 `Command` / `ZoneDirective -> RuleEngine`。
- 不执行 Xcode build、XCTest、模拟器、app 启动或 MapEditor UI 烟测。

## 导出优先级

`MapEditorExporter` 当前地点合并顺序：

1. `document.keyLocations`：编辑器显式字段优先。
2. `metadata.keyLocations`：仅当 `keyLocationsAreAuthoritative == false` 时兜底；旧文档缺字段会解码为 false。
3. 从 city / fortress / supply hex 派生的地点：补齐旧文档或未显式设置的城池、关隘、粮仓。

合并时仍按 id、objectiveId 和 coord 去重，避免默认 `wude_618` 覆盖保存生成重复地点。

删除派生地点时，`MapEditorDocument.suppressedKeyLocationCoordKeys` 会记录该坐标，避免 city / fortress / supply 语义在下一次导出中把地点补回。保存新的地点记录会清除该坐标的抑制状态。

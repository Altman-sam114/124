# v3.7-preflight.55 legacy fallback 单位与防区展示文案收口记录

完成日期：2026-07-06

## 目标

继续推进隋末唐初迁移发布候选前置收口，处理上一轮复扫后 legacy fallback 数据中仍可能进入玩家视野的泛化单位名、阿登场景“预检”口径和阿登 region “州郡”口径。

本轮只改 JSON 展示字段，不改 id、rawValue、schema、坐标、faction、templateId、objective kind、加载顺序、规则或命令管线。

## 改动范围

- `WWIIHexV0/Data/ardennes_v0_scenario.json`
  - 场景展示名从“阿登预检剧本”改为“阿登战局”。
  - dataNotes 从“阿登预检场景数据”改为“阿登战局场景数据”。
  - `initialUnits[].name` 不再使用“师 N”，改为按阵营和兵种区分的德军/盟军步兵、炮兵、装甲、摩托化展示名。
- `WWIIHexV0/Data/ardennes_v02_regions.json`
  - 数据集展示名从“阿登地区州郡”改为“阿登战区分区”。
  - `regions[].name` 从“方位州郡”改为“防区”口径，其中西北分区使用“巴斯托涅防区”。

## 并发复查来源

本轮使用两个并发只读子 agent 辅助定位：

- Bohr 扫描 `ardennes_v0_scenario.json` 的 41 个 `initialUnits[].name`，确认“师 N”为剩余单位展示名风险。
- Zeno 复扫 legacy fallback JSON 展示字段，指出“阿登预检”和阿登 region “州郡”仍偏工程或隋唐口径；`generals.json` 与 `unit_templates.json` 未发现额外明显展示文案残留。

## 边界

- 不改 JSON key、id、rawValue、coord、faction、templateId、objective kind 或 schemaVersion。
- 不改 `DataLoader` 加载顺序或 fallback 逻辑。
- 不改地图显示、胜负规则、补给规则、AI 决策或命令管线。
- 单位名采用阵营和兵种泛化展示，不宣称还原完整历史番号。

## 遗留风险

- 未启动 app 或 MapEditor 验证 legacy fallback 场景在 UI 中的实际展示效果。
- 阿登 fallback 仍保留二战势力 raw id、unit template id 和内部文件名，这是 legacy 兼容层的一部分，不在本轮展示文案范围内。
- 本轮未尝试把 legacy 单位名替换为完整历史番号，避免引入未经运行时验证的数据语义变化。

# v3.7-preflight.96 legacy phase 存档规范化收口记录

完成日期：2026-07-06

## 目标

继续处理 v3.7-preflight.95 后留下的低风险迁移残留：旧 `.germanAI` / `.alliedPlayer` phase rawValue 仍在命令校验、自动回合判断和回合推进中分散绑定 `.germany` / `.allies`。这对合法 legacy 阿登存档是兼容路径，但对隋唐或未知自定义坏存档会把行动阶段语义带回旧双势力判断。

本轮只集中 phase 规范化口径，不改变 `GamePhase` case、rawValue、Codable、存档字段、命令 schema、规则数值、AI 决策或合法 legacy 阿登双势力推进。

## 改动范围

- `WWIIHexV0/Core/GamePhase.swift`
  - 新增 `normalized(forActiveFaction:playerFaction:)`。
  - 新增 `allowsCommandExecution(forActiveFaction:playerFaction:)`。
  - 合法 `.germanAI` 仅在当前行动势力是旧东路且不是玩家执掌势力时保留；合法 `.alliedPlayer` 仅在当前行动势力是旧西路且也是玩家执掌势力时保留；其他旧 phase 会按 active/player 关系落到 `.playerCommand` 或 `.aiCommand`。
- `WWIIHexV0/Core/GameState.swift`
  - 解码存档时先确定 `playerFaction`，再规范化 `phase`。
- `WWIIHexV0/App/AppContainer.swift`
  - 初始化、bootstrap、继续存档、新局、重置和切换执掌势力后统一规范化 phase。
  - 玩家可操作守卫、总管军令入口、玩家命令 bookkeeping 和自动回合判断读取规范化 phase。
- `WWIIHexV0/Rules/CommandValidator.swift`
  - `phaseAllowsCommands(in:)` 复用 `GamePhase` helper，不再本地维护 legacy faction 绑定。
- `WWIIHexV0/Rules/CommandExecutor.swift`
  - `advanceActiveFaction(in:)` 先规范化当前 phase；合法 legacy 阿登阶段仍走原双势力推进，否则进入通用 turn order。
- `WWIIHexV0/Turn/TurnManager.swift`
  - `isAITurn(faction:state:)` 改为按 active faction 和规范化 phase 判断自动回合是否可执行，保留观察模式下 AI 代走玩家阶段的可执行语义。

## 明确不改

- 不改 `GamePhase` enum case、rawValue、displayName 或 Codable 合同。
- 不改 `Faction` case、rawValue、`Faction.legacyCombatants` 或 `Faction.suitangTurnOrder`。
- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` schema 与规则数值。
- 不改合法 `wude_618_scenario.json`、legacy `ardennes_v0_scenario.json` 或存档 JSON key。
- 不删除 `TurnManager.runGermanAITurn` 兼容 API。
- 不跑本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full。

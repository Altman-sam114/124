# v3.7-preflight.9 外交议和与纳降命令闭环记录

完成日期：2026-07-05

## 1. 本轮定位

本轮继续补 v3.7 发布候选前置的“外交或归附事件”缺口，目标是让玩家侧外交动作从只读面板进入 `Command -> CommandValidator -> RuleEngine -> CommandExecutor -> DiplomacyState` 统一命令管线。

本轮只做最小议和 / 纳降关系更新，不改变地图控制权、军队归属、州郡控制、动态战区、部署、防区、胜负条件或存档 schema。

## 2. 实现摘要

### 2.1 Command 管线

`Command` 新增：

```text
updateDiplomacy(issuer: Faction, target: Faction, status: DiplomaticStatus)
```

该命令无 acting division，不参与移动、攻击或玩家计划锁定。

### 2.2 校验与执行

`CommandValidator` 新增外交校验：

- 当前 phase 必须允许命令。
- issuer 必须等于 `GameState.activeFaction`。
- issuer 和 target 不能相同。
- 双方必须在 `DiplomacyState.countries` 中存在。

`CommandExecutor` 新增外交执行：

- 调用 `DiplomacyState.updateRelation(issuer:target:status:turn:)` 更新双方国家关系。
- 写入 `GameLogCategory.diplomacy` 战报事件。
- `submitted` 显示为“目标归附 issuer”；`truce` 显示为双方议定停战。

### 2.3 DiplomacyState

`DiplomacyState` 新增关系更新 helper：

- 对 issuer / target 下所有 country pair 更新或补建 `DiplomaticRelation`。
- 按 status 设置 tension。
- 更新 `lastUpdatedTurn`。
- 保持 relation 排序稳定。

该 helper 不改变 `CountryProfile.faction`，也不改 bloc、hex、region、unit 或 theater。

### 2.4 玩家入口

`AppContainer` 新增：

- `playerDiplomacyTarget`
- `canIssuePlayerDiplomacy`
- `proposeTruceToDiplomacyTarget()`
- `acceptSubmissionFromDiplomacyTarget()`

目标选择当前玩家势力的第一个敌对势力；按钮只在玩家势力行动、非观察模式、存在敌对对象时可用。

`DiplomacyPanelView` 新增“使者”动作区：

- “议和”提交 `.updateDiplomacy(..., status: .truce)`。
- “纳降”提交 `.updateDiplomacy(..., status: .submitted)`。
- 面板明确说明外交动作经 `Command / RuleEngine` 更新关系，不直接改占领或军队。

## 3. 架构边界

本轮数据流是：

```text
DiplomacyPanelView button
  -> AppContainer.submit(Command.updateDiplomacy)
  -> RuleEngine
  -> CommandValidator
  -> CommandExecutor
  -> DiplomacyState.updateRelation
  -> GameLogCategory.diplomacy
```

它不写回：

- `HexTile.controller`
- `Division.coord`
- `RegionNode.controller`
- `hexToTheater`
- `hexToFrontZone`
- `WarDeploymentState`
- `VictoryRules`
- 本地存档 JSON schema

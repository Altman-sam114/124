# v3.7-preflight.13 归附事件记录链

完成日期：2026-07-05

## 目标

补上外交关系变化的最小结构化事件记录链。玩家或 AI 通过 `Command.updateDiplomacy` 执行议和、停战、称臣或归附后，系统不仅更新 `DiplomacyState.relations`，还要留下可回放、可关联战报的 `DiplomacyEventRecord`。

## 范围

- `DiplomacyState` 新增 `DiplomacyEventRecord` 和 `diplomacyEventRecords`。
- `CommandExecutor.executeDiplomacyUpdate` 在更新关系后追加事件记录，并把 `.diplomacy` 战报的 `relatedRecordId` 指向记录 id。
- `DiplomacyPanelView` 显示最近外交事件、回合和边界说明。
- `ReleaseChecklistView`、README、flow、flowchart、plan、update_log 同步到 v3.7-preflight.13。

## 非目标

- 不转移 hex controller。
- 不转移 region、军队、动态方面、前线或部署归属。
- 不实现完整归附后续事件、借兵、贡赋、天命/民心连锁或地图交接。
- 不新增本机 Xcode / XCTest / 模拟器验证。

## 架构边界

外交命令仍是：

```text
Command.updateDiplomacy
  -> CommandValidator
  -> RuleEngine
  -> CommandExecutor
  -> DiplomacyState.updateRelation
  -> DiplomacyState.appendDiplomacyEventRecord
  -> GameState.appendEvent(category: .diplomacy, relatedRecordId: record.id)
```

`DiplomacyEventRecord` 是外交审计记录，不是战术执行器。归附当前只表示关系状态和事件记录；后续如果做地图或军队交接，必须另走结构化命令或独立 validator / executor，不能绕过 hex 权威。

## 验收标准

- 外交命令执行后会追加 `DiplomacyEventRecord`。
- 外交战报能通过 `relatedRecordId` 关联记录。
- 外交面板能显示最近事件和边界说明。
- 旧存档缺 `diplomacyEventRecords` 时能解码为空数组。
- 文档一致说明：归附事件记录已接入，地图/军队归属交接仍未实现。

## 遗留风险

- 尚未做真实 UI 点击、AI 多回合、存档读写或 Xcode build 验证。
- 事件记录 id 使用 turn、追加序号、issuer、target 和 status 组合；后续若需要跨存档合并或网络同步，可再引入更强事件 id 策略。

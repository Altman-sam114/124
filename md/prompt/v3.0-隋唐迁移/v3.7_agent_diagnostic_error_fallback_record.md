# v3.7-preflight.47 Agent 诊断与错误兜底文案收口记录

完成日期：2026-07-06

## 目标

继续推进完整 v3.7 发布候选前置收口，处理 Agent / 战报面板、legacy Agent D 映射失败、数据加载校验失败中可能进入玩家视野的英文角色名、工程词和异常兜底。

本轮使用两个并发子 Agent 做只读扫描：

- 子 Agent Kierkegaard：复核 legacy agent role / fallback / prompt / record 边界，建议只改展示名与展示净化，不改 prompt/schema/raw/id。
- 子 Agent Leibniz：复核 UI/App/Agents/Data 中剩余低风险英文，建议以诊断/异常兜底文案为 `.47` 主题。

## 改动范围

- `AgentRole.displayName` 改为中文：君主、元帅、行军总管。
- `AgentPanelView` 与 `EventLogView` 的诊断净化表补齐 legacy 角色、古德里安、schema/model/fallback/directive/diagnostic/breakthrough 等兜底替换。
- `CommandResultSummary.mappingFailed` 不再直接保存未包装的 `error.localizedDescription`，优先使用 `LocalizedError.errorDescription`，否则给出中文通用兜底。
- `AgentCommandMapper` 的 region mapping 失败路径统一使用中文映射详情，避免任意底层错误文案直透。
- `DataLoaderError` 与 `DataLoader` 的常见数据加载/校验失败描述改为中文，包括缺少资源、重复地块、未知地形、未知目标、模板/军队/补给源/代理配置错误。
- `AgentDecisionParser` 的 UTF-8 解析细节改为中文。

## 明确非目标

- 不改 JSON schema、JSON key、id、enum rawValue、record id 或资源文件名。
- 不改 `AgentPromptBuilder`、`general_agents.json.personalityPrompt`、LocalLLM prompt 合同或 mock/legacy 解析 schema。
- 不改 AI 决策策略、命令结构、规则数值、`Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 管线。
- 不把内部调试 prompt、schema 示例、record id 或历史测试名当作玩家可见文案强行中文化。
- 不跑 Xcode build、XCTest、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full、性能测试或 app 启动。

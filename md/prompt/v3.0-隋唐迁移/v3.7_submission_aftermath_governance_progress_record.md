# v3.7-preflight.22 善后处置进度摘要

## 目标

在 v3.7-preflight.21 的 `SubmissionAftermathGovernanceRecord` 基础上，补上最小复盘摘要：外交面板能够显示最新善后压力记录下已经处置了多少个受影响州郡，便于玩家判断善后是否被既有州郡经营命令覆盖。

## 范围

- 在 `DiplomacyState` 中新增按 `linkedAftermathRecordId` 筛选善后处置记录的只读 helper。
- 在 `DiplomacyState` 中新增按受影响州郡去重统计已处置数量的只读 helper。
- 在 `DiplomacyPanelView` 的“善后”区新增“处置进度”，显示 `已处置/受影响` 州郡数量。
- 保留最近处置记录摘要和边界说明。
- 更新发布前检查面板、README、核心流程文档、流程图、计划文档和更新日志。

## 非目标

- 不新增命令类型，不增加 AI 回合行动次数。
- 不改变 `Command.governRegion` 的校验、消耗或效果。
- 不清零、降级、删除或解决 `SubmissionAftermathRecord`。
- 不新增忠诚、叛乱、贡赋、俘虏、安置、治安长期状态机或额外资源变化。
- 不做历史善后记录筛选、分页或完整时间线。

## 数据流

```text
SubmissionAftermathRecord
  -> linked SubmissionAftermathGovernanceRecord 列表
  -> DiplomacyState 按 linkedAftermathRecordId 筛选
  -> DiplomacyState 按 affectedRegionIds 去重统计已处置州郡
  -> DiplomacyPanelView 显示处置进度
```

## 架构边界

- Hex 仍是战术权威；本轮不修改 `HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- Region 仍是战略聚合层；进度摘要只说明哪些受影响州郡已有经营审计记录。
- 善后压力仍是审计和提示层；进度摘要不代表压力被规则层解决。
- AI 善后治理仍沿用既有每回合最多一条 `Command.governRegion` 的节奏。

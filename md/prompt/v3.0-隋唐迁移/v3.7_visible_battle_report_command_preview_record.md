# v3.7-preflight.35 战报与总管预览文案收口记录

完成日期：2026-07-05

## 目标

在玩家可见 AI 诊断文案收口之后，继续处理上一轮遗留的战报 metadata、战报摘要和总管预备军令预览中的 raw id、英文工程词和调试格式。

## 实现范围

- `EventLogView` 的战报 metadata 不再展示 `relatedRecordId` 原始字符串，改为“外交审计 / 局势审计 / 审计记录”等玩家语义标签。
- `EventLogView` 的“AI 意图”标签改为“军议意图”，并对 legacy `move / attack / hold / resupply` 和 `v0.* / directive / JSON / agent / mock / command / RuleEngine` 等调试意图做中文或泛化展示。
- `EventLogView` 普通战报正文和“本回合重点”复用展示层 formatter，对常见审计 id 前缀做整段泛化，避免 `war_directive_*`、`player_directive_*`、`submission_*`、`diplomacy_event_*` 继续出现在玩家战报里。
- `GeneralCommandPanelView` 的预备军令摘要不再拼接 `targetRegionId.rawValue`、`sourceRegionId.rawValue` 或 `zoneId.rawValue`，改用当前目标州郡名、目标州郡、本防区州郡或防区名。
- `GeneralCommandPanelView` 的预备军令格式从工程式斜线分隔改为中文冒号摘要；头像 badge 在缺少本地化名称时使用中文占位，不再 fallback 到 raw `general.name`。

## 关键边界

- 不改 `GameLogEntry.relatedRecordId`、`PlayerPlannedOperation`、`WarDirectiveRecord`、Codable schema、命令结构、AI 决策、规则执行、存档字段、JSON 数据或地图权威层。
- 战报原始 message 仍来自上游事件；本轮只做展示层的工程词和常见审计 id 兜底收口，不建立完整本地化资源表。
- `GeneralCommandPanelView` 没有完整 region / zone 名称表，因此非当前选中目标只显示泛化名称，不猜测 raw id 对应的州郡名。
- 不执行 Xcode build、XCTest、模拟器、app 启动或 UI 烟测。

## 后续风险

- 战报正文仍可能从更早或更深层上游 message 带入未覆盖的 raw id 模式，需要后续按实际命中继续补 formatter 或上游源头。
- 真实战报滚动、动态字体、VoiceOver 读法和总管面板预备军令实际布局未经过 app 启动验证。
- 完整发布候选仍需要授权 Xcode build、iOS/macOS 启动、UI 点击烟测和观察者多回合验证。

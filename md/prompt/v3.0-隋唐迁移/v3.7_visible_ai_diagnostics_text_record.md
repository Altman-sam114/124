# v3.7-preflight.34 玩家可见 AI 诊断文案收口记录

完成日期：2026-07-05

## 目标

在外交/朝堂文案收口之后，继续清理 AI 决策面板、方面军令诊断、legacy Agent D 映射错误和命令结果摘要里可能露出的英文、raw id、工程术语和 enum rawValue。

## 实现范围

- `TurnManager` 的 AI 回合失败、空军令、方面军令空结果、命令拒绝、结束回合失败、太守/使者/归附交接跳过诊断和部署诊断改为中文玩家语义。
- `TurnManager.contextSummary` 不再显示 agent id 或英文 divisions/objectives，改为第几回合、己方军队、已知敌军和可见目标数量。
- `TurnManager.canonicalCourtDirectiveJSON` 的附加标题改为中文，并不再把朝堂步骤中的 agent id 写入审计标题列表。
- `CommandValidationError` 新增中文 `displayName`，`RuleEngine` 和 `AgentDecisionRecord` 统一使用中文校验错误摘要，避免 `wrongPhase`、`noPath` 等 raw case 进入面板。
- `Command.displayName` 从 `Move(...)`、`Attack(...)`、`QueueProduction(...)` 等工程格式改为行军、进攻、坚守、补给、征发、州郡经营、外交、归附交接和结束回合的中文摘要。
- `CommandIntentAdapterError`、`AgentDecisionParserError` 和 `AgentCommandMappingError` 的可见错误描述中文化，覆盖 legacy Agent D 解析和映射失败路径。
- `AgentPanelView` 的决策者、来源、君主、朝堂步骤、重点方面、目标州郡和方面指令摘要不再直出 raw agent id、provider suffix、front zone id 或 region id；raw JSON 改为折叠的“审计原文”。
- `AgentPanelView` 对残余诊断文本做展示层兜底转译，避免 `ZoneDirective`、`WarDeploymentState`、`FrontZone`、`Division`、`Command`、`RuleEngine` 等工程词直接出现在普通面板。

## 关键边界

- 不改 `Command` case、Codable schema、AI 决策策略、命令校验顺序、规则执行、存档字段、JSON 数据或地图权威层。
- `AgentDecisionRecord.id`、`agentId`、`provider` 和 `rawJSON` 仍作为内部审计/回放数据保留；本轮只改普通 UI 展示和可见错误描述。
- raw JSON 仍可在面板中通过折叠项查看，供审计使用；默认不直接展开。
- 本轮不处理战报 metadata、总管预备军令目标、其他面板深层 raw id 或完整本地化资源表。
- 不执行 Xcode build、XCTest、模拟器、app 启动或 UI 烟测。

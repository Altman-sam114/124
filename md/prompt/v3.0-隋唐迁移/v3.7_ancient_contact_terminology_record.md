# v3.7-preflight.113 玩家可见接触态势术语古代化记录

切片 ID：v3.7-preflight.113

来源：人工最新方向：“古代作战不存在战线，骑兵强大，补给粮草很重要，作战模式需要多研究、修改，可参考三国志14之类的，把这个写进 readme/agents。”本轮先处理主游戏玩家可见和 AI 自由文本中的现代连续战线口径。

当前基线：

- 底层仍有 `FrontLine` / `FrontZone` / `frontLine` 等兼容类型名和 rawValue。
- v3.7-preflight.6 已有普通地图层接触线显示，但历史文档和部分 UI 文案仍称“前线/战线”。
- README 和 AGENTS 已写入古代作战原则，但源码玩家可见路径仍需首轮复扫。

## 目标

- 玩家可见文案优先使用“接触态势、边境压力、接敌、先阵、接触地段”等古代作战口径。
- AI rationale、MockAI stance、战报和 UI 标签不再把体验引回现代连续战线。
- README / AGENTS / 总提示词明确后续规则方向：天下局势、行军道、关隘、粮道、骑兵和隋唐英雄人物。

## 非目标

- 不重命名 `FrontLine` / `FrontZone` 类型、文件、rawValue 或 schema。
- 不改动态战区、接触线派生、部署池、AI 调度、命令执行或存档字段。
- 不清理历史 `update_log.md` 中的旧版本记录，避免破坏项目追溯。

## 实现摘要

- `MapDisplayLayer.frontLine.displayName` 改为“接触态势”。
- 发布检查、州郡、军队、外交、将领、战报和 App 交互提示继续转为“接触态势 / 边境压力 / 接敌 / 先阵 / 接触地段”。
- `StrategicStateBootstrapper`、`WarCommandExecutor`、`MockAIClient`、`ZoneCommanderAgent` 和 `RulerAgent` 的玩家可见自由文本同步改口径。
- README / AGENTS / `md/prompt/...总提示词.md` 增加或维护古代作战原则：古代无现代连续战线、骑兵强机动与冲击、粮草粮道重要、可参考三国志14体验但不得照搬。

## 并发子 Agent 发现

- 文案定位子 Agent 建议保留底层 `FrontLine` / `FrontZone` 作为兼容名，展示层统一适配为“接触态势 / 战区态势 / 边境压力 / 接敌 / 先阵”。
- 粮草调查子 Agent 确认补给、粮仓、府库粮草、渡口/港口和 AI 太守已有基础权重；后续可继续做地图粮道显示和 AI 目标权重切片。
- 骑兵调查子 Agent 建议低风险先强化骑兵基础攻击、移动和平原冲击，再单独处理名将技能与 AI tactic。

## 轻量检查

按 `md/test/test.md`，本轮只运行允许的轻量检查：

- `swiftc -parse` 针对改动 Swift 文件。

- `git diff --check`。

本轮不运行 Xcode build / XCTest / 模拟器 / Probe / Smoke / Full；重验证交给 GitHub Actions。

## 风险与后续

- 底层工程名和历史记录仍会出现 `FrontLine` / “前线”，这是兼容和追溯用途，不代表玩家体验目标。
- 后续还需要运行时截图复核每个面板是否真正完成古代口径，并继续把天下局势、行军道、关隘、粮仓和名将存在感做成玩法。

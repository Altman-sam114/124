# v3.7-preflight.77 AppContainer 交互日志与存档反馈 legacy 文案收口记录

## 目标

收口 `AppContainer` 中交互日志、存档反馈、地图选择、军队选择和 fallback 总管名称里的 legacy 势力名、旧地名、旧单位词与 raw id 展示风险。

## 范围

本轮只改 `AppContainer` 展示 helper 和文档；不改 `GameState`、`GameSaveStore`、`GameSaveStatus`、`GameLogEntry`、`Command` 合同、`Division.name`、`RegionNode.name`、`FrontZone.name`、JSON、id / rawValue、AI 决策、命令结构、规则执行或存档 schema。

## 实现摘要

- 存档载入、继续、自动保存、新局和切换执掌势力反馈中的势力名改走 `displayFactionName(_:)`，legacy `.germany` / `.allies` 显示为“旧剧本势力”。
- 军队点击日志中的 `division.name` 改走 `displayDivisionName(_:)`，清理 `division_` / `unit_` raw id 和旧“德军 / 盟军 / 装甲 / 摩托化 / 炮兵 / 步兵 / 师”口径。
- 州郡经营拒绝、地图地块选择和总管防区提交消息中的 region / front zone 名称改走 `displayMapName(_:fallback:)` 或 `displayFrontZoneName(_:)`。
- fallback 临时总管和自动总管名称中的势力 / 防区展示改走同一套 helper。
- `submit(_:)` 的交互日志不再直接使用外交 / 归附命令的 `Command.displayName`，在 AppContainer 层用 `displayCommandName(_:)` 重新净化势力展示；`Command.displayName` 合同保持不变。

## 并发扫描结论

- 只读子 Agent 复扫确认本轮主要风险集中在 `AppContainer` 的 `activeFaction.displayName`、`faction.displayName`、`submitted.displayName`、`division.name`、`region.name` 和 `displayFrontZoneName(_:)` 原样返回 `zone.name`。
- 子 Agent 追加提醒 `submit(_:)` 的 `command.displayName` 可能通过外交 / 归附命令带出 legacy 势力名，本轮已在 AppContainer 展示层收口。
- 规则 / 战报源头和 legacy prompt 输入仍是下一轮优先复扫对象。

## 边界

- 不改存档读写路径、文件名、JSON 编解码、自动保存触发点或 `GameSaveStatus` 字段。
- 不改 `Command.displayName`、`Faction.displayName`、`Division.name`、`RegionNode.name`、`FrontZone.name` 的数据层语义。
- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `CommandValidator` / `RuleEngine` 管线。
- 不把本地展示 helper 抽成全局 sanitizer；后续若更多入口重复，再评估共享层。

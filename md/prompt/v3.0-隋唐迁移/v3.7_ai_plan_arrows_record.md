# v3.7-preflight.5 AI 计划箭头记录

完成日期：2026-07-05

## 1. 本轮定位

本轮继续补 v3.7 发布候选前置的战局可读性缺口，目标是把最近 AI 战区指令以地图计划箭头展示出来。

本轮只读 `WarDirectiveRecord`，不新增 AI 决策、不改变 directive 执行、不修改 `GameState` 战术权威，也不改变玩家和 AI 仍需走 `ZoneDirective -> WarCommandExecutor -> RuleEngine` 的执行边界。

## 2. 实现摘要

### 2.1 渲染状态接入指令记录

`BoardRenderState` 新增：

- `recentDirectiveRecords: [WarDirectiveRecord]`

`BoardSceneAdapter.renderState(from:)` 优先使用 `AppContainer.lastWarDirectiveRecords`；若为空，则回退到 `gameState.warDirectiveRecords` 的最近记录。这样新回合 UI 可以显示刚执行的 AI 指令，继续存档后也能从持久化战局记录中恢复最近 directive 线索。

### 2.2 AI 计划箭头

`BoardScene.drawPlannedOperations` 保留原有玩家计划线，并追加绘制最近 6 条非玩家 directive：

- `issuerId == "player"` 的玩家总管军令不在 AI 计划层重复绘制。
- 有 `zoneId` 和 `directiveType` 的记录才参与绘制。
- 攻击指令优先使用 `commandTarget.region`，否则使用 `targetRegionIds.first` 推导目标州郡。
- 防御指令或无明确目标的记录在来源防区画防守圈。
- AI 计划使用虚线和势力色，和玩家实线计划区分。

### 2.3 架构边界

来源点仍按现有部署层和将军分配推导：

```text
WarDirectiveRecord.zoneId
  -> FrontZone
  -> generalAssignment.hqRegionId 或 zone.regionIds.first
  -> representativeHex
```

目标点仍是州郡代表 hex。该层只画复盘/计划提示，不参与移动、攻击、占领、前线或部署刷新。

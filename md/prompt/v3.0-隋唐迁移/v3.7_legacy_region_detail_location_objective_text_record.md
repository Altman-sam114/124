# v3.7-preflight.73 州郡详情 legacy 地名与目标名展示收口记录

## 目标

收口州郡详情面板中 legacy 地名、目标名、驻军名和 raw id 的玩家可见直出风险，让 `RegionInspectorView` 成为州郡详情展示出口的最后一道本地净化层。

## 范围

本轮只改 `RegionInspectorView` 的州郡详情展示层和文档，收口 legacy fallback 地名、州郡名、目标名、驻军名或 raw objective/location id 的玩家可见直出风险；不改 region / hex / objective / division 存储，不改 JSON schema、id、rawValue、objectiveId、keyLocations、胜负规则、地图数据、AI 决策、命令结构、`RuleEngine`、hex / region / theater / front / deploy 权威或运行时状态。

## 实现摘要

- 州郡标题从直接显示 `state.region.name` 改为 `displayMapName(state.region.name, fallback: "州郡")`。
- 城邑、当前方面、行军防区、dominant 方面 / 防区展示统一进入 `displayOptionalMapName(_:fallback:)`。
- 要地列表由 `objectiveNameSummary(_:)` 汇总，逐项清理 objective/location 名称再拼接展示。
- 己方军队和可见敌军列表由 `displayDivisionName(_:)` 处理，避免单位 raw id 或旧兵种词继续直出。
- 地名 helper 覆盖 `region_*`、`theater_*`、`front_zone_*`、`obj_*`、`hex_*`、`germany_*`、`france_*`、`allied_*`、`axis_*` 等常见 raw id，并替换 `阿登 / 巴斯托涅 / 圣维特 / Ardennes / Bastogne / St. Vith / St Vith / Sedan` 等 legacy 地名。

## 并发扫描结论

- Faraday 定位 `RegionInspectorView` 中州郡名、城邑名、方面名、防区名、目标名和驻军名的直接展示风险。
- Herschel 确认本轮记录命名和需要同步的文档范围。
- James 只读核对 `MapDisplayAdapter`，确认 `RegionInspectorState` 仍由 adapter 聚合输入；adapter 已对 theater/front zone 名做基础 fallback，但不净化 `region.name`、`city.name` 和 `objectiveNames`。本轮因此保持 `MapDisplayAdapter` 不变，只在 `RegionInspectorView` 做展示净化。

## 边界

- `MapDisplayAdapter` 不改，`RegionInspectorState` 合同不扩展。
- `hexToTheater` 和 `hexToFrontZone` 仍是运行时动态归属权威。
- `regionToTheater` / `regionToFrontZone` 仍只作初始、dominant 或 fallback 依据。
- objective id、keyLocations、胜负规则和地图数据不改。
- `Division.name`、templateId、rawValue、兵种显示权威和战斗规则不改。

# v3.7-preflight.56 legacy static fallback 目标兼容与展示文案收口记录

完成日期：2026-07-06

## 目标

继续推进隋末唐初迁移发布候选前置收口，处理 legacy 阿登静态 fallback 路径中仍可能进入玩家视野的英文地点、单位和胜负原因；同时避免前序中文化地点名后，旧阿登胜负规则和 MockAI 目标选择仍只按英文展示名查找导致目标失配。

本轮只改静态 fallback 展示文本和目标查找兼容，不改 JSON key、id、rawValue、schema、坐标、faction、templateId、规则阈值、胜负条件语义或命令管线。

## 改动范围

- `WWIIHexV0/Core/MapState.swift`
  - 静态 `MapState.ardennesV0()` 中的巴斯托涅、圣维特、侯法利兹和巴斯托涅要塞展示名中文化。
  - 静态 objective 的 `name` 中文化，保留 objective id 不变。
- `WWIIHexV0/Core/GameState.swift`
  - 静态 `GameState.initial()` legacy 单位展示名和初始化战报中文化。
- `WWIIHexV0/Core/VictoryState.swift`
  - legacy 阿登胜负原因从“旧剧本 / AI / 玩家”口径改为“阿登 / 德军 / 盟军”口径。
- `WWIIHexV0/Rules/VictoryRules.swift`
  - 阿登胜负判断优先按 objective id 查找，再兼容中文和英文 objective 展示名。
- `WWIIHexV0/Rules/RegionVictoryRules.swift`
  - region 级 legacy 胜负评估优先按 objective id 查找，再兼容中文和英文城名。
- `WWIIHexV0/Agents/MockAIClient.swift`
  - legacy MockAI 目标选择优先按 `bastogne` objective id 查找，再兼容“巴斯托涅 / Bastogne”展示名。

## 边界

- 保留 `"Bastogne"` / `"St. Vith"` 作为旧数据兼容查找值，不作为玩家可见展示文本。
- 不改变 `VictoryReason` case、存档字段、objective id、胜负阈值或行动策略。
- 不改默认隋唐 `wude_618` 数据与运行时胜负条件。
- 不运行 Xcode build、XCTest、模拟器或 app。

## 遗留风险

- 未启动 app 或 MapEditor 验证静态 fallback 样例在 UI 中的实际展示效果。
- 英文地点名仍作为旧数据兼容查找值存在于规则和 MockAI 代码中；这不是玩家展示文本。
- MapEditor 默认导出 metadata、隋唐将领 `name` 字段和 victory condition `status` 的进一步收口留给后续独立切片。

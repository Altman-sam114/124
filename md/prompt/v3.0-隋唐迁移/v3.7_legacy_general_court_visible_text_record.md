# v3.7-preflight.70 legacy 将领与朝堂记录可见文案复扫

完成日期：2026-07-06

## 目标

在 UI/战报/外交记录净化 helper 对齐之后，继续抽样复扫 legacy 将领、朝堂步骤和结构化军令记录中的玩家可见旧题材 / 工程词。目标是减少旧 Agent D / legacy 将领配置、朝堂记录和外交面板摘要继续暴露“装甲”“AI 自动回合”“JSON”等词的机会。

## 实现范围

- `GameAgent.guderianFallback` 的 legacy fallback prompt 改为“破阵突击 / 突击部队”口径。
- `GameAgent.localizedPersonalityPrompt(for:)` 对 legacy `guderian` 数据的可见 prompt 兜底同步改为隋唐化突击口径。
- `general_agents.json` 中 legacy `guderian` 的 `personalityPrompt` 去掉“装甲部队”。
- `generalSkillDisplayName` 中 `armor_expert` / `armor_theory` 展示名改为“突击战法”。
- `RulerAgent` 使者层 rationale 去掉“AI 自动回合”，改为“自动轮转”。
- `TurnManager` 的结构化军令记录标题去掉 `JSON` 字样，保留 `rawJSON` 字段和编码内容。
- `DiplomacyPanelView.displayRecordText` 补齐与 AI / 战报面板同类的旧军语、工程词和 legacy 将领名净化。
- `md/plan/plan.md` 修正当前结果标题和下一轮阅读清单。

## 边界

- 不改 `guderian`、`ger_panzer_*`、`armor_expert`、`armor_theory`、`rawJSON` 等内部 id、rawValue 或字段名。
- 不改 `GeneralAgentDefinition`、`GameAgent`、`CourtAgentStepRecord`、`AgentDecisionRecord`、`DirectiveEnvelope` 或存档 schema。
- 不改 legacy MockAI / LocalLLM / MarshalAgent 的策略、排序、解析、编译或 fallback 行为。
- 不改 UI 布局，不新增公共 sanitizer，不改命令结构、规则执行或运行时状态。

## 并发扫描结论

- `RulerAgent` 使者步骤 rationale 会进入朝堂记录和面板展示，`AI 自动回合` 应改为玩家语义。
- `TurnManager` 写入 `rawJSON` 的标题含 `JSON`，可改展示标题但不能改字段名或实际结构化原文。
- `GeneralCommandPanelView` 的 `armor_expert` / `armor_theory` 展示名仍是“装甲战法”，会被将领面板复用。
- `md/plan/plan.md` 当前结果标题仍停在 `.67`，下一轮阅读清单没有点名 `.68/.69` 记录，容易让接手口径回退。

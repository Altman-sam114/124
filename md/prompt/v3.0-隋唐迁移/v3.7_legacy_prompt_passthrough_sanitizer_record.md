# v3.7-preflight.64 legacy prompt 直通文本净化

完成日期：2026-07-06

## 目标

在 legacy prompt 内部编号分层之后，继续处理旧 `AgentPromptBuilder` 中 `recentEvents.message` 与 `playerDirective` 仍原样进入 LocalLLM prompt 的问题，避免上游战报、玩家意图或历史记录中残留的 raw id、工程词和旧英文口径再次诱导模型写入 `intent`、`reason`、`stance`。

## 实现范围

- `AgentPromptBuilder` 对近期战报逐条调用局部 `sanitizePromptText(_:)` 后再拼入 prompt。
- `AgentPromptBuilder` 对 `playerDirective` 调用同一局部净化后再拼入 prompt。
- 新增 `sanitizePromptText(_:)`：
  - 复用当前展示层已验证过的内部词替换口径，例如 `Heinz Guderian`、`rawJSON`、`local-model`、`ZoneDirective`、`RuleEngine`、`MockAI`、`hexToTheater` 等。
  - 新增 prompt-local raw id 正则净化，覆盖 `war_directive_*`、`player_directive_*`、`submission_handoff_*`、`diplomacy_event_*`、`court_decision_*`、`ruler_decision_*`、`region_*`、`theater_*`、`front_zone_*`、`obj_*`、`hex_*`、`division_*`、`unit_*`、`command_*`、`agent_*` 和朝堂角色前缀记录。
- 净化只用于 legacy LocalLLM prompt 的自由文本区，不作用于“内部编号”小节和 JSON schema。

## 边界

- 不改 `EventLogView`、`AgentPanelView`、`DiplomacyPanelView` 或 `TurnManager` 的现有展示 sanitizer，不做共享 formatter 重构。
- 不净化 `divisionId`、`targetDivisionId`、`toRegionId` 内部编号表，不改 JSON key、schema、type rawValue 或 parser / mapper 合同。
- 不改默认战争 AI 主路径；旧 `AgentPromptBuilder` 仍是 legacy LocalLLM / Agent D 回归参考路径。
- 不改 AI 决策策略、命令结构、规则执行、存档字段或运行时状态。

## 遗留风险

- `context.agentId`、内部编号表和 `type` rawValue 仍必须暴露给 prompt 以维持解析合同，不能完全隐藏英文 key / rawValue。
- 上游 `recentEvents` 与 `playerDirective` 的原始存储文本不被改写；本轮只净化进入 legacy prompt 的自由文本副本。
- 本轮没有启用真实 LocalLLM 或 legacy Agent D 管线验证输出质量，只做 prompt 文本入口收口。

# v3.4 朝堂 AI Agent 分层记录

完成日期：2026-07-05

性质：AI 上游分层与审计记录最小迁移。仍不是完整内政、外交、真实 LLM 或发布级 AI。

## 目标

- 在现有 `MarshalAgent -> TheaterDirectiveEnvelope -> TheaterDirectiveCompiler -> ZoneDirective` 之上接入隋唐朝堂分层。
- 让 AI 回合可以解释“君主想要什么、谋主如何转成方面目标、行军总管下发了什么、太守/将领/使者记录了什么”。
- 保持最终执行仍统一进入 `ZoneDirective -> WarCommandExecutor -> RuleEngine`。

## 已完成

- `DiplomacyState` 新增 `CourtAgentRole`、`CourtAgentStepRecord`、`CourtDecisionRecord` 和 `courtRecords`，并提供 Codable 兼容解码、`latestCourtRecord`、`appendCourtRecord`。
- `RulerStrategicPosture.displayName` 改为隋唐中文姿态：锐意进取、谨守根本、维系盟从、稳固战线。
- `RulerAgent.adjust` 在塑形 attack / defense directive 时保留 focus、support、convergence、coordinated zones、maxCommittedUnits、exploitDepth、fallback、counterattack、strongpoint 等字段。
- 新增 `CourtAgent` 最小分层：
  - Sovereign / 君主：调用 `RulerAgent` 选择姿态并塑形 `DirectiveEnvelope`。
  - Strategist / 谋主：记录元帅意图或 fallback 方面目标。
  - Governor / 太守：只记录补给、围城和州郡后勤关注点。
  - MarchCommander / 行军总管：只记录将下发的 `ZoneDirective`。
  - General / 将领：记录战术偏好和投入节奏。
  - Diplomat / 使者：只记录当前外交态势，暂不执行外交动作。
- `TurnManager` 在 `.marshalDirective` 和 `.zoneDirective` 两条 AI directive 路径中调用 `CourtAgent`；执行使用 court 调整后的 `DirectiveEnvelope`。
- `TurnManager.executeDirectiveEnvelope` 追加 `RulerDecisionRecord` 和 `CourtDecisionRecord` 到 `GameState.diplomacyState`，并写一条朝堂决策事件；不直接修改 hex、单位、战区、部署或外交关系。
- `AgentPanelView` 增加“朝堂链路”展示，显示每个角色的 agent id、摘要、目标方面/州郡和 directive 数。
- `RootGameView` 将 `latestCourtRecord` 传给 AI 面板。

## 关键边界

- `CourtAgent` 不是规则执行器，不生成底层 `Command`，不绕过 `WarCommandExecutor`。
- 君主层只能塑形和审计现有 `DirectiveEnvelope`，不能直接改 `GameState` 战术权威字段。
- 太守和使者当前只记录内政/外交态势，不执行征发、屯田、归附、停战或同盟。
- 元帅 JSON 解码或编译失败时仍使用 `TheaterCommanderPool` fallback，再进入朝堂层审计和塑形，不执行半成品 JSON。

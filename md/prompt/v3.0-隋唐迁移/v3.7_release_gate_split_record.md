# v3.7-preflight.25 发布检查门禁拆分

## 目标

在 v3.7-preflight.8 已有发布说明、资产边界和验证口径，且 v3.7-preflight.24 已把善后完成状态补进发布检查后，把发布前检查面板进一步拆成清晰门禁：代码已接入、运行时未验证、后续功能。避免把“代码候选已经接入”“需要授权运行时重测”和“未来玩法功能”混在同一个待确认列表里。

## 范围

- `ReleaseChecklistView` 版本更新为 `天命开唐 Agent · v3.7-preflight.25`。
- 发布前检查顶部状态新增“代码状态：候选接入”和“运行时门禁：未授权”。
- 原“已接入”清单改为“代码已接入”，继续列出现有代码路径和规则管线接入项。
- 原“仍待确认”拆成“运行时未验证”和“后续功能”，分别承载授权重测门禁与尚未实现的功能方向。
- 同步 README、核心流程文档、流程图、计划文档和更新日志。

## 非目标

- 不新增、修改或删除任何 `Command`、validator、executor 或 `RuleEngine` 行为。
- 不改变 `GameState`、hex controller、unit、theater、front、deploy、diplomacy 或 save schema。
- 不声称已完成 Xcode build、iOS/macOS 启动、UI 点击烟测、观察者多回合或性能验证。
- 不实现忠诚、叛乱、安置、水战、siege progress、真实模型接入或完整天命/民心效果。

## 数据流

```text
ReleaseChecklistView
  -> releaseNotes / assetBoundaries
  -> codeReadyItems
  -> runtimeGateItems
  -> futureFeatureItems
  -> 玩家看到代码候选、运行时门禁、后续功能三类状态
```

该链路只读取 `GameState`、本地存档状态和 `GameSaveStatus` 以展示当前发布检查，不向规则系统提交命令，也不直接修改运行时状态。

## 架构边界

- Hex 仍是战术权威；本轮不修改 `HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- 玩家和 AI 行动仍统一进入 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- 发布检查面板只做状态说明，不能替代云端 CI、Xcode build、模拟器启动、真实 UI 点击或多回合运行验证。

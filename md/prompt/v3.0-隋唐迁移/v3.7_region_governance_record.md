# v3.7-preflight.10 州郡经营与太守命令闭环记录

完成日期：2026-07-05

## 目标

补上最小州郡经营命令，让玩家能在州郡面板通过“太守”动作提交修道、屯田和安民，并让这些动作经 `Command -> CommandValidator -> RuleEngine -> CommandExecutor` 执行。

## 范围

- 新增 `Command.governRegion(regionId:policy:)` 和 `RegionGovernancePolicy`。
- `CommandValidator` 校验当前阶段、行动势力、己方实际控制州郡、府库资源和经营上限。
- `CommandExecutor` 扣除府库资源，并更新 `RegionNode.infrastructure`、`supplyValue` 或 `occupationState`。
- `AppContainer` 提供选中州郡经营能力判断和提交入口。
- `RegionInspectorView` 新增“太守”动作区，按钮只提交命令，不直接改状态。
- `ReleaseChecklistView` 更新到 `天命开唐 Agent · v3.7-preflight.10`。

## 非目标

- 不让 AI 太守自动生成内政命令。
- 不新增天命、民心、叛乱、完整政务模型或新 JSON schema。
- 不直接修改 `HexTile.controller`、`Division.coord`、`hexToTheater`、`hexToFrontZone` 或 `regionToTheater`。
- 不执行 Xcode build、XCTest、模拟器、UI test、性能测试或完整发布候选重测。

## 命令语义

`RegionGovernancePolicy` 当前有三类：

- `repairRoads`：修道，消耗军械和粮草，提升 `RegionNode.infrastructure`，上限 6。
- `organizeTuntian`：屯田，消耗丁口、军械和粮草，提升 `RegionNode.supplyValue`，上限 6。
- `pacifyPopulation`：安民，消耗丁口和粮草，降低 `OccupationState.resistance` 并提升 `compliance`。

这些效果只影响州郡战略字段和后续经济派生，不替代 hex 战术占领。

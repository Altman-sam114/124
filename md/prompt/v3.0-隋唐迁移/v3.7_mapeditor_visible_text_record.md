# v3.7-preflight.41 MapEditor 可见文案收口记录

完成日期：2026-07-06

## 目标

继续总提示词中的发布候选前置收口，处理 MapEditor 中仍可能被编辑者直接看到的旧题材、工程化和内部路径文案，使默认隋唐编辑器入口更接近“地块、州郡、方面、军队、地点、战局数据”的产品语义。

## 实现摘要

- `MapEditorView` 的州郡、方面和信息面板展示改为名称优先，不再把州郡 ID、方面 ID 或完整资源路径作为普通可见信息。
- MapEditor 四处势力 Picker 改用 `Faction.suitangTurnOrder` 作为编辑器可选列表，隐藏 legacy 势力选项；`Faction.allCases` 本身仍保留完整兼容语义。
- 编辑器按钮、提示和状态文案从 “JSON / ID / 快捷键” 等工程口径调整为战局数据、编号和中文操作说明。
- `MapEditorViewModel` 的创建、读取、保存、导出和错误反馈改用中文泛化说明，避免把 raw id、系统错误详情或完整路径直接显示给编辑者。
- `MapEditorCanvasScene` 的单位短标改为步、骑、弩、器、守、甲、炮、机、军等中文短标，减少 `ARM / ART / MOT / INF` 残留。
- `MapEditorExporter` 与 `MapEditorGameResourceBridge` 的导出/资源桥错误改为中文可见说明，避免把 raw id、系统错误或完整路径带入用户反馈。

## 边界

- 本轮只改 MapEditor 可见文案、状态反馈、错误描述、单位短标和路径显示。
- 不改主游戏运行规则、不改 `GameState`，不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- 不改 JSON schema、rawValue、内部 id、导出 key、objective id、unit id 或资源文件名。
- legacy `Faction.germany` / `Faction.allies` 等兼容枚举仍保留；MapEditor UI 只是在默认隋唐编辑入口隐藏旧势力选项。

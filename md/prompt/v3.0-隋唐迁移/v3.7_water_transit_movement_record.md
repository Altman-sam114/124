# v3.7-preflight.108 己控渡口港口移动渡河减免记录

切片 ID：v3.7-preflight.108

来源：总提示词 §0.2 当前可执行队列中的“完整水战、移动渡河、港口补给源与粮道扩展规则”子问题。

当前基线：

- `origin/main` 最新提交：`c473966 data: add water transit river edges`。
- 顶部版本状态：v3.7-preflight.107 已让默认 `wude_618` 四个水路地点自身 hex 有非空 `riverEdges`。
- 受控水路补给 anchor 已由 v3.7-preflight.106 落地；跨河补给成本减免已由 v3.7-preflight.103 落地。

目标：

- 在战术移动路径成本中消费己方控制的 `ferry` / `port` / `harbor` marker。
- 非道路跨 `riverEdges` 行军时，若跨河两端任一 hex 是移动方己控水路通行点，则免除该段既有渡河额外成本。
- 让玩家移动高亮、移动校验、AI 选点和执行方向通过既有 `MovementRules.shortestPath` / `movementRange` 自动复用该规则。

非目标：

- 不实现完整水战、水师单位、河面控制、港口部署点或完整港口补给源。
- 不改 JSON schema、默认地图尺寸、单位、地点、MapEditor 或补给规则。
- 不自动双写邻接 hex 反向河边。
- 不改变公开 `movementCost(from:to:direction:)` 的纯地形语义。

- `MovementRules.shortestPaths` 改用内部 state-aware 移动成本函数。
- 新增内部 helper，按 `marker.kind.isWaterTransit` 与 marker 所在 hex `controller == movingFaction` 判断己控水路通行点。
- 己控水路点只影响非道路跨河移动的额外成本；敌控或未控水路点不提供移动减免。
- 新增 `RuleEngineCoreTests.testControlledWaterTransitReducesRiverCrossingMovementCost`，作为云端 XCTest 语义保护。

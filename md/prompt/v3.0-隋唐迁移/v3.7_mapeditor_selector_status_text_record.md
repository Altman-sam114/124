# v3.7-preflight.76 MapEditor 选择器与状态消息 legacy 文案收口记录

## 目标

收口 MapEditor 中州郡 / 方面选择器和编辑状态消息的 legacy 名称、raw id 与旧题材词展示风险。

## 范围

本轮只改 `MapEditorView` / `MapEditorViewModel` 的展示 helper 和文档；不改 `MapEditorDocument`、导出 JSON、id / rawValue、默认资源桥、MapEditor 存储 schema、主游戏数据加载、地图规则、AI、命令或运行时状态。

## 实现摘要

- `MapEditorView` 的“当前州郡”“当前方面”下拉项不再直接显示 `region.name` / `theater.name`，改为走 `MapEditorViewModel.displayName(for:)`。
- `MapEditorViewModel.displayName(for:)` 增加局部展示净化，清理常见 region / theater / objective / location / unit raw id、legacy 地名、旧势力词和旧“战区”口径。
- 创建州郡、创建方面、保存地点、删除地点的状态消息显示前先走同一展示净化。
- 原始名称仍保存在 `MapEditorDocument` 中，TextField 编辑和导出仍使用原始数据；本轮只改可见展示兜底。

## 并发扫描结论

- 主线程 UI / MapEditor 扫描定位 `MapEditorView` 下拉项仍直接显示 `region.name` / `theater.name`，且 `MapEditorViewModel` 状态消息直接拼接地点名。
- Godel 标记下一步 AI/prompt 重点：`AgentPromptBuilder` prompt 输入和 sanitizer、`ZoneCommanderAgent` fallback marshal config、`RulerAgent` 朝堂 step agent id。
- Pauli 标记下一步规则 / App 重点：`AppContainer` 交互日志、`CommandExecutor` / `SupplyRules` 事件、`WarCommandExecutor` 动态推进战报、`EconomyRules` 经济战报。

## 边界

- 不改 MapEditor 的文档数据、资源覆盖保存、导出 metadata、keyLocations、scenario / regions JSON schema。
- 不改 `RegionId`、`TheaterId`、`ObjectiveId`、`Location`、单位模板 id 或 faction rawValue。
- 不把 MapEditor 展示 helper 抽成全局 sanitizer；后续若重复继续扩大，再评估共享 helper。

- `swiftc -parse MapEditor/MapEditorView.swift`

- legacy/raw 词扫描只允许命中 helper 词表或正则。

# v3.7-preflight.59 战报意图屏蔽与中文分类收口

完成日期：2026-07-06

## 目标

在外交面板名称与记录净化收口之后，继续处理战报面板中仍可能把 v3.x 版本诊断、二战旧专名或中文隋唐战报降级展示的问题。该版本只改 `EventLogView` 展示层，不改变日志 schema、事件来源、record id、AI 决策、命令结构或规则执行。

## 实现范围

- `displayedIntent(_:)` 的结构化意图屏蔽从只识别 `v0.` 扩展为识别 `v[0-9]+(.N)*` 版本号，避免 v3.x 诊断文本直出为军议意图。
- `displayedIntent(_:)` 对 `attack_front`、`move_to_hex`、`hold_position` 等复合 intent 使用中文分类摘要；未知 snake_case intent 统一显示“已记录军议意图”，不回显原始字符串。
- 审计 record id 净化前移到 `sanitizeRawIdentifiers(in:)`，避免普通 `directive` 文本替换先执行后留下 `war_军令_xxx` 一类半清洗文本。
- `sanitizeRawIdentifiers(in:)` 补齐 `hex_*`、`division_*`、`unit_*`、`command_*` 等常见 raw id 前缀。
- 战报诊断净化将旧 `Heinz Guderian` / `Guderian` 统一显示为“历史总管”，减少旧二战专名回流。
- 战报诊断净化把 `model`、`legacy pipeline`、`RuleEngine` 调整为“军议来源”“备用军议路径”“军令校验”等玩家语义。
- `LogDisplayCategory` 的 fallback 分类补齐中文关键词：
  - 撤退、退却、溃退。
  - 补员、整补、补充、整军。
  - 围困、包围、断粮。
  - 攻击、进攻、进军、战斗、打击、伤亡、损失、命中。
  - 粮道、补给、粮草、军粮。

## 边界

- 不改 `GameLogEntry`、`AgentDecisionRecord` 或 `WarDirectiveRecord`。
- 不改日志写入源头、事件 category schema、record id 或存档字段。
- 不改战斗、补给、撤退、围困、外交、AI 决策或命令管线。
- 不新增运行时结论，不代表完整战报 UI 已完成授权截图或多回合验证。

## 轻量检查

本轮按 `md/test/test.md` 仅执行允许的轻量检查；未运行 Xcode build、XCTest、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full、性能测试或 app 启动。

## 遗留风险

- 战报分类仍是展示层关键词兜底，不能替代源头事件 category 的长期治理。
- 未通过真实运行时多回合战报样本确认所有组合的分类效果。

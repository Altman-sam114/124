# v3.7-preflight.106 受控渡口港口补给投送点记录

完成日期：2026-07-07

## 目标

在 v3.7-preflight.103 已让渡口/港口减免跨河补给成本之后，进一步让己方控制的渡口、港口、海港作为战术补给投送点参与单位补给线和撤退判定。

## 实现

- `SupplyRules` 内部新增 `effectiveSupplyAnchors(for:in:)`。
- effective anchors 由两部分组成：
  - 既有 `state.map.supplySources(for:)` 坐标。
  - 己方控制且可通行的 `MapFeatureKind.ferry` / `port` / `harbor` marker 坐标。
- 控制权以 marker 所在 hex 的 `HexTile.controller == faction` 为准，不读取 `marker.faction` 作为运行时权威。
- `hasSupplyLine(for:in:)` 改用 effective anchors。
- `isSafeRetreatTile(_:for:in:)` 改用 effective anchors。
- `retreatSortKey(for:faction:in:)` 改用 effective anchors 计算路径成本和距离。
- `RuleEngineCoreTests` 增加己控 ferry 作为补给 anchor 的语义断言。

## 非目标

- 不写入或扩展 `MapState.supplySources`。
- 不改变 JSON schema。
- 不改变省级 `RegionSupplyRules`。
- 不改变经济部署点或新兵落点。
- 不改变移动、战斗、水战或渡河命令。
- 不新增港口建筑、运输命令或水师规则。

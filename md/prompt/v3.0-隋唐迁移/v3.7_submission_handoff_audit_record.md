# v3.7-preflight.17 归附交接审计记录

完成日期：2026-07-05

## 目标

在 v3.7-preflight.16 已经通过 `Command.resolveSubmissionHandoff` 完成归附实体交接的基础上，补上最小结构化审计记录：

- 交接结果可进入 `DiplomacyState` 存档。
- 外交战报可通过 `relatedRecordId` 关联到交接记录。
- 外交面板可展示最近一次交接摘要。

## 非目标

- 不删除外交档案、关系或历史事件。
- 不新增忠诚、叛乱、贡赋、俘虏、安置或交接后治理系统。
- 不改变 v3.7-preflight.16 的交接规则：仍只转移未毁灭军队和可通行受控 hex。
- 不绕过 `Command -> CommandValidator -> RuleEngine -> CommandExecutor`。

## 实现摘要

- `DiplomacyState` 新增 `SubmissionHandoffRecord`，字段包含 turn、submitted、recipient、transferredDivisionCount、transferredHexCount、affectedRegionIds、summary 和 boundaryNote。
- `DiplomacyState.submissionHandoffRecords` 通过 Codable 保存，旧存档缺字段时默认空数组，最多保留最近 80 条。
- `CommandExecutor.executeSubmissionHandoff` 在完成归属转移和派生层刷新后追加交接记录，并把外交战报 `relatedRecordId` 指向该记录。
- `DiplomacyPanelView` 新增“交接”区，显示最近交接摘要、影响州郡数量、回合和边界说明。
- `ReleaseChecklistView` 更新为 `天命开唐 Agent · v3.7-preflight.17`。

# v3.7-preflight.44 Legacy MockAI 与元帅解析诊断中文化记录

完成日期：2026-07-06

## 目标

在 AI 元帅/方面军令摘要中文化之后，继续处理 legacy Agent D / MockAI 和元帅 directive 解析失败路径中仍可能进入 `AgentDecisionRecord`、`AgentPanelView`、`EventLogView` 或战报摘要的英文工程口径，减少 `mock directive(s)`、`Mock Commander`、`v0.33 deployment`、`FrontZone`、`Theater directive JSON` 等残留。

## 实现摘要

- `MockAICommander` 的 `DirectiveEnvelope.theaterContext` 改为中文“本地模拟总管生成 N 条方面军令”。
- `MockAICommander` 默认总管名称从 `Mock Commander` 改为“本地模拟总管”。
- `MockAIClient` legacy `AgentDecisionEnvelope.intent` 与 `AgentOrder.reason` 改为中文军议说明，不再在可见审计里输出 `Bastogne`、`v0.33 deployment`、`FRONT`、`DEPTH`、`GARRISON` 等工程口径。
- `TheaterDirectiveDecoderError.errorDescription` 改为中文，避免元帅军令解析失败时经 `error.localizedDescription` 继续进入可见 fallback 诊断。
- `EventLogView.displayDiagnosticText` 对齐 Agent 面板的净化表，补上 `rawJSON`、`JSON/json`、`provider`、`WarDeploymentState`、`legacy pipeline`、小写 `agent`、`directive` 和 `diagnostic` 的中文替换。

## 边界

- 本轮只改 legacy MockAI / Agent D 可见摘要、reason、intent、解析错误和战报诊断净化文案。
- 不改 `MockAIClient` 的启发式分支、目标选择、行动排序、stance rawValue、order schema 或 JSON key。
- 不改 `TheaterDirectiveEnvelope` / `TheaterDirective` schema，不改 decoder 提取 fenced JSON 的合同。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`，不改变移动、攻击、补给、外交、州郡经营、胜负或回合推进规则。
- 数据子 agent 发现的 legacy JSON `MapEditor Scenario`、`City q,r`、`Supply q,r`、旧 unit/general displayName 等可见残留留给后续独立数据切片。

# v3.7-preflight.63 legacy prompt 内部编号分层收口

完成日期：2026-07-06

## 目标

在 legacy 总管配置中文兜底之后，继续处理旧 `AgentPromptBuilder` 中 raw id、英文命令 type 和英文 schema 示例值混在战场摘要正文里，可能诱导 legacy LocalLLM 把内部编号或英文 type 回写到 `intent`、`reason`、`stance` 的问题。

## 实现范围

- `AgentPromptBuilder.systemPrompt` 明确要求 `intent`、`reason`、`stance` 不复述内部编号、JSON key 或英文 type 值，除非字段本身要求填写编号。
- 战场摘要中的目标、己方军队、敌军和可见州郡改为中文名称优先：
  - 州郡位置显示展示名，不再混写 `regionId.rawValue`。
  - 军队摘要显示军队名、兵力、位置、补给和行动状态，不再把 `division.id` 放在正文开头。
  - 可见州郡的相邻项显示相邻州郡名，不再直接拼接 neighbor rawValue。
- 单独增加“提交军令时必须使用的内部编号”小节：
  - `divisionId` 只列己方军队编号。
  - `targetDivisionId` 只列敌军目标编号。
  - `toRegionId` 只列可行军州郡编号。
- JSON schema 示例中的英文占位短语改为中文说明，同时保留 `schemaVersion`、`agentId`、`turn`、`intent`、`orders`、`type`、`divisionId`、`toRegionId`、`targetDivisionId`、`stance`、`reason` 等解析合同字段。
- `type` 字段仍明确只能填 `move`、`attack`、`hold` 或 `resupply`，避免破坏 `AgentOrderType` Codable 解析。

## 边界

- 不改 `AgentDecisionEnvelope`、`AgentOrder`、`AgentOrderType`、`AgentDecisionParser` 或 `AgentCommandMapper`。
- 不改 JSON key、schema version、agent id、division id、region id、target id 或命令 type rawValue。
- 不改默认战争 AI 主路径；旧 `AgentPromptBuilder` 仍是 legacy LocalLLM / Agent D 回归参考路径。
- 不改 AI 决策策略、命令结构、规则执行、存档字段或运行时状态。

## 遗留风险

- 旧 LocalLLM prompt 仍必须向模型暴露解析需要的内部编号和 `type` rawValue，不能完全去除英文合同值。
- 本轮没有启用真实 LocalLLM 或 legacy Agent D 管线验证输出质量，只做 prompt 文本结构收口。
- 默认元帅链路不依赖 `AgentPromptBuilder`，本轮收益主要是降低旧路径回归或未来启用 LocalLLM 时的英文/内部编号回流风险。

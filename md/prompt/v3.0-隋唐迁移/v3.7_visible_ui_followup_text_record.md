# v3.7-preflight.39 UI 文案复扫收口记录

完成日期：2026-07-05

## 1. 本轮目标

在 v3.7-preflight.38 剩余 UI 文案抽样收口后，按并发子 agent 的复扫结果继续处理仍可能出现在玩家界面里的发布验收口径、规则工程口径、旧剧本 fallback、斜线分隔和 `N/M` 兵力显示。

## 2. 范围

- `ReleaseChecklistView`：把“首发说明 / 已完成事项 / 待实机复核 / 基础检查”等发布验收口径改为战局说明、当前可用、待继续观察和局势快照口径。
- `AppContainer`：把交互日志里的默认剧本、命令结果、规则校验和规则结算口径改为当前战局、军令结果和战局判定口径。
- `DiplomacyPanelView`：把外交边界说明、`boundaryNote` 展示、旧剧本势力 fallback 和善后进度 `N/M` 改为玩家语义。
- `EventLogView`、`RegionInspectorView`、`UnitInspectorView`、`UnitTooltipView`、`GeneralProfileView`、`UnitNode`：继续收口斜线、横线和 `N/M` 兵力显示。
- 文档：`README.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/plan/plan.md`。

## 3. 关键改动

- `ReleaseChecklistView` 的可见标题改为“战局说明 / 地图标识 / 当前可用 / 待继续观察”，快照结论改为“当前快照：可继续观察”，不再把该面板描述成发布或实机验收门禁。
- `AppContainer` 的存档失败、新局重置、自动回合、州郡经营、外交行动、玩家方面军令和普通军令反馈改为“当前战局 / 军令结果 / 战局判定 / 暂不可用”口径。
- `DiplomacyPanelView` 不再直出 `boundaryNote`，统一经 `displayBoundaryNote` 转成外交档案、后续处置、善后重点等玩家语义；旧剧本 fallback 改为历史势力或当前战局角色。
- `EventLogView` metadata 用中文逗号拼接；`RegionInspectorView`、`UnitInspectorView` 列表摘要用顿号；单位提示、将领所属军队和地图单位短标不再显示 `N/M`。

## 4. 架构边界

- 不改变 `GameState`、存档 schema、命令结构、规则执行、AI 决策、地图数据或 SpriteKit 布局规则。
- 不改 enum rawValue、内部审计 id 或数据 schema；只处理玩家可见文本、展示层 formatter 和辅助读法。
- 不运行 Xcode build、XCTest、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full、性能测试或 app 启动。

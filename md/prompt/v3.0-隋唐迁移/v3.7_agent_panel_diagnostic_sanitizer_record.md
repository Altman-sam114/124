# v3.7-preflight.61 AI 诊断净化口径对齐

完成日期：2026-07-06

## 目标

在 MapEditor 导出元数据 fallback 收口之后，继续处理 AI / 朝堂诊断中仍可能显示旧诊断口径的问题，让 `TurnManager.userFacingDiagnostic` 源头诊断和 `AgentPanelView` 面板展示与战报面板的诊断净化口径保持一致。

## 实现范围

- `TurnManager.userFacingDiagnostic(_:)` 与 `AgentPanelView.displayDiagnosticText(_:)` 将 `Heinz Guderian` / `Guderian` 显示为“历史总管”。
- `local-model` / `model` 改为“本地军议来源”/“军议来源”。
- `legacy pipeline` 改为“备用军议路径”。
- `RuleEngine` 改为“军令校验”。
- `AgentPanelView.displayDiagnosticText(_:)` 改为先执行 raw id 正则净化，再执行普通工程词替换，避免 `war_directive_*` 被半清洗成 `war_军令_*` 后失去匹配。
- `AgentPanelView` 的命令结果标题和本地 provider 展示同步走玩家语义。
- 两处 `sanitizeRawIdentifiers(in:)` 补齐 `war_directive_*`、`player_directive_*`、`player_operation_*`、`submission_handoff_*`、`submission_aftermath_*`、`diplomacy_event_*`、`court_decision_*`、`ruler_decision_*`、`hex_*`、`division_*`、`unit_*`、`command_*` 等常见审计 id 净化。
- `TurnManager` 中无行军防区的跳过诊断从“旧管线”改为“备用军议路径”。

## 边界

- 不改 `AgentDecisionRecord`、`WarDirectiveRecord`、`CourtDecisionRecord` 或任何存档 / 日志 schema。
- 不改 AI 决策、prompt、解析、命令结构、规则执行或运行时状态。
- 不触碰 `EventLogView`、`DiplomacyPanelView` 的已完成逻辑。

# v3.7-preflight.4 渡口和港口地图标识记录

完成日期：2026-07-05

## 1. 本轮定位

本轮继续补 v3.7 发布候选前置的地图可读性缺口，目标是把总提示词中“渡口 / 港口图标”做成最小运行时显示。

本轮不是水战、河流规则、渡河移动规则或水路补给规则；渡口和港口只是 scenario key location 进入运行时后的只读地图标识，不改变移动、攻击、占领、补给、前线或部署权威。

## 2. 实现摘要

### 2.1 运行时地点标识

`MapState` 新增：

- `MapFeatureKind`
- `MapFeatureMarker`
- `featureMarkers: [MapFeatureMarker]`

`featureMarkers` 从 scenario `keyLocations` 加载，保留 id、name、kind、coord、faction、objectiveId。旧存档或旧 JSON 缺少该字段时，`MapState.init(from:)` 会默认解码为空数组。

### 2.2 数据接入

`DataLoader.makeMapState(from:)` 会把 `ScenarioDefinition.keyLocations` 映射为 `MapFeatureMarker`，并放入 `MapState.featureMarkers`。

`wude_618_scenario.json` 新增 4 个水路地点：

- `loc_pujin_ferry`：蒲津渡，`kind=ferry`，关中东出渡口标识。
- `loc_mengjin_ferry`：孟津渡，`kind=ferry`，洛阳北向渡口标识。
- `loc_liyang_port`：黎阳津，`kind=port`，河北/黎阳仓水路节点标识。
- `loc_luokou_port`：洛口津，`kind=port`，洛口仓水路节点标识。

这些地点不进入 `objectives`，不改变胜负规则。

### 2.3 SpriteKit 显示

`BoardScene.drawTiles` 按 coord 把 `featureMarkers` 传给 `HexNode`。

`HexNode` 对 `kind=ferry` 绘制小渡船标识，对 `kind=port/harbor` 绘制小帆船/港口标识。图标和现有城池、关隘、粮仓一样是代码绘制，不引入外部素材。

## 4. 架构边界

- `featureMarkers` 是展示数据，不是战术权威。
- 渡口/港口不改变 `HexTile.controller`、`Division.coord`、`regionToTheater`、`hexToTheater` 或 `hexToFrontZone`。
- 不新增 `Command`、`ZoneDirective`、validator、executor 或水战规则。
- 不新增 objective，不影响 v3.7-preflight 胜负条件。
- 不引入外部图片、第三方库或 asset catalog。

## 7. 遗留事项

- 渡口/港口图标尚未经过真实 SpriteKit 截图检查；图标尺寸、遮挡、层级和动态字体旁路需授权运行时验证。
- `featureMarkers` 暂只用于显示；后续若做水路移动、渡河、港口补给或水师规则，必须另走 `Command` / validator / executor。
- 地图仍缺正式资产、前线墨线、AI 计划箭头和完整发布候选重测。

# v3.7-preflight.11 AI 太守主动经营记录

完成日期：2026-07-05

## 1. 目标

在 v3.7-preflight.10 已有玩家侧 `Command.governRegion` 的基础上，补上 AI/观战自动回合的最小太守经营路径：

- AI 太守能把朝堂记录中的州郡关注点转化为最多一条州郡经营命令。
- 经营命令必须继续走 `Command -> CommandValidator -> RuleEngine -> CommandExecutor`。
- 不直接改 `GameState`、hex controller、军队位置、动态方面、前线或行军防区。

## 2. 实现范围

### 2.1 AI 回合接入点

`TurnManager.executeDirectiveEnvelope` 在完成以下事项后执行太守经营：

1. `WarCommandExecutor` 执行战争 directive。
2. 写入 `RulerDecisionRecord`。
3. 写入 `CourtDecisionRecord` 和朝堂事件。

随后调用 `executeGovernorCommands(for:in:courtRecord:)`，并在 `.endTurn` 前完成最多一条 `Command.governRegion`。

### 2.2 州郡选择

候选州郡分两层：

- 优先使用 `CourtDecisionRecord` 中 `.governor` 步骤的 `targetRegionIds`。
- 其余州郡按评分排序，评分关注粮仓低、道路低、治安阻力、顺从不足和己方驻军。

候选必须满足：

- `region.controller == faction`
- `region.isPassable == true`
- 州郡内至少存在一个实际由该势力控制的 hex

### 2.3 政策选择

政策复用 `RegionGovernancePolicy`：

- `supplyValue <= 2`：优先屯田，再修道，再安民。
- `infrastructure <= 2`：优先修道，再屯田，再安民。
- 其他：优先安民，再屯田，再修道。

返回命令前必须检查：

- `policy.canApply(to:)`
- 当前势力府库能承担 `policy.cost`

### 2.4 结果记录

`CommandResultSummary` 新增：

```swift
static func systemCommand(
    idPrefix: String,
    commandIndex: Int,
    command: Command,
    result: CommandResult
) -> CommandResultSummary
```

AI 太守命令结果通过该 summary 进入 `AgentDecisionRecord.commandResults`。命令被拒绝时，`TurnManager` 会把拒绝原因追加到 diagnostics。

## 3. 边界

- 不新增新的内政状态机。
- 不改变 `Command.governRegion` 的校验和执行语义。
- 不直接修改 `RegionNode`、`EconomyState`、hex controller、unit、theater、front 或 deploy。
- 不处理 AI 使者外交、归附、借兵或地图归属转移。
- 不做多州郡排程；每个 AI/观战自动回合最多一条经营命令。

## 6. 遗留风险

- AI 太守主动经营尚未经过真实 AI/观战多回合运行验证。
- 经营策略是 deterministic 最小启发式，不是完整长期内政规划。
- 安民仍只更新 `OccupationState`，未接入完整民心、天命或叛乱事件链。
- AI 使者主动外交和完整归附事件链仍待后续。

# v3.7-preflight.52 命令错误与源头战报可见文案收口记录

完成日期：2026-07-06

## 目标

本轮在 v3.7-preflight.51 之后继续做发布候选前置补洞，聚焦 `Command` / `RegionCommand` / `TheaterDirectiveDecoderError` / `CommandIntentAdapterError` / `WarCommandExecutor` / `CommandExecutor` / `SupplyRules` 等源头产生的命令结果、拒绝原因、推进战报和撤退战报，避免这些文本进入战报、Agent 面板、总管面板或交互日志时直出 raw 坐标、raw id、英文工程词或底层解析细节。

## 范围

- 命令与州郡命令展示名。
- 元帅军令解码错误与 legacy 命令映射错误。
- 方面军令拒绝、州郡控制权变化、动态推进、行军、撤退和补给结算战报。
- 总管面板与玩家交互日志中的防区名 fallback。
- Agent / 战报 / 外交面板以及 `TurnManager` 诊断文本中的 raw agent id 正则净化。

## 非目标

- 不改变 `Command`、`RegionCommand`、`ZoneDirective`、`TheaterDirective`、record id、schema、rawValue、JSON key、命令校验、AI 决策、战斗/补给/移动/经济数值或 `RuleEngine` 执行语义。
- 不运行 Xcode build/test、XCTest、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full、性能测试或 app 启动。

## 并发扫描

本轮继续按人工要求使用只读子 agent：

- 命令/规则方向：扫描 `WWIIHexV0/Commands` 与 `WWIIHexV0/Rules` 中可能流入 UI 的错误描述、命令展示名、战报源头和拒绝原因。
- UI/Agent 方向：扫描 `App`、`UI`、`Agents`、`Turn` 中命令结果、诊断、战报和记录摘要展示路径，定位绕过已有净化函数的 raw id 风险。

子 agent 均只读，未编辑文件，未运行重测试。

## 关键变更

- `Command.displayName` 的移动命令不再显示 `q,r`，改为“行军至目标地块”；`RuleEngine` 成功结果改为“军令已执行”。
- `RegionCommand.displayName` 不再输出 `RegionMove/RegionAttack/RegionHold/RegionResupply`、军队 id、region rawValue 或 `unknown`，改为中文动作名。
- `CommandIntentAdapterError`、`AgentCommandMappingError`、`AgentDecisionParserError` 和 `TheaterDirectiveDecoderError` 不再拼接军队 id、agent id、防区 id、州郡 id、directive id 或底层 JSON / DecodingError 详情。
- `WarCommandExecutor` 的方面军令拒绝、州郡控制权变化和动态推进战报不再拼 `command.displayName` 坐标、不再写“动态推进”，改为动作名和“推进范围”。
- `CommandExecutor` 的行军与动态推进战报不再直出目标坐标。
- `SupplyRules` 的撤退战报和位置 fallback 不再直出起止坐标。
- `GeneralCommandPanelView` 与 `AppContainer` 对防区名做展示 fallback，避免 `NorthWest`、`theater_*`、`front_*` 等 raw name 进入面板或交互战报。
- `ZoneCommanderAgent` 元帅 fallback 诊断不再显示 `config.id`，改为“当前元帅”。
- `AgentPanelView`、`EventLogView`、`DiplomacyPanelView` 和 `TurnManager` 的 raw id 净化补齐 `marshal_*`、`mock_*`、`sovereign_*`、`strategist_*`、`diplomat_*`、`governor_staff_*`、`march_commander_*`、`general_staff_*`。

## 遗留风险

- 本轮没有真实启动 app、执行命令、触发 AI 回合或打开战报/Agent/总管面板确认视觉效果。
- 内部 record id、agent id、zone id、region id、rawValue 和 JSON schema 仍按兼容合同保留。
- 完整 v3.7 发布候选仍需要授权构建、启动、AI 多回合、界面点击、MapEditor 操作烟测和云端结果包验收。

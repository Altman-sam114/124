# v3.7-preflight.100 MapEditor 非法 unit faction 导入诊断收口

完成日期：2026-07-07

## 切片说明

切片 ID：P3 MapEditor 非法 unit faction fallback。

来源：总提示词 §0.2 / §0.4 当前可执行队列；v3.7-preflight.99 并发只读审计已确认 `MapEditorGameResourceBridge.makeDocument` 仍存在 `Faction(rawValue: unit.faction) ?? .allies`。

当前基线：`update_log.md` 顶部为 v3.7-preflight.99，核心文档和流程图均标注 P3 待修；当前工作区已有 v3.7-preflight.99 相关未提交改动，本轮只在其上追加 P3 收口。

## 目标

- MapEditor 默认游戏资源导入时，非法 `unit.faction` 不再静默变成旧 `.allies`。
- 坏 unit 必须有可见诊断或跳过记录，方便人工修数据。
- 不改变主游戏 `DataLoader`、`Faction` enum、JSON schema、MapEditor 导出 schema、命令管线、规则执行或动态权威。

## 实现

- `MapEditorGameResourceBridge` 新增 `MapEditorGameResourceImportDiagnostic` 和 `MapEditorGameResourceImportResult`。
- `loadDefaultDocument()` 保持兼容，内部委托给新的 `loadDefaultDocumentResult()` 并只返回 document。
- `makeDocument(scenario:regionData:)` 返回导入结果；解析 `scenario.initialUnits` 时先显式解析 `Faction(rawValue: unit.faction)`。
- 如果 unit faction 无法识别，跳过该 unit，并记录 `军队 <id> 的势力值 <raw> 无法识别。`，不再 fallback `.allies`。
- `MapEditorViewModel.loadDefaultGameResources()` 改用 `loadDefaultDocumentResult()`，读取成功后把跳过数量和前三条原因写入状态消息。

## 验收标准

- P3 旧命中 `Faction(rawValue: unit.faction) ?? .allies` 消失。
- 非法 unit faction 的处理路径明确为跳过并记录诊断。
- MapEditor 读取默认资源时可以向用户展示诊断，不再静默修正坏数据。
- 总提示词 §0.2 / §0.4、`update_log.md`、`md/flow/flow.md` 和 `md/flow/flowchart.md` 均同步 P3 已收口状态。

# v3.7-preflight.24 善后完成状态提示

## 目标

在 v3.7-preflight.21/.22/.23 已有善后处置审计、进度摘要和 AI 未处置优先治理之后，补上玩家可见的完成状态：最新善后记录应能显示还剩多少受影响州郡未处置，以及本次善后是否已全部产生处置记录。

## 范围

- 在 `DiplomacyState` 中新增未处置善后州郡数量和善后处置完成状态的只读 helper。
- 在 `DiplomacyPanelView` 的“善后”区显示待处置数量和完成状态。
- 调整 `TurnManager.aftermathGovernorRegionIds`：当最新善后记录的受影响州郡都已有处置记录后，不再把该善后记录作为 AI 太守特殊优先来源。
- 保持 AI 太守每回合最多一条既有 `Command.governRegion`。
- 更新发布前检查面板、README、核心流程文档、流程图、计划文档和更新日志。

## 非目标

- 不清零、降级、删除或解决 `SubmissionAftermathRecord`。
- 不删除或压缩 `SubmissionAftermathGovernanceRecord`。
- 不新增命令类型，不增加 AI 回合行动次数。
- 不改变 `Command.governRegion` 的校验、消耗或执行效果。
- 不新增忠诚、叛乱、贡赋、俘虏、安置、治安长期状态机或额外资源变化。
- 不批量处理历史善后压力；当前仍只读取最新一条善后记录。

## 数据流

```text
latest SubmissionAftermathRecord
  -> affectedRegionIds
  -> linked SubmissionAftermathGovernanceRecord
  -> DiplomacyState 统计未处置 region 数量 / 完成状态
  -> DiplomacyPanelView 显示待处置数量和状态
  -> TurnManager.aftermathGovernorRegionIds
  -> 已全部处置时退出该善后的特殊优先队列
  -> 其他太守候选仍按既有评分进入 Command.governRegion
  -> RuleEngine
```

## 架构边界

- Hex 仍是战术权威；本轮不修改 `HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- Region 仍是战略聚合层；本轮只统计受影响州郡是否已有处置记录。
- 善后完成状态是审计和 UI 口径，不代表善后压力在规则层被清除。
- AI 完成后退出特殊优先队列只影响候选排序来源，不改变命令、校验、消耗或治理效果。

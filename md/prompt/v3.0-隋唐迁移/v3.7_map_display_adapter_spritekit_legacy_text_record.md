# v3.7-preflight.75 MapDisplayAdapter / SpriteKit 地图展示入口 legacy 文案收口记录

## 目标

收口 `MapDisplayAdapter` 输出给 SpriteKit 地图标签和详情面板入口的 legacy 地名、目标名、旧势力词和 raw id 展示风险，并对详情面板既有 id / faction 展示做不扩展状态合同的兜底。

## 范围

本轮主改 `MapDisplayAdapter` 的展示 helper，并对 `RegionInspectorView` / `UnitInspectorView` 中既有 id 与势力名展示做局部兜底；不改 `HexDisplayState`、`RegionInspectorState`、`UnitInspectorStrategicState` 字段结构，不改地图 JSON、rawValue、objective id、keyLocations、`RegionNode` / `HexTile` / `Objective` 存储、`hexToTheater`、`hexToFrontZone`、`regionToTheater`、`regionToFrontZone`、AI、命令、规则或存档 schema。

## 实现摘要

- `HexDisplayState.cityName` / `fortressName` 由 `MapDisplayAdapter.hexDisplayState` 输出前进入展示净化，避免 `HexNode` 直接画出 legacy 地名或 raw id。
- `RegionInspectorState.objectiveNames` 在 adapter 内先净化 objective name，再交给州郡详情面板。
- `RegionInspectorState.objectiveStatus` 中 legacy `Faction.displayName` 经 adapter 局部势力展示 helper 收口。
- `RegionInspectorView` / `UnitInspectorView` 不再依赖新增展示字段；对既有 region / theater / front zone id 和 legacy faction display 做局部净化兜底，避免 raw id 或旧势力词直出。

## 并发扫描结论

- Avicenna 确认本轮适合写成窄版 `.75`：只锁定 `MapDisplayAdapter` 输出给 SpriteKit 地图标签与详情面板的展示名净化，并给出文档同步清单。
- Turing 定位四个直接风险：`cityName` / `fortressName` 进入 `HexNode` 标签、`objectiveNames` 原样进入 `RegionInspectorState`、详情面板 region / theater / front zone 展示仍可能回到 raw id、旧势力名可能直出。本轮已按这些点做 adapter 与面板展示层最小修补。

## 边界

- `HexNode`、`BoardScene`、`UnitNode` 不需要本轮改动；修 adapter 后地图标签和详情入口自然收口。
- `Division` 对象仍原样传递；下游 `UnitInspectorView` / `UnitTooltipView` 已有单位展示名净化。
- inspector state 仍只传递既有 id / faction 字段，不增加展示名字段。
- 动态方面 / 防区权威仍来自 `hexToTheater` 和 `hexToFrontZone`。
- `regionToTheater` / `regionToFrontZone` 仍只作初始、dominant 或 fallback 依据。
- 不改地图数据、objective id、keyLocations、胜负规则、AI 决策、命令结构或规则执行。
